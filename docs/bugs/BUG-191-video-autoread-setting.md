## BUG-191 · 关闭查词时自动阅读后视频字幕查词仍自动阅读
- **报告**：2026-06-11（用户：小说页关「查词时自动阅读」生效，视频字幕查词仍自动发音）
- **真实性**：✅ 真 bug — 根因 `hibiki/lib/src/media/sources/reader_hibiki_source.dart:456`（修复前 `autoReadOnLookup` getter 优先读全局静态 `readerSettings?.autoReadOnLookup`）。`readerSettings`（`reader_hibiki_source.dart:373`）是**阅读器页面专属**的 per-profile 设置快照，只在 app 初始化（`app_model.dart:1305`）和打开阅读器页（`reader_hibiki_page.dart:450`）赋值/刷新。视频页（`video_hibiki_page.dart`，走 `DictionaryPageMixin`）**从不解析自己的 profile、也从不刷新 `readerSettings`**，查词判定点 `dictionary_page_mixin.dart:438` 读 `ReaderHibikiSource.instance.autoReadOnLookup` 时拿到的是「最后一次打开阅读器的 profile / app 启动 default」的陈旧快照，而非视频当前应生效的偏好。小说页正常是因为打开阅读器时切了 profile 并刷新了 `readerSettings`。
- **[x] ① 已修复** — `reader_hibiki_source.dart:456` 的 `autoReadOnLookup` 改为直接读本 source 的 profile-aware 偏好缓存 `getPreference<bool>(key:'auto_read_on_lookup')`（`MediaSource._preferences`，由 `AppModel.refreshPrefCache` 在每次 profile 切换时 `refreshPreferencesFromDb` 重载，见 `app_model.dart:1512`）；`toggleAutoReadOnLookup` 同步改为只走 `setPreference`，删掉冗余的 `if(readerSettings!=null)` 第二写分支（两条路径写穿 DB 同一行 `src:reader_ttu:auto_read_on_lookup`，无迁移、无双源）。提交：见本轮 commit。
- **[x] ② 已加自动化测试** — `hibiki/test/media/sources/reader_hibiki_source_test.dart` group「autoReadOnLookup is profile-aware (TODO-080B)」3 例：①核心复现守卫（视频上下文 DB=false 但陈旧静态 readerSettings=true 时，`autoReadOnLookup` 必须为 false；撤修复转红，已实测红→绿）②profile 切换后立即跟随 ③toggle 写穿 DB 且读写对称。
- **备注**：TODO-080 子问题2；任务A（字幕双层）已先合 develop。范围只动 `reader_hibiki_source.dart`，未改 `video_hibiki_page.dart`——根因在共享 source 而非视频页，无需在视频页加 profile 刷新。fontSize/writingMode 等其它阅读器专属属性保留 `readerSettings` 优先（不在视频/mixin 查词路径上，不受影响）。真机发音留用户验证。
