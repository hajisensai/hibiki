## BUG-314 · TODO-524: Windows desktop drag-drop import can miss targets or fail silently
- **报告**：2026-06-18（用户：0.9.24-debug.5191 “电脑没办法拖动导入了”）
- **真实性**：✅ 真 bug。沿真实 Windows 拖拽链路验真伪：`desktop_drop 0.5.0` 的 Windows 插件用 Win32 `RegisterDragDrop`/`IDropTarget` 接收拖入文件，Dart `DropTarget` 在 `DropDoneDetails` 同时提供 `globalPosition` 与 `localPosition`；Hibiki wrapper 只把 `localPosition` 透给业务，书架/视频库再用页面 `RenderBox.localToGlobal(localPosition)` 二次换算，容易在嵌套/缩放/页面与 wrapper render box 不一致时命中错位。隐藏或非当前 route 的 DropTarget 也仍可能收到事件，且书架/视频/词典 surface 对“已识别但当前 surface 不能处理”的文件存在静默 ignore。
  - 根因：`hibiki/lib/src/media/drag_drop/hibiki_file_drop_target.dart:26` 使用 `detail.localPosition`；`hibiki/lib/src/pages/implementations/reader_hibiki_history_page.dart:2224` 与 `hibiki/lib/src/pages/implementations/home_video_page.dart:281` 把该坐标再次转全局；`hibiki/lib/src/pages/implementations/dictionary_dialog_page.dart:405` 对非词典拖入静默返回。
- **[x] ① 已修复** — 本提交：
  - `hibiki_file_drop_target.dart` 改为把 `DropDoneDetails.globalPosition` 作为业务回调坐标；新增 `enabled` / `debugLabel`、route 可见性运行时门控与 `[hibiki-drop]` enter/update/exit/done 诊断日志。`DropTarget(enable:)` 只跟随平台/功能启用状态，避免启动期 route 不可见时原生 listener 没注册，导致真实 Windows 拖入完全收不到事件。
  - 书架与视频库使用全局坐标直接命中卡片，不再把 wrapper 的局部坐标二次 `localToGlobal`；识别到文件但当前 surface 不能处理时走 `DropIntent.unsupportedSurface` 并显示 SnackBar/Toast，不再静默 ignore。
  - 词典页新增 `HibikiFileDropTarget`，`.zip/.dsl/.mdx`（以及作为词典包配套的 `.css`）从查词主页路由到 `DictionaryDialogPage(initialImportPaths:)`，管理页自身拖入也给出 unsupported 反馈。
  - 视频播放页保留 BUG-133 页级 DropTarget 守卫，并对字幕/音频/其它已识别文件分别走 OSD 反馈与 `[hibiki-drop]` 日志。
- **[x] ② 已加自动化测试** — `hibiki/test/media/drag_drop/drop_decision_test.dart` 覆盖 `unsupportedSurface` / unknown-only `ignore` 边界；`hibiki/test/media/drag_drop/drag_drop_platform_guard_test.dart` 守卫 wrapper 仅集中依赖 `desktop_drop`、使用 `globalPosition`、运行时 route 门控、注册期 `enable: enabled`、调用点不再二次坐标换算且错误 surface 有反馈；`hibiki/test/pages/dictionary_dialog_layout_static_test.dart` 覆盖词典拖入可从查词主页带 `initialImportPaths` 进入管理页；既有 `hibiki/test/pages/video_subtitle_fixes_guard_test.dart` 保留 BUG-133 页级目标。
- **验证**：`dart format .`（无关旧测试换行噪音已撤回）、`flutter test test/media/drag_drop/ test/pages/video_subtitle_fixes_guard_test.dart test/pages/dictionary_dialog_layout_static_test.dart`（69 passed）、`flutter analyze`（No issues found）。
- **Windows 真拖拽证据**：`.codex-test/todo-524/01-book-drop-log-after-routeguard.txt` 记录 WinForms/OLE 文件源拖入书架后 `[reader-shelf] enter/update/done` 与 book 分类；`.codex-test/todo-524/02-dictionary-drop-log.txt` 记录真实 CF_HDROP 同时携带 dictionary zip 与 css；`.codex-test/todo-524/03-wrong-target-video-on-books-log.txt` 记录视频拖到书架错误 surface 时仍进入 `[reader-shelf] done` 与 unsupported 分类。截图与拖拽源脚本同目录。
- **备注**：本轮真实 Windows OLE 路径确认了 listener 注册、书架导入、错误目标反馈日志与词典 zip+css 文件列表；受桌面自动化插件启动失败与前台窗口切换干扰，视频库/播放页/词典页的肉眼矩阵未完全自动跑通，交给负责人复核时应优先补这三项手动真拖拽。
