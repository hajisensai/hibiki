## BUG-112 · 有声书暂停后点「前进/后退」(按句模式) 会跳两次（下一句跳回当前句、上一句乱跳）
- **报告**：2026-06-07（用户：「有声书音频暂停以后，点击前进后退会怪怪的」→ 追问确认：「会跳转两次，下一句会跳回这一句，上一句可能会跳」）。采番注：BUG-110 已被并发「竖排高亮遮字」、BUG-111 已被并发「桌面缩放阅读器半屏」占用，本条取 112。
- **真实性**：✅ **真 bug（沿真实代码路径定位 + TDD 红灯精确复现）**。前进/后退默认走「按句」模式（`reader_hibiki_source.dart:480` `skipActionSeconds` 默认 0）→ `AudiobookPlayerController.skipToPrevCue/skipToNextCue` → `skipToCue`（`audiobook_controller.dart:604`）。`skipToCue` 已把目标 cue 作为**权威值**写入 `_currentCue` + notify，并立 `_explicitSeekInFlight` 抑制窗（BUG-061），靠 `_player.seek(index:)` 跳位。根因在 `_updateCurrentCue`（`audiobook_controller.dart:801`）顶部抑制窗的放行判据 `reachedExplicitSeekTargetForTesting`（`:1191`）= `posMs >= targetMs - 300ms`：**暂停态下没有真实播放推进，但 `seek` 仍会吐瞬态位置 tick**（注释 :168-171 自述「0/旧章首/旧位置」）。这些瞬态 tick 被该判据误判为「落定」并**清旗 + 用瞬态旧位置 `findCueIndex` 重解析**，把刚写入的权威 cue 覆盖回旧句：① 前向（下一句）——暂停在本句末尾时旧位置已落进 `target-300ms` 窗内 → 高亮跳到下一句后**又跳回当前句**；② 后向（上一句）——旧位置本就高于 target → 判据恒真 → 第一帧瞬态就乱解析 → **上一句乱跳**。播放态（BUG-061 路径）靠真实位置爬到 target 才落定，本就正确。
