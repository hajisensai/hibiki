## BUG-177 · 插画查看器无法右键复制(win)/长按分享(android)
- **报告**：2026-06-11（用户：飞书巡检 TODO-093）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/pages/implementations/illustrations_viewer_page.dart`。
  「查看插画」走 `IllustrationsViewerPage` 的全屏画廊 `_FullScreenGallery`（纯 Flutter `Image.memory` + `InteractiveViewer`），**完全没有任何复制/分享入口**——Windows 无右键菜单、移动端无长按分享。TODO-023 给的复制/分享只覆盖阅读器正文 WebView 内联图片（走 `onImageContextMenu`/`onImageLongPress` JS 桥 + `_readerImageFileForUrl`），WebView 专属，不经过插画查看器这条纯 widget 路径。数据结构层面：`_FullScreenGallery` 原本只持有 `List<Uint8List>`（字节），`_extractImages` 读完字节后丢弃了源 `File`，而复用的两条底层能力 `Share.shareXFiles(XFile(file.path))` 与 `HibikiChannels.clipboardImage.invokeMethod('copyImageFile', {'path': file.path})` 都需要磁盘文件路径。
- **[x] ① 已修复** — commit `84bf3d166`。改 `_images` 数据结构为 `List<_Illustration>{bytes, file}` 保留源磁盘文件；`_FullScreenGallery` 新增 `_copyCurrentImageToClipboard`（Windows 原生剪贴板 channel）/`_shareCurrentImage`（移动端 `Share.shareXFiles`）/`_showImageContextMenu`（Windows 右键弹复制菜单），复用 TODO-023 范式与已有 i18n key（`reader_copy_image`/`copied_to_clipboard`/`reader_image_file_unavailable`/`reader_image_copy_failed`/`reader_image_share_failed`）。接线：Windows `onSecondaryTapDown` 右键复制、移动端 `onLongPress` 分享、顶栏 `actions` 放复制/分享图标按钮做可发现入口。未新增 i18n key。
- **[x] ② 已加自动化测试** — `hibiki/test/pages/illustrations_viewer_actions_guard_static_test.dart`（源码守卫：断言保留源 File、Windows `copyImageFile` channel + 右键菜单、移动端 `Share.shareXFiles` + 长按、不退化成 `Clipboard.setData`）。
- **备注**：headless 无法真触发 PopupMenu / 系统分享面板 / 原生剪贴板 channel，用源码守卫接线。**Windows 右键复制 + 移动端长按分享需真机/模拟器肉眼复测**（CLAUDE.md 验证纪律）。
