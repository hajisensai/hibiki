## BUG-430 · Windows IME 激活时全表面快捷键失效
- **报告**：2026-06-26（用户：）
- **真实性**：✅ 真 bug。根因：Windows 微软 IME 激活时，Flutter 引擎把 `KeyDownEvent.logicalKey` 改写成 `LogicalKeyboardKey.process`（输入法占用逻辑键），而快捷键解析 `HibikiShortcutRegistry.resolveKeyboard`（`hibiki/lib/src/shortcuts/shortcut_registry.dart:241`）用 `InputBinding.==`（`hibiki/lib/src/shortcuts/input_binding.dart:233`，只比 logicalKey+modifiers）做精确相等 → 与任何 binding 的 logicalKey 都不等 → 恒返回 null → 阅读器/首页/全局所有键盘快捷键全部失效。`physicalKey`（USB HID 扫描码）不受 IME 改写，是唯一可靠的回退判据。两个直读 `event.logicalKey` 的纯函数 `resolveReaderSpaceOverride` / `resolveReaderArrowPageTurn`（`hibiki/lib/src/shortcuts/reader_space_override.dart:14/40`）同样失效：IME 下有声书裸 Space 不再播放/暂停、RTL 书裸左右键翻页方向反转（落回注册表写死的 Right=前进）。
- **[x] ① 已修复** — `resolveKeyboard` 加可选参数 `PhysicalKeyboardKey? physicalKey`；正常路径（精确相等遍历）逐字不变，仅当 `key == LogicalKeyboardKey.process && physicalKey != null` 时启用物理键回退分支（在 modifiers 完全相同前提下按 `InputBinding.physicalKey` 比对）。`InputBinding` 新增 `static final Map<LogicalKeyboardKey,PhysicalKeyboardKey> _logicalToPhysical`（覆盖与 `_knownKeys` 相同的非 game* 键集）+ 派生 getter `physicalKey`；`==`/`hashCode`/`serialize` 不动。两个 override 纯函数也加 `physicalKey` 参数，`key==process` 时按物理键还原 Space/方向键语义。四调用点透传，门控 `physicalKey: focusedEditableText() == null ? event.physicalKey : null`（文本框 composing 时传 null 关回退，保证 IME 打字不被吞）：`reader_hibiki/caret.part.dart`（directReaderAction + reader/audiobook scope chain + spaceOverride + arrowOverride）、`home_page.dart`（home + global scope）、`global_navigation.dart`（`_handleGlobalBack` global scope）。`home_page.dart` 的 `arrowTraversalDirection`（焦点方向导航降级，非数据破坏）列为 follow-up 未纳入本轮。**已知限制：physical↔logical 一一对应仅在 US-QWERTY 物理布局下成立**；非美式物理键盘（AZERTY / 德语 QWERTZ）上字母/符号键的物理位与逻辑键不一致，回退可能错绑到相邻键。因回退仅在 `key==process`（IME 激活，本就全失效）时启用，最坏只是“按某键触发了相邻快捷键”，绝不会比现状（全部失效）更糟；Space/方向键物理位在所有常见布局一致，实际不受布局影响。提交哈希：见下。
- **[x] ② 已加自动化测试** — `hibiki/test/shortcuts/shortcut_registry_ime_test.dart`（新增）：合成 `(LogicalKeyboardKey.process, physicalKey)`，覆盖回退命中（process+pageDown→readerPageForward）/ physicalKey:null 仍 null / key!=process 不走回退（合取条件）/ modifiers 精确（Ctrl+Digit1→homeTabBooks，无 Ctrl 不命中）/ 正常路径（真实 pageDown）不变 / 跨 scope 不泄漏。`hibiki/test/shortcuts/input_binding_test.dart`（扩展）：physicalKey 派生映射 + `_knownKeys⊇` 守卫（遍历全部非 game* 键断言 `physicalKey != null`，防漏键）+ `==`/`hashCode`/`serialize` 改前后不变式。`hibiki/test/reader/reader_space_pause_test.dart`（扩展）：spaceOverride / arrowOverride 的 IME 回退（process+physical Space→播放暂停、RTL process+physical Left→前进不反转、physicalKey null 不覆写、带修饰不覆写）。红→绿已验证：临时禁用回退分支后仅 “fallback resolves” 两条用例失败，恢复后 308 测试全绿。
- **备注**：根因 B（IME 改写 logicalKey）。follow-up：`home_page.dart:280` `arrowTraversalDirection` 的 IME 物理回退（焦点导航降级，非数据破坏）；引擎若日后在 IME 下提供稳定逻辑键则可清理整套物理回退兼容层。真机门禁待验：Win 微软 IME 下阅读器正文（PageDown/方向/M/Ctrl+C 等）、首页 Ctrl+数字切 tab、查词框打字不误触、RTL 书与有声书裸 Space 翻页/播放方向正确。
