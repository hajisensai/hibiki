## BUG-111 · 桌面端放大「界面大小」后进阅读器，正文初始只铺半边（需手动 resize 才铺满）
- **报告**：2026-06-07（用户：「变 ui 大小文字没跟着变」→ 澄清为「app 窗口大小，阅读器里文字没变动，还是小窗口」；截图为纵书正文只占窗口右侧约 40%、左侧大片空黑）。复现条件：设置里「界面大小」≠100%（用户确认调过）；「重新调整下窗口大小就行了」＝resize 能修复。
- **真实性**：✅ **真 bug（沿真实代码路径定位）**。分页宽度来源 `reader_pagination_scripts.dart:1160` `pageWidth = dartW || window.innerWidth`——优先用 Dart 注入的 `dartPageWidth`（`reader_hibiki_page.dart:_buildReaderSetupScript` 取 `MediaQuery.of(context).size.width`）。scale≠1.0 时阅读器外包 `HibikiAppUiScale` + `HibikiAppUiScaleNeutralizer` 两层 `LayoutBuilder`/`FittedBox`，初始注入若早于缩放层把「还原后的真实视口宽」settle，`dartPageWidth` 取到被缩小的视口宽（≈窗口/scale，正好对应正文只铺 1/scale）。**根因不在「resize 不重排」**（`_syncPageSize` 工作正常，故 resize 能修），而在 content-ready 的初始重排校验**自我抹平**：`_onRestoreComplete`（`reader_hibiki_page.dart:2245`）/`_onChapterLoadComplete`（:2149）先把 `_lastSyncedWidth = 当前 MediaQuery 宽`，再 postFrame 调 `_syncPageSize`；而 `_syncPageSize` 判定是 `w != _lastSyncedWidth`——刚被设成相同值 → `widthChanged` 恒 false、初始重排恒 no-op，于是初始偏窄版面无任何兜底纠正，直到用户手动 resize 改变宽度才触发重排。
- **[x] ① 已修复** — 本提交。根因修（恢复被自我抹平的初始重排不变式，非补丁）：新增 `_paginatedWidth/_paginatedHeight` 字段，在 `_buildReaderSetupScript` 记下 JS 实际分页用的 `screenSize`（= dartPageWidth/Height 权威值）；content-ready 收尾改用 `_lastSyncedWidth = _paginatedWidth`（而非当前 MediaQuery）作基线。这样 settle 后的真实视口宽若与「已分页宽度」不等，`_onRestoreComplete` 的 postFrame `_syncPageSize` 即检出 `widthChanged` 并重新分页铺满；100%（中和器直通、MQ 从首帧即真实）与既有 resize 路径行为不变。`_syncPageSize` 加一行诊断日志便于桌面复测确认。
- **[x] ② 已加自动化测试** — `hibiki/test/pages/reader_init_page_width_guard_static_test.dart`（源码守卫：断言 setup 脚本把 `screenSize` 记进 `_paginatedWidth/Height`、content-ready 基线取 `_paginatedWidth/Height`、整文件不再出现 `_lastSyncedWidth = screen.width` / `= screenSync.width` 这类当前 MQ 抹平写法；缩放层 settle 时序需真 WebView headless 不可跑）。
- **备注**：reader/WebView/缩放/布局类。`flutter analyze` 改动文件 0 issue + 守卫 3 绿 + 相邻 reader 守卫（neutralizer wired / bottom chrome gate / paginate lyrics）6 绿无回归。**Windows 真机肉眼复测待用户**：界面大小调到非 100% → 开书/进阅读器即满铺、无需手动 resize；调回 100% 不回归；运行中 resize 仍正常。可在 debug 控制台看 `[ReaderHibiki] _syncPageSize ... paginated=...` 确认初始重排是否触发及宽度数值。
