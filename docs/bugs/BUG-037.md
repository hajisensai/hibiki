## BUG-037 · 设置「同步与备份」页手机触摸上下滑动会跳跃
- **报告**：2026-06-04（用户：「同步与备份里面，上下滑动会跳跃」）。澄清确认：**手机触摸滑动**（Android/iOS）。
- **真实性**：✅ **真 bug（懒加载列表 extent 估算漂移），确定性复现**。沿真实代码路径定位：自滚动的设置详情（`MaterialSettingsRenderer.buildDetailContent`，`shrinkWrap:false`，手机点目的地走 `buildDetailPage` 窄屏 push 页 / 宽屏走 `settings_home_page.dart:173` 主从详情）用懒加载 `ListView.builder`。`RenderSliverList` 只布局可见 section，用**已布局子项的平均高**估算视口外子项的 extent；而同步页 section 高度悬殊（1 行开关 vs. 很高的 `_LanDiscoveryWidget`/URL 列表/`_HibikiServerConfigWidget`），估算值随滚动位置漂移 → `maxScrollExtent` 在手指下变化 → 弹道滚动按旧上界算的落点被重新 clamp → 视口跳跃。**与 BUG-004 不同**：004 是 touch 行回收→`_maybeRevealOnRepair` reveal 拽回（焦点机制，已 `f6ef60d27` 门控修复）；本条是 reveal 已门控后**剩下的结构性机制**——`settings_scroll_no_rollback_test.dart:138-139` 注释当时已点名「懒加载列表 extent 不稳」却当测试假象绕过，正是此 bug。根因点 = `material_settings_renderer.dart` 自滚动路径用懒加载 `ListView.builder`。冒烟枪：新测试实测同步详情页 `maxScrollExtent` 从顶部 1028px 漂到中部 846px（182px 摆动）。
- **[x] ① 已修复** — `5b79c3ffe`：Linus 式消除特殊情况——设置页 section 数量有界且小，懒加载零收益却制造不稳 extent。`shrinkWrap:false` 自滚动路径改为非懒加载 `SingleChildScrollView + Column(crossAxisAlignment: stretch, 全 section)`，全部 section 一次布局 → extent 精确恒定，弹道落点不再被二次 clamp，跳跃从源头消失。`shrinkWrap:true`（嵌入父滚动的 cupertino/桌面设置/阅读器快捷设置/display 4 个调用方）**保持** `ListView.builder(shrinkWrap:true)` 不变——shrinkWrap 本就布局全部子项、extent 已精确，不在 bug 路径。向后兼容：手柄整页翻页（`primary` 解析两控件同一 SDK 公式 + `HibikiPageScaffold` 的 `automaticallyInheritForPlatforms`）/ 焦点 `Scrollable.ensureVisible` reveal / `stretch` 交叉轴布局全部等价。opus 子代理审查 🟢 Ready（逐项用真实代码核实 eager-build 无副作用、翻页/reveal/布局零变化）。`flutter analyze` 0 issue、`dart format` 0 改动。
- **[x] ① 孪生同修** — `2af383162`：审计「同款其它发生点」时发现 `CupertinoSettingsRenderer.buildDetailContent`（`cupertino_settings_renderer.dart`）是同一 bug 的 Cupertino 孪生——其自滚动路径（`shrinkWrap:false`，宽屏 iPad/macOS-Cupertino master-detail 详情面板走到，由 `settings_home_page.dart:173` 按 `isCupertinoPlatform` 选 renderer 触发）也是懒加载 `ListView.builder` 渲染同批高度悬殊 section。照 Material 同样手法改非懒加载 `SingleChildScrollView + Column`（`shrinkWrap:true` 路径保持 `ListView.builder(shrinkWrap:true)+NeverScrollable` 不变；裸 Column 在有界 Expanded 会 RenderFlex 溢出 BUG-009 R1，SingleChildScrollView 可滚不溢出）。注意 iOS **手机**窄屏走 `buildDetailPage`→`shrinkWrap:true`（嵌入 CustomScrollView，已非懒加载、安全），故手机两端本就覆盖，此孪生是 iPad/桌面-Cupertino 端的同根 bug。
- **[x] ② 已加自动化测试** — `5b79c3ffe`（Material）+ `2af383162`（Cupertino）· `hibiki/test/settings/settings_detail_scroll_stable_test.dart`：真实渲染器渲染同步详情页（hibikiServer 后端 + 5 个 URL，放大 section 高度悬殊），断言 `maxScrollExtent` 在**顶/中/底逐像素相等**；用例**参数化覆盖 Material 与 Cupertino 两渲染器**。修复前红（Material 实测 1028→846 漂移）、修复后绿。既有焦点 reveal 门控测试（`settings_scroll_no_rollback_test.dart` / `sync_settings_no_touch_scroll_test.dart`）仍绿，证明 reveal 行为未被破坏。`test/settings` 全量绿无回归。
- **备注**：设置/滚动/布局类。代码 + 单测绿、`flutter analyze` 0 issue、`dart format` 0 改动、opus 审 🟢。**真机肉眼复测原始失败路径待用户**：手机进设置→同步与备份（Hibiki 互联后端、内容超一屏）→ 上下滑动应平滑不跳（修复前 section 高度悬殊处必跳）。该修复同时惠及所有自滚动设置详情页（外观/查词等），非仅同步页。**审计其它同款点结论**：① 上述 Material + Cupertino 两渲染器（所有自滚动设置详情）已修；② **待评估、未动**：`dictionary_popup_native.dart:126`（原生弹窗词典结果，可变高词条自滚动懒加载）与 `dictionary_result_page.dart`（划词词典结果，仅 `lowMemoryMode` 退回懒加载时漂移，平时靠超大 `cacheExtent` 掩盖）——这两处词条富文本/振假名/释义渲染重，非懒加载有性能权衡，不照搬本修法，留待与用户确认后单独处理。其余列表已逐个排除（等高 / `shrinkWrap` 嵌入 / `NeverScrollable` / 非懒加载 List delegate）。
