## BUG-195 · 三星 OneUI 6.5 系统默认焦点框与 app 自绘焦点环双重重叠
- **报告**：2026-06-11（用户：）
- **真实性**：✅ 真 bug —— 根因是 Android 平台行为，不在 Flutter/Dart 代码里。Android API 26+ 起所有 `View` 的 `defaultFocusHighlightEnabled` 默认 `true`，退出 touch mode（键盘/手柄导航）后，框架会给当前获焦的 View 画一个系统默认焦点矩形。Hibiki 整个 UI 宿主在程序化创建的 `FlutterSurfaceView` 上，该系统焦点框落在宿主 View 上，与 app 自绘的键盘/手柄焦点环（`hibiki/lib/src/utils/components/hibiki_focus_ring.dart`，Flutter 层 CustomPaint/装饰）**双重重叠**。三星 OneUI 6.5 上这层系统框尤其明显。两层在不同渲染层（Android 原生 vs Flutter），代码上互不相干，纯属系统默认值未关闭。
- **[x] ① 已修复** —— `hibiki/android/app/src/main/java/app/hibiki/reader/MainActivity.java:112,126-133`：在 `onCreate` 的 `super.onCreate(savedInstanceState)` 之后调用新增的 `disableSystemFocusHighlight()`。该 helper 在 `Build.VERSION.SDK_INT >= Build.VERSION_CODES.O`（API 26+）门控下，对 `getWindow().getDecorView()` 调用 `setDefaultFocusHighlightEnabled(false)`，关闭 Android 系统默认焦点高亮框（app-wide 全屏默认关）。放在 `super.onCreate` 之后是因为 `FlutterSurfaceView` 宿主在 `super.onCreate` 内程序化创建、decorView 此时才 attach，代码级在 decorView 上关比主题属性传播更确定。**只关系统那层，app 自绘焦点环（`hibiki_focus_ring.dart`）和焦点导航逻辑（`global_navigation.dart`/`FocusDriver`）完全不动。** 未叠加主题级方案 A（`android:defaultFocusHighlightEnabled=false`）——代码级已确定生效，主题级需补 values-v26+values-v31 两份完整 styles，属冗余；commit 见下。
- **[x] ② 已加自动化测试** —— `hibiki/test/platform/android_focus_highlight_guard_test.dart`：源码扫描守卫，断言 MainActivity 含 `setDefaultFocusHighlightEnabled(false)` + `Build.VERSION_CODES.O` 版本门控 + `getWindow().getDecorView()`，且 `disableSystemFocusHighlight()` 真接线在 `super.onCreate(savedInstanceState)` 之后（不是只定义不调用）。撤改（删调用）实测转红：`+0 -1`，命中 "must be invoked from onCreate" 断言。`:app:compileDebugJavaWithJavac` 编译通过（BUILD SUCCESSFUL）。
- **备注**：残留风险——这是 Android 系统行为，host/普通模拟器未必复现 OneUI 特例，需三星 OneUI 6.5 真机复测「系统焦点框消失、只剩 app 自绘环」（留用户）。诚实声明：若 OneUI 在 `defaultFocusHighlightEnabled` 之外另有私有系统级焦点提示（厂商定制），则代码级关不掉，需真机确认后再针对 FlutterView 单独处理或叠加主题级方案 A。
