## BUG-359 · 收藏后字幕列表favorites档延迟
- **报告**：2026-06-20（用户：）
- **真实性**：✅ 真 bug（缓存键缺收藏状态）。根因 `hibiki/lib/src/media/video/video_subtitle_jump_panel.dart:374` 的 `_visibleCueIndexes()`：缓存命中键只含 `(cues 身份, 长度, _filter)`（旧 :375-378），**不含收藏状态/版本**。收藏档（favorites）的成员由实时 `widget.isCueFavorited` 谓词决定，而该谓词可在 panel widget 身份不变时变化（页面层 `isCueFavorited: _isCueFavorited` 是稳定的 State 方法 tear-off → 收藏 toggle 不改 widget 身份 → 不触发 `didUpdateWidget` 的兜底 `_clearCueCaches()`，:214）。于是收藏状态变后 `(cues,length,filter)` 三键都没变 → 任一 panel 内部 `setState`（播放推进的控制器 tick `_onControllerChanged`）触发 `build` 时命中陈旧缓存返回收藏前的成员集 → 列表延迟，要等下次 `_setFilter` / `didUpdateWidget` 清缓存才更新。计数 chip（TODO-631 的 `_favoriteCueCount`，未缓存、实时谓词）即时更新，故症状为「计数对了，列表行延迟」。
- **[x] ① 已修复** — `hibiki/lib/src/media/video/video_subtitle_jump_panel.dart:384-385`：收藏档**不缓存**（`final bool cacheable = _filter != VideoSubtitleListFilter.favorites;` 门控缓存命中读取），每次重算可见集（收藏档条目通常不多，成本可接受）；仍同步刷新 `_cachedVisibleIndexByRawIndex`（`_visibleIndexForRawIndex` 非 all 档读它，保证收藏档自动滚动定位正确）。`all` / `selected` 档缓存行为不变（`all` 纯结构；`selected` 经 onToggleCueSelection→页面 setState→didUpdateWidget 清缓存），保留缓存性能。提交 51b25cf29。
- **[x] ② 已加自动化测试** — `hibiki/test/media/video/video_subtitle_jump_panel_test.dart`（`TODO-632/BUG-359: favorites list reflects favorite-set changes that happen behind a stable predicate`）：收藏集藏在稳定谓词闭包背后变化 + 控制器 tick 驱动 panel 内部 setState → 断言列表即时含/移除该行（计数 chip 同步）。撤销 `cacheable` 门控即复红（命中陈旧缓存 → 行不出现/不消失）；已实测 red→green。
