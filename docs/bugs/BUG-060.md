## BUG-060 · 有声书音频高亮随阅读累积偏移（~1 万字处偏 2 字）
- **报告**：2026-06-05（用户：「现在在一万多字的时候是高亮和实际的文字偏移了两个字」）。采番注：与同轮 B/C 连号取 060（编号可能被并发 agent 抢，以提交为准）。
- **真实性**：✅ **真 bug（两套文本抽取定义/消费同一坐标系，逐字不一致 + 单调游标放大成累积偏移）**。沿真实代码路径定位：高亮坐标在**匹配时**由 Dart `package:html`（`epub_book.dart::chapterPlainText` → `AudioTextNormalizer` 白名单计数）算出归一化偏移 `normCharStart/End` 存进 `sasayaki://...&ns=&ne=`；**渲染时**由浏览器实时 DOM（`reader_pagination_scripts.dart::collectSasayakiCueRanges` 走 TreeWalker + `isMatchableChar` 计数）消费。两套抽取靠**手工镜像规则**对齐（白名单三份手抄：`audio_text_normalizer.dart::_isKeepable` / JS `ttuRegex` / `audiobook_bridge.dart::__hoshiIsSkippable`——逐码点核对当前**完全一致**，0 差异），但 `package:html` 与真 DOM 的白名单字符序列仍会差几个字（实体/ruby `<rtc>`/markup 边界等）。**放大成「越往后越偏」的根因**：旧 `collectSasayakiCueRanges` 用**单一单调 cursor 从 body 头累加**，任一处差 2 字就把其后**所有** cue 整体推移。git 历史 `b57e51fa4`/`cc14e198c`/`945244a02`/`32b2c1cda` 是一串「消除 offset drift」打地鼠补丁，印证这是架构脆弱性而非某次抄错。
- **[x] ① 已修复** — 根因修（消除「两套坐标系」这个特殊情况，让实时 DOM 成为唯一坐标权威）：① `audiobook_bridge.dart` 抽 `buildSasayakiPayload`，payload 除 `start`/`length`（降级为**提示**）外带上 cue 原文 `text`。② JS `collectSasayakiCueRanges` 重写为「文本就近 + 单调 + 窗口 + 回落」：`buildSasayakiNormIndex` 一次性按实时 DOM 文本建归一化全文 + char→node 反查表；每条 cue 用 `normalizeText(cue.text)` 在 `[max(cursor,hint-256), hint+256]` 窗口内取**离提示最近**的整句出现位置（`rangesForNormSpan` 映回 DOM range），游标单调不回退；窗口内未命中回落提示偏移（不劣于现状，绝不空高亮）。**漂移在结构上被打断**（每条 cue 独立按原文锚定，单处 2 字差不再传播）；**整句 needle + 有界窗口 + 单调 ⇒ 不会跳到远处重复句**（用户担心的「来回跳动」被堵死）。**无需数据迁移**：cue.text 本就在库里，旧书立即生效。reader/bridge `flutter analyze` 0 issue；新 JS 经 `node --check` + 烟雾运行验证语法与定位正确。
- **[x] ② 已加自动化测试** — ① `hibiki/test/media/audiobook/audiobook_bridge_payload_test.dart`（payload 带 text + section 过滤 + 跳过非 sasayaki，3 例）。② `hibiki/test/reader/sasayaki_cue_mapping_drift_test.dart`：纯 Dart 影子 `ReaderPaginationScripts.resolveCueNormStartsForTesting`（与 JS 同算法）7 例覆盖「无漂移 / +2 漂移自愈到真实位置 / 漂移不传播 / 远处重复不抢就近 / 单调不回退 / 未命中回落 / 越界裁剪」+ 源码守卫 1 例（钉住 JS 有 `buildSasayakiNormIndex`/`normalizeText(cue.text`/`full.indexOf(needle`/`WINDOW`/`rangesForNormSpan`，防回退到死偏移）。`test/reader/` + `test/media/audiobook/` 全量 493 例绿。
- **备注**：有声书高亮/坐标映射类。JS 运行时逻辑（DOM range 真实构建）单测不可达，靠 Dart 影子同算法 + 源码守卫 + node 烟雾兜底。**真机肉眼复测原始失败路径待用户**：打开有声书播放到 ~1 万字、~10 万字处核对高亮贴合实际文字、快速跳转不抖动。
- **审查修正（opus，W-1）**：初版 JS `buildSasayakiNormIndex` 的 `full`（UTF-16 码元串，`indexOf` 返回码元偏移）与 `map`（逐码点）索引空间不一致，含 CJK 扩展 B+ 代理对汉字（白名单含 `\u{20000}+`）的书会在该字之后高亮错位（即要消除的偏移在扩展区复发）。已修：`map` 改按码元粒度（代理对 push 两条同区间反查项），node 烟雾验证 `full.length===map.length` 且 ext-B 前缀下目标句仍精确命中；守卫加 `for (var u = 0; u < ch.length` 断言防回退。needle 用 `normalizeText`（仅白名单、无 kata→hira 折叠）：靠假名折叠才命中的 cue 运行时搜不到会回落提示偏移（退化为旧行为，可接受）。
