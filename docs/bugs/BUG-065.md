## BUG-065 · 桌面端 Anki「获取牌组/模型」秒报错 `Cannot connect to AnkiConnect: ClientException with SocketException: Write failed (errno=10053)`（常驻 http.Client 复用 keep-alive 死连接）
- **报告**：2026-06-05（用户截图：红字 `Cannot connect to AnkiConnect: ClientException with SocketException: Write failed (OS Error: 你的主机中的软件中止了一个已建立的连接, errno = 10053), address = localhost, port = 4392, uri=http://localhost:8765`，并指出「秒超时啊」）。采番注：动手前查 develop 当前最大号 BUG-064，BUG-063 已被并发分支（Drive token 刷新）占用，故本条取 065。
- **真实性**：✅ **真 bug（非「Anki 没开/防火墙/超时」，是连接池复用死连接）**。沿真实代码路径定位：① 报错文案来自 `packages/hibiki_anki/lib/src/ankiconnect/ankiconnect_repository.dart:92` 的兜底 `catch (e)`（`fetchConfiguration`），**不是** `AnkiConnectService.checkConnection()`——后者对 `http.ClientException` 有专属分支返回 `HTTP error:`，故这条「Cannot connect」只能是 fetch 在 `checkConnection()`（首发 `version` 探测）**通过之后**、`getDeckNames()` 失败时报的。即**第一发请求成功、第二发失败**——证明 Anki 在跑、AnkiConnect 在监听、端口/地址对、防火墙没拦首连。② `errno=10053`（WSAECONNABORTED：已建立连接被本地中止）发生在 **write 阶段**（请求字节刚要发就被拒），所以是「秒失败」而非走到 `_timeout=10s` 超时。③ 根因：`AnkiConnectService` 持有一个常驻 `http.Client`（`_client`），底层 dart:io `HttpClient` 默认 keep-alive 连接池。AnkiConnect 的简易 HTTP server 会关闭空闲连接；客户端复用了**已被服务端关闭的死连接**，第二发请求往死 socket write → 立刻失败。是经典 keep-alive 复用竞态。
- **[x] ① 已修复** — 提交 `22e3f8b88`。根因修（非补丁）：把 `_request` 里的 `_client.post(...)` 抽成 `_post()`，外包 `_postWithStaleConnectionRetry(body, {required idempotent})`——捕获 `http.ClientException`，若是连接丢弃类（`_isConnectionDrop`）**且 action 幂等**则用新连接（死连接已被池丢弃）重试一次，否则 rethrow。**安全性按 action 幂等性 gate，而非猜「请求是否已送达」**（后者不可靠：package:http 把 write 与读响应头包在同一 try，`Connection reset` 可能发生在请求已抵达 Anki 之后）：`_nonIdempotentActions={addNote, createModel}` **永不自动重试**（杜绝重复卡 / `model exists`）；其余（version/deckNames/modelNames/modelFieldNames/findNotes/storeMediaFile/createDeck）幂等才重试。判据优先用 `osError.errorCode`（OS ABI 常量，跨平台/跨 package 版本稳定：Win 10053/10054、POSIX 32/104/54），无 osError 时回退文案（write failed/connection reset/broken pipe/connection aborted）。这是「幂等请求重试 stale 连接」业界标准（cf. Go net/http、java.net.http）。`flutter analyze` 0 issue。
- **[x] ② 已加自动化测试** — `packages/hibiki_anki/test/ankiconnect_service_test.dart` 新增「stale keep-alive connection」组 7 例（最强可落地层——`runWithClient`+`MockClient` 在 zone 内注入会抛连接异常的 client，无真 socket）：errno 路径（`_FakeClientSocketException` 同时 implements ClientException+SocketException、消息故意无关键词，证明 errno 而非文案驱动决策）、文案回退、`Connection reset` 幂等读重试、恰好重试一次不死循环、非连接丢弃错误不重试、**`addNote` 遇连接丢弃也绝不重试（断言只发 1 次=不造重复卡）**、`storeMediaFile` 幂等重试。red→green 实测（未加幂等 gate 时 addNote 会重试致 attempts==1 断言红）。`hibiki_anki` 全量 32 绿；主 app `test/anki/` 无回归（拷入 gitignored `google_oauth_secret.dart` 后 `anki_view_model_lapis_test` 4 例亦绿）。opus 审 🟡→已采纳其全部 Warning（幂等 gate 根治「connection reset 可能已送达」、errno 判据、补 addNote 测试），未换方案 B（禁用 keep-alive）因会破坏 `http.Client` 跨平台抽象与 `runWithClient` 测试注入、改动面大且收益等价。
- **备注**：Anki 集成/HTTP IPC/连接池类。修复在纯 Dart（`hibiki_anki` 独立包），单测已覆盖真实链路。**真机肉眼复测原始失败路径待用户**（桌面 Windows + Anki 桌面端开 AnkiConnect）：Anki 设置页点「获取牌组/模型」应稳定成功、不再秒报 `Write failed`；连续多次 fetch / 制卡验证连接复用稳定。
