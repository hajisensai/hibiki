## BUG-498 · video: bottom/bottom-right tap cannot toggle controls (TODO-1073)
- **报告**：2026-07-01（用户：）
- **真实性**：✅ 真 bug。根因：`third_party/media_kit_video/lib/media_kit_video_controls/src/controls/material.dart`（修前）的单一手势层 `Positioned.fill(bottom: 16.0 + subtitleVerticalShiftOffset)` 把“单击唤控制栏”的 `onTap` 与防边缘误触的 16px 拖动 buffer 揁在同一层。`subtitleVerticalShiftOffset`（:596）= padding.bottom + bottomButtonBarMargin.vertical + buttonBarHeight（非空时）≈ 屏底 80–130+px 成为 `onTap` 死区：控制栏隐藏（`visible=false`）时该底带无任何 media_kit 手势层，上层 Hibiki 字幕层 / 侧栏都不接管 → 底部（尤其右下角）点空唤不出控制栏。Hibiki 外层 `_handleVideoPointerUp`（`hibiki/lib/src/pages/implementations/video_hibiki_page.dart:4517-4521`）移动端故意不自做 toggle，靠 translucent 下探到 media_kit，因此死区无人兜底。
- **[x] ① 已修复** — `third_party/media_kit_video/lib/media_kit_video_controls/src/controls/material.dart`：把单一手势层拆成两层：① 全画面 tap/long-press/double-tap 层（`Positioned.fill` 无 bottom 内缩，:953）承载 `onTap: onTap`（:965）；② 仅拖动识别器的 inset 层（`Positioned.fill(bottom: 16.0 + subtitleVerticalShiftOffset)`，:1003）保留边缘 buffer 语义。inset 拖动层在上、标 `behavior: HitTestBehavior.translucent`（:1019）让指针继续命中下方 tap 层 → 两层同进同一手势竞技场：边缘拖动仍归拖动识别器，平单击（含底部/右下）落到全画面 `onTap`。**选型理由**：首选改 fork（而非 Hibiki 侧外层 `_handleVideoPointerUp` 直驱 media_kit toggle），因为命中归属（hit-region）的真相源就在 fork；在 Hibiki 侧另接一套 toggle 入口会与 media_kit 自己的 `visible`/计时器形成双源（正是 TODO-364 已治的旧 BUG 根因），且拆层只动几何/hit-region、不碰 drag buffer 语义，改动小范围可控。提交：a1c64bb22
- **[x] ② 已加自动化测试** — `hibiki/test/third_party/media_kit_video_tap_hit_region_guard_test.dart`（源码守卫：锁 tap 层不再施 bottom `subtitleVerticalShiftOffset` 内缩、drag 层保留 inset + translucent、tap 层承载 onTap）。真实竞技场时序 headless 跑不了，故锁 vendored 源码结构不变量（与同目录现有 tap-arena / play-pause 守卫同套路）。
