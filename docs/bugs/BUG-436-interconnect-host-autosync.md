## BUG-436 · 互联host模式不应显示自动同步开关
- **报告**：2026-06-27（用户：）
- **真实性**：✅ 真 bug（UI 配置漏门控）。根因 `hibiki/lib/src/sync/sync_settings_schema.dart:137-147`：`sync.auto_sync` 开关在 Group 3「同步什么」section，**无任何 `visible:` 门控 → 全局显示**，不分后端/角色。但「自动同步」是出站同步开关：host（服务端）模式 backendType=hibikiServer 经 `resolveSyncBackend` 解出 `HibikiClientSyncBackend`，因角色互斥 client URL/token 必空 → `_ops==null` → `isAuthenticated==false`（`hibiki_client_sync_backend.dart:113,127-152`）→ 所有自动同步入口（app-open / background / book-close / 单书）早退什么也不做（`sync_auto_trigger.dart:170-172,244,303-305`）。即 host 是被动数据源，无出站同步，该开关在 host 模式恒无效，不该显示。对照同 schema 里 sync_now/compare 已按 `_isHostingInterconnect` 门控（BUG-084），唯独 auto_sync 漏了。
- **[x] ① 已修复** — 给 `sync.auto_sync` 项加 `visible: (ctx) => !_isHostingInterconnect(ctx)`（仅 host 隐藏，client 模式 hibikiServer 仍有出站同步故保留显示）。复用现有 `_isHostingInterconnect` 谓词（serverEnabled && backendType==hibikiServer），与 sync_now/compare 同一机制（`SettingsSwitchItem` 经 `super.visible`→`isVisible`→`SettingsSection.visibleCopy` 原生消费 `visible` 字段）。`hibiki/lib/src/sync/sync_settings_schema.dart` · 提交 001d578d2
- **[x] ② 已加自动化测试** — `hibiki/test/sync/sync_settings_visibility_test.dart`：新增「auto-sync is gated on the hosting role; other content switches are not」（断言 auto_sync 带 visible 谓词、其余 content 开关无谓词）+「auto-sync gate keys off the hosting-interconnect role」源码守卫（断言 auto_sync 块含 `!_isHostingInterconnect(`）。· 提交 001d578d2
