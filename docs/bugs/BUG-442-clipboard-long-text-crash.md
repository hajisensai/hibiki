- **报告**：2026-06-28（用户：）
- **真实性**：✅ 真 bug。桌面「外部复制 → 自动查词」管线对剪贴板文本全程零长度上限。崩点 `hibiki/lib/src/utils/components/clipboard_lookup_text_panel.dart:55,75`：`build` 取 `trimmed.characters.toList()` 后逐字符建一个 `MouseRegion(GestureDetector(Text))` 塞进 `Wrap`，widget 数 = 文本码点数。超长剪贴板（成千上万码点）→ 构建期巨量 widget → 主 isolate OOM / 引擎崩溃。上游 `hibiki/lib/src/sync/desktop_lookup_service.dart` 的 `_queueLookupRequest`(:75) / `_readClipboardText`(:216) 原样透传无上限；`app_model.dart` `normalizeSearchTerm`(:255) 也对超长串逐次 `replaceAll` 线性清洗。（FFI/mecab scanLength 默认 16 只扫前 16 码元，搜索清洗正则锚定首尾线性，均非根因。）
