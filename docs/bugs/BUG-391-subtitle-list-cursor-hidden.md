## BUG-391 · 视频字幕列表侧栏鼠标光标消失
- **报告**：2026-06-21（用户：鼠标从左边（视频区/控制区）移到字幕列表侧栏后光标消失，不可见）。**第三次修复**（前两次真机仍无效；第二次补丁 0dc687501 经核实是 no-op）。
- **真实性**：⚠️ 真 bug（Windows 真机可见）；**widget 层不可复现 + 集成测试也照不到**——headless 测试不接 Win embedder，框架的 `none→basic` 在测试环境正常回落 `basic`，且 `tool/run_windows_itest.ps1` 在主 checkout 不存在、焦点驱动 itest 无法移动鼠标 → 只能 Windows 真机肉眼/截图验证。
- **根因（第三轮定位，纠正前两轮）**：
  - 视频区 cursor:none 来源：`hibiki/lib/src/pages/implementations/video_hibiki_page.dart:3398`（`hideMouseOnControlsRemoval: true`）+ 顶层 `_buildCursorOverlay`（`cursor: SystemMouseCursors.none`，由 `_cursorHidden` 驱动，几何上**只盖视频区、不盖 push-aside 侧栏**）+ media_kit fork `material_desktop.dart` 控制条 `mount=false` 时其 `MouseRegion(cursor: none)`。
  - **前次补丁 0dc687501（TODO-677）是 no-op**：它只给侧栏包了 `_withSubtitleListCursorReveal → _handleSubtitleHover(true)`，但 `_handleSubtitleHover(true)` 的两臂对侧栏全空转：① `_setCursorHidden(false)` 只翻 `_cursorHidden`（同值写入被 `ValueNotifier` 去重不 notify；且它只控 `_buildCursorOverlay` 那层 overlay，几何上**不覆盖侧栏**）；② `_pokeControlsVisible()` 开头 `if (_subtitleListVisible.value) return;`（`controls_visibility.part.dart:34`）字幕列表开启时直接早退。两臂都不触碰侧栏 OS 光标 → 真机依旧隐藏。
  - **真根因**：从视频区（OS 光标已被 media_kit `SetCursor(nullptr)` 隐藏）移进侧栏时，Win embedder 那次 `none→basic` 的 `SetCursor` 在帧序窗口期没生效 / 被 `WM_SETCURSOR` 竞态回退（Flutter #84039 同类）；而框架 `MouseCursorManager.handleDeviceCursorUpdate`（`mouse_cursor.dart:75`）的 `lastSession == next` 去重会吞掉「再次声明 basic」。**没有任何代码在「鼠标进侧栏」时强制让 OS 真光标可见**，纯声明式 `MouseRegion(cursor:basic)` 救不了（被去重吞）。
- **[x] ① 根因修复** — 新增直发通道 helper `_forceRevealOsCursorForPanel(int device)`（`controls_visibility.part.dart`）：**双门控**①桌面 `_isDesktopVideoControls` ②`_cursorHidden.value == true`（仅光标确为隐藏态才发，避免压掉 cue 行 InkWell 手型 + 与框架 lastSession 去同步）；满足时直发 `SystemChannels.mouseCursor.invokeMethod('activateSystemCursor', {'device': device, 'kind': 'basic'})`，绕开框架 `lastSession==next` 去重、强制 OS 真 `SetCursor(IDC_ARROW)`。`_withSubtitleListCursorReveal` 的 `onEnter` **必发一次** `_forceRevealOsCursorForPanel(event.device)`（外加保留无害的 `_handleSubtitleHover(true)`）；`onHover` 仅经该 helper 的内部 `_cursorHidden` 门控补发（不无条件每次发）。`_handleSubtitleHover` 加注释讲清其两臂对侧栏是 no-op、真正起作用的是新直发通道。提交：（见下方本轮提交哈希）
- **[x] ② 自动化测试（源码守卫，最强可落地层）** — `hibiki/test/pages/video_subtitle_list_cursor_reveal_guard_test.dart`：经 `readVideoHibikiSource()` 合并语料断言 `_withSubtitleListCursorReveal` 的 `onEnter/onHover` 出现 `_forceRevealOsCursorForPanel(event.device)`；`_forceRevealOsCursorForPanel` 体内出现 `activateSystemCursor` + `kind 'basic'` + 门控 `_isDesktopVideoControls` + 门控 `_cursorHidden`（`if (_cursorHidden.value != true) return;`）。撤掉 onEnter 直发或去掉 `_cursorHidden` 门控即变红（已负向验证：删 `_cursorHidden` 门控 → 「门控②」断言转红）。
- **复测项（诚实标注）**：**widget 层 + 集成测试均照不到**（headless 不接 Win embedder/`SetCursor(nullptr)` 不复现；`tool/run_windows_itest.ps1` 主 checkout 不存在、焦点驱动 itest 动不了鼠标）。**仅 Windows 真机肉眼/截图可验**：视频播放等控制条 2s 淡出（OS 光标隐藏）后，把鼠标从画面区移到字幕跳转列表侧栏，确认光标重新可见（箭头）；并确认侧栏内 cue 行悬停时仍显手型（click）、不被强制成箭头。
