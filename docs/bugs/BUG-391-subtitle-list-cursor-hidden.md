## BUG-391 · 视频字幕列表侧栏鼠标光标消失
- **报告**：2026-06-21（用户：鼠标从左边（视频区/控制区）移到字幕列表侧栏后光标消失，不可见）
- **真实性**：⚠️ widget 层不可复现（同构布局探测下侧栏光标正确回落 `basic`），但根因成立、桌面真机需确认。根因 `file:line`：
  - 视频区 cursor:none 来源：`hibiki/lib/src/pages/implementations/video_hibiki_page.dart:3398`（`hideMouseOnControlsRemoval: true`）+ `:5649` `_buildCursorOverlay`（顶层 `cursor: SystemMouseCursors.none`，由 `_cursorHidden` 驱动）+ media_kit fork `third_party/media_kit_video/lib/media_kit_video_controls/src/controls/material_desktop.dart:746-749`（控制条 `mount=false` 时其包裹 controls 的 `MouseRegion(cursor: none)`）。
  - 缺陷点：字幕列表侧栏 `hibiki/lib/src/pages/implementations/video_hibiki/subtitle.part.dart:630` `_subtitleJumpSidePanel` 是 push-aside（`_videoWithSubtitlePanel` 的 Row 兄弟列），**没有任何 region 主动唤回光标 / 续命控制条**——与画面字幕盒 BUG-283 同根（cursor:none 胜出 + 缺 hover 唤回）。对照：字幕盒经 `video_subtitle_overlay.dart` `onHoverChanged: _handleSubtitleHover`（`video_hibiki_page.dart:5766`）已修。
  - 机制验真：Flutter `MouseCursorManager.handleDeviceCursorUpdate`（`firstNonDeferred(cursorCandidates) ?? SystemMouseCursors.basic`）无粘滞——命中链全 defer 则回落 basic。同构 widget 探测（视频区 cursor:none 全覆盖 + Row 兄弟侧栏，含真实 AnimatedSize/OverflowBox/SafeArea/Material/InkWell 几何）实测侧栏所有子区域光标 = `basic`、不粘 none。故 widget 层不可复现；真机「消失」只可能来自桌面 OS 光标在视频区→侧栏切换瞬间 media_kit `mount=false` 仍生效、侧栏无 region 续命 controls 导致的残留（需真机确认）。
- **[x] ① 根因修复** — 给字幕列表侧栏内容包一层 `_withSubtitleListCursorReveal`（`controls_visibility.part.dart`，仅桌面、`opaque:false`、`onEnter/onHover → _handleSubtitleHover(true)`），复用字幕盒同款救场（`_setCursorHidden(false)` 唤回光标 + `_pokeControlsVisible()` 续命控制条避免 media_kit `mount=false` 自身 cursor 置 none）；`_subtitleJumpSidePanel` 用它包住 `VideoSubtitleJumpPanel`。`opaque:false` 不阻断 cue 行点击/查词/滚动。提交：（见本分支 commit）
- **[x] ② 自动化测试** — `hibiki/test/pages/video_subtitle_list_cursor_reveal_guard_test.dart`：源码守卫（存在 `_withSubtitleListCursorReveal` 且 `opaque:false`/桌面门控/复用 `_handleSubtitleHover`、侧栏面板真包裹、`_handleSubtitleHover` 仍唤回+续命）+ 同构布局 widget 行为测试（视频区 none、移入侧栏触发 hover 救场且光标不粘 none）。撤包裹调用守卫即变红验证有效。
