## BUG-391 · 视频字幕列表侧栏鼠标光标消失
- **报告**：2026-06-21（用户：鼠标从左边（视频区/控制区）移到字幕列表侧栏后光标消失，不可见）。**第四次修复**（前三次真机仍无效；第二次补丁 0dc687501 经核实是 no-op；第三轮直发门控有悖论恒早退）。
- **真实性**：⚠️ 真 bug（Windows 真机可见）；**widget 层不可复现 + 集成测试也照不到**——headless 测试不接 Win embedder，框架的 `none→basic` 在测试环境正常回落 `basic`，且 `tool/run_windows_itest.ps1` 在主 checkout 不存在、焦点驱动 itest 无法移动鼠标 → 只能 **Windows 真机肉眼/截图/录屏验证**（本轮源码改动同样**未真机验**，真机截图/录屏是合入硬门槛）。
- **定性（关键，纠正前三轮「native SetCursor」口径）**：视频区光标隐藏是**框架层 MouseRegion**，**不是** native `SetCursor`：fork `material_desktop.dart:746-750` 的控制条 MouseRegion 在 `mount=false`（控制条 2s 淡出）时取 `cursor: none` 分支、否则 `basic`，走框架 `MouseTracker`，**几何只覆盖视频列 Expanded**（不覆盖 push-aside 侧栏）。叠加 `hideMouseOnControlsRemoval: true`（`controls_theme.part.dart:60`，把控制条淡出与该 cursor:none 绑定）+ 顶层 `_buildCursorOverlay`（`cursor: SystemMouseCursors.none`，由 `_cursorHidden` 驱动，几何同样**只盖视频区**）。
- **真根因（平台缺陷，本仓库只能缓解）**：从视频列（控制条淡出 → 框架 MouseRegion 经 MouseTracker 下发 `none→basic` 光标会话）移进侧栏时，Win embedder 那次 `SetCursor` 在帧序窗口期没生效 / 被 `WM_SETCURSOR` 竞态回退（Flutter **#84039**）；叠加框架 `MouseCursorManager`（`mouse_cursor.dart:75`）的 `lastSession == next` 去重吞掉「再次声明 basic」。纯声明式 `MouseRegion(cursor:basic)` 救不了（被去重吞）。
- **前三轮失效原因**：① 补丁 0dc687501（TODO-677）是 no-op——`_handleSubtitleHover(true)` 两臂对侧栏全空转（`_setCursorHidden(false)` 只翻 `_cursorHidden`、几何不覆盖侧栏且同值被去重；`_pokeControlsVisible()` 开头 `if (_subtitleListVisible.value) return;` 列表开时早退）。② 第三轮的直发 helper `_forceRevealOsCursorForPanel` 用了 `_cursorHidden.value == true` 当第二门控，但列表开态 `_hasVideoOverlay` 含 `_subtitleListVisible` → `_applyControlsVisibilityFromMediaKit`（`controls_visibility.part.dart:105`）把 `_cursorHidden` 恒置 false → 该门控**恒早退** = 第三轮空转。
- **本轮（第四轮）双管缓解**：
  - **[x] 管 1（源头层，机理最硬，缓解非根因）** — `controls_theme.part.dart:60`：`hideMouseOnControlsRemoval: true` → `!_subtitleListVisible.value`。机制：列表开时视频列 controls MouseRegion 走 `basic` 分支不再 `none`，列表开态鼠标跨进侧栏前视频列光标本就 `basic`、从未隐藏 → 跨列那次根本不存在 `none→basic` 转换 → **从源头消除竞态来源**（不是缩小窗口）。仅桌面 theme。
    - **关键防哑火（同时改第二处否则白改）** — `layout.part.dart:_buildVideoControlsInner` 旧用 `ValueListenableBuilder<VideoControlLayout>(valueListenable: _controlLayoutNotifier, ...)` 只监听布局 → `_subtitleListVisible` 翻转时桌面 theme 不重建（哑火）。改为 `ListenableBuilder(listenable: Listenable.merge([_controlLayoutNotifier, _subtitleListVisible]), builder: ...)`，builder 内重读 `_controlLayoutNotifier.value`。
  - **[x] 管 2（侧栏直发，改法 B，推测性缓解）** — `controls_visibility.part.dart`：`_forceRevealOsCursorForPanel` 门控**只保留** `_isDesktopVideoControls`、**去掉** `_cursorHidden` 那条（解除上一轮恒早退悖论）；`_withSubtitleListCursorReveal` 的 `onEnter` 跨列入侧栏**无条件直发一次** `activateSystemCursor{kind:'basic'}`，`onHover` 维持现状（列表开态因 `_cursorHidden` 恒 false 已等于 no-op）。**诚实标注**：onEnter 直发 = 重发框架刚发过的同条 `basic`，有效性 = 重发能否赢 embedder 竞态的**未证假设**，与第三轮同模型只差门控 → 是「推测性缓解」**不是根因修复**。cue 行手型由 cue 行 `InkWell` 声明式 `MouseRegion(click)` 保证、**不靠 onHover 续命**。
- **[x] 自动化测试（源码守卫，最强可落地层）** — `hibiki/test/pages/video_subtitle_list_cursor_reveal_guard_test.dart`：经 `readVideoHibikiSource()` 合并语料断言 ① `hideMouseOnControlsRemoval` **非裸 true**、是 `!_subtitleListVisible.value`（管 1·2a）；② 构造桌面 theme 的 builder 的 `listenable` 含 `_subtitleListVisible`（防哑火硬前置·管 1·2b）；③ `_withSubtitleListCursorReveal` 的 `onEnter` 路径**无条件直发** `_forceRevealOsCursorForPanel(event.device)`、`_forceRevealOsCursorForPanel` 体内有 `activateSystemCursor`+`kind 'basic'`+门控 `_isDesktopVideoControls` 且**不再有** `_cursorHidden` 门控（管 2 改法 B）。负向验证：2a 改回裸 true / 2b 去掉 `_subtitleListVisible` 应转红。
- **复测项（诚实标注）**：**widget 层 + 集成测试均照不到**（headless 不接 Win embedder、`none→basic` 测试环境正常回落，且 `tool/run_windows_itest.ps1` 主 checkout 不存在、焦点驱动 itest 动不了鼠标）。**仅 Windows 真机肉眼/截图/录屏可验（合入硬门槛）**：视频播放等控制条 2s 淡出后，把鼠标从画面区移到字幕跳转列表侧栏，确认光标重新可见（箭头）；并确认侧栏内 cue 行悬停时仍显手型（click）、不被强制成箭头。
- **已知局限（本轮不做）**：可选第三处「拆 `_hasVideoOverlay` 让 `_subtitleListVisible` 不再压 `_cursorHidden`」**高 BUG-258 回归风险**（`_hasVideoOverlay` 是 OS 光标隐藏的总门控，拆它可能让真 overlay 上光标错误隐藏），故**不纳入**。当前管 1 已从源头让视频列光标不隐藏、管 2 兜瞬态，二者不依赖拆 `_hasVideoOverlay`。
- **备注**：media_kit controls 跑不了 headless，源码守卫只锁结构不变量、对真机有效性零增益。本轮两管都是平台缺陷（#84039）+ 框架去重的**缓解层**，非根因修复；若 Flutter 修了 #84039 / 去重逻辑，可回退本直发通道并把 `hideMouseOnControlsRemoval` 收回裸 true。
