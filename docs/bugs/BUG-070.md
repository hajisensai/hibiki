## BUG-070 · 桌面端（Windows/media_kit）拖动有声书播放速度滑条闪退（进程级崩溃）
- **报告**：2026-06-06（用户：「调整有声书的播放速度会闪退，修复」；平台为桌面 media_kit 端）。采番注：动手前本 worktree 基线最大号 BUG-067，原取 068；收尾核对 develop 已被并发分支占用至 BUG-069（068=非日语 UI 字体发怪 / 069=从本句播放跨章文字跟随），故本条改取 070（仍可能被并发抢，以提交为准）。
- **真实性**：✅ **真 bug（桌面 media_kit 调速路径 native 崩溃）**。沿真实代码路径定位：① **同一个 `adaptiveSlider` widget** 同时承载音量滑条（`reader_quick_settings_sheet.dart:888 _buildVolumeSection`）与速度滑条（`:903 _buildSpeedSection`，`gamepadSeekableSlider` value 已 `clamp(0.25,3.0)`、min/max/divisions 自洽），**音量调节正常、速度调节崩溃** → 排除滑条 UI。② 速度滑条 `onChanged` → `AudiobookPlayerController.setSpeed`（`audiobook_controller.dart:748`）→ `_player.setSpeed` → just_audio_media_kit `MediaKitPlayer.setSpeed`（`mediakit_player.dart:292`）→ `media_kit` `Player.setRate`。`setRate` 在 `configuration.pitch == true` 时（`real.dart:817-830`）走「重写 mpv `af` 音频滤镜图」分支：`_setPropertyString('af', 'scaletempo:scale=…')`——**每次调速都重配一次音频滤镜图**。③ 而 `JustAudioMediaKit.pitch` 默认 `true`（`just_audio_media_kit.dart:41`），`main.dart:105-107` 启动时**只设了 `title`、没改 `pitch`** → 全程走脆弱的 af 重配路径。在 Windows 上播放过程中反复重配 libmpv 音频滤镜图会触发**进程级 native 崩溃**（segfault，Dart `try/catch` 与 `runZonedGuarded` 都拦不住，故表现为「闪退」而非错误弹窗——亦解释了 app 数据目录里无任何 Dart 侧崩溃日志）。④ 反证：音量走 mpv `volume` 属性、不重配滤镜图，故不崩。**根因 = 启用了 app 根本用不到的 pitch-shift 控制（无变调 UI、从不调 `setPitch`），把调速从 mpv 稳定的原生 `speed` 属性改道到易崩的 af 滤镜重配路径。** 移动端走 just_audio 原生（ExoPlayer / AVAudioEngine）的 setSpeed，路径稳定、不受影响。
- **[x] ① 已修复** — 提交 `ef79c9c42`。`main.dart` 启动时显式 `JustAudioMediaKit.pitch = false`（消除「启用从不使用的脆弱 pitch 路径」这个特例，非补丁/非吞异常——native segfault 也吞不住）：调速改走 `Player.setRate` 的 else 分支 `_setPropertyDouble('speed', rate)`（mpv 原生 `speed` 属性，久经考验、不重配滤镜图），mpv 默认 `audio-pitch-correction=yes` 仍自动保留音高 → 有声书加速不变「花栗鼠音」。一行修复覆盖全部桌面平台（Win/macOS/Linux 同走 media_kit）；`setPitch` 在 pitch=false 时会抛 ArgumentError，但本 app 从不调它，零功能损失。`flutter analyze` 0 issue、`dart format` 0 改动。
- **[x] ② 已加自动化测试** — `hibiki/test/media/audiobook/media_kit_pitch_guard_test.dart`（源码扫描守卫，最强可落地层；`test/media/audiobook/` 全量 289 例绿）：native 崩溃无法在 host 单测复现（flutter test 无 media_kit 后端），故钉住 `main.dart` ① 含 `JustAudioMediaKit.pitch = false`、② 不含 `pitch = true`、③ 该配置在 `ensureInitialized()` 之前（pitch 在 Player 构造时读取，须先设置）。与 BUG-031/034 源码守卫同范式。
- **备注**：有声书播放 / 桌面 media_kit / native 崩溃类。修复仅一行启动配置 + 守卫测试，`analyze` 0 issue。**真机肉眼复测原始失败路径待用户**（桌面 Windows）：打开带有声书的 EPUB → 播放 → 拖动「播放速度」滑条来回调速 → 不再闪退，且加速后人声音高正常（不变调）；并复测重置按钮（设回 1.0x）正常。
