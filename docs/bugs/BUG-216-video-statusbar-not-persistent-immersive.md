## BUG-216 · 视频沉浸状态栏不持续隐藏（后台返回残留）
- **报告**：2026-06-12（用户：）
- **真实性**：✅ 真 bug。根因双重：① 视频沉浸只在 `hibiki/lib/src/models/app_model.dart:2300` `openMedia` 打开媒体时一次性 `SystemChrome.setEnabledSystemUIMode(SystemUiMode.immersiveSticky)`，`hibiki/lib/src/pages/implementations/video_hibiki_page.dart` 全文 0 处自设系统栏模式（`_immersiveLocked` 是控制条 UI 锁，**不是**系统状态栏）；② 从不重申——`didChangeAppLifecycleState` 的 `resumed` 分支（约 line 664）只重启观看计时器 + 收回焦点，不重设系统栏。于是后台返回 / 通知栏下拉 / 多任务切回 / 转屏后，Android 把系统栏恢复显示，视频页没有任何代码把它重新隐藏，状态栏残留在视频上。`openMedia` 是书/视频共用入口，书随后被 reader（`reader_hibiki_page.dart:2727` 自设 `edgeToEdge`）覆盖，唯独视频页没有自己的系统栏所有者。
- **[x] ① 已修复** — 把视频系统栏沉浸所有权移到视频页且持续重申：新增 `_applyVideoImmersiveMode()`（`isMobilePlatform` 门控 + `setEnabledSystemUIMode(immersiveSticky)`，桌面 no-op，镜像同页 `_lockLandscapeForVideo` / `_restoreOrientationOnExit` 方向锁范式）；`initState` 显式调用（不再只靠 openMedia 一次性传递，对换集/全屏路由也稳健）；`didChangeAppLifecycleState` 的 `resumed` 分支重申（回前台后系统栏被恢复 → 主动重隐）。退出由 `AppModel.closeMedia` 的 `setHomeShellSystemUiMode()` 统一还原（既有，不动）。**严格限视频页**：不动 `openMedia`（书/视频共用入口，竖排小说由 reader 自设 edgeToEdge 覆盖、首页由 setHomeShellSystemUiMode 接管），app_model 里系统栏沉浸仍只 openMedia 一处。保留 immersiveSticky（与既有基线一致，上划临时露栏后自动重隐）而非 immersive 非 sticky：immersiveSticky + resumed 重申已覆盖「后台返回残留」主诉求，引入 immersive 非 sticky 需自管 didChangeMetrics 重申、复杂度上升无收益。提交见 git log（TODO-158）。
- **[x] ② 已加自动化测试** — `hibiki/test/pages/video_statusbar_immersive_guard_test.dart`（与控制条 UI 锁的 `video_immersive_lock_guard_test.dart` 区分）：断言①`_applyVideoImmersiveMode` 定义含 `isMobilePlatform` 门控 + `immersiveSticky`；②`initState` 调用它；③`resumed` 分支调用它；④`openMedia` 的 immersiveSticky 仍是 app_model 唯一一处（视频专属重申未下沉污染共用入口）。
- **备注**：media_kit 跑不了 headless、系统栏模式切换 / 回前台生命周期无法在 widget 测试真实驱动，用源码守卫钉住接线点；实际「一直隐藏」由用户真机（后台返回/通知栏交互后看状态栏）确认（needsDevice）。
