## BUG-056 · 视频制卡（及 Windows 阅读器/词典制卡）不带单词发音音频（Anki 媒体落库路径漏判 Windows 盘符本地路径，与 BUG-046 同源）
- **报告**：2026-06-05（用户：「视频制卡没带单词音频，修复一下」，附 Anki 卡片截图：心配 卡只有句子/视频音频按钮、无单词发音按钮）。采番注：动手前查 `docs/BUGS.md` 当前最大号为 BUG-055（被并发 agent 占用），故本条取 056。
- **真实性**：✅ **真 bug（Anki 媒体落库只认 `file://`/`/`/`http`，静默丢弃 Windows 盘符路径 `C:\…`，与播放侧 BUG-046 同一类缺陷在制卡侧的复发）**。沿真实代码路径定位：视频页制卡 `video_hibiki_page.dart:670 onMineEntry` 复用共享 WebView 弹窗（popup.js 把单词发音写进 `fields['audio']`）→ `repo.mineEntry(rawPayloadJson)`。单词发音经 `WordAudioResolver.resolveConfigured` 解析；**桌面本地音源**（NHK/Forvo 等本地库）`extractLocalAudio` 返回的是 Windows 盘符临时路径 `C:\Users\…\Temp\hibiki_audio_*.mp3`（与 BUG-046 一致）。`{audio}` → `payload.audio` → 落库 helper：`AnkiConnectRepository._storeRemoteAudio`（`ankiconnect_repository.dart:307`）与 `AnkiRepository._addRemoteAudio`（`anki_repository.dart:316`）都只分支 `url.startsWith('file://')` / `'/'` / `'http'` —— Windows 盘符路径 `C:\…`（或 `C:/…`）**三条都不命中** → `audioFile` 留 null → 返回 null → `processedAudio` 空 → `ExpressionAudio` 字段空 → Lapis 卡 `addAudioButtons` 只剩句子/视频音频按钮，单词发音消失。**非视频专有**：Windows 桌面阅读器/词典页制卡同样经 AnkiConnect 同一 helper，本地音源都会丢单词发音；用户恰由视频制卡撞见。BUG-046 当初只修了播放分发（`tts_channel.classifyAudioRef`「非 http 即文件」），制卡落库这条孪生路径漏修。
- **[x] ① 已修复** — 根因修（Linus 式：消除特例而非再加一条 `startsWith` 分支）。新增纯函数分类器 `AnkiAudioRef`（`packages/hibiki_anki/lib/src/anki_models.dart`，镜像 BUG-046 的 `classifyAudioRef`）：`http(s)://…`→`remoteUrl`（下载）；**其余一律 `localFile`**（`file://` URI 或裸绝对路径，Unix `/…` 或 Windows `C:\…`），`localPath()` 用 `Uri.toFilePath()` 解 `file://`、裸路径原样返回。`_storeRemoteAudio` / `_addRemoteAudio` 改用 `switch(AnkiAudioRef.classify(url))`，`localFile` 分支 `File(AnkiAudioRef.localPath(url))` —— Windows 盘符路径自此命中本地分支、落库成 `[sound:…]`。`dart format` 通过；hibiki app 上下文 `flutter test test/anki/` 全绿（含包编译）。
- **[x] ② 已加自动化测试** — `hibiki/test/anki/anki_audio_ref_test.dart`（纯函数守卫，最强可落地层——`_storeRemoteAudio` 私有且需 `AnkiConnectService`，故守在被提取出的分类器上）9 例：空/`http(s)`→url、Unix 绝对/`file://`/**Windows 盘符（反斜杠与正斜杠两种）→ localFile**、`localPath` 对盘符/Unix 原样返回、`file://` 去 scheme。red→green：分类器未定义时全红（undefined name），实现后 9 例全绿；`test/anki/` 全量 32 例（新 9 + 旧 23）全绿无回归。
- **备注**：Anki 制卡/音频落库/Windows 路径类。修复在 `hibiki_anki` 包（AnkiConnect 桌面 + AnkiDroid 两 helper 同步改，Android 本不产生盘符路径但同因消除）。**真机肉眼复测原始失败路径待用户**（Windows 桌面，AnkiConnect，单词发音用本地音源）：视频/阅读器/词典页对某词制卡 → Anki 卡 `ExpressionAudio` 应有单词发音按钮并能播；修复前 Windows 本地音源会丢该按钮。**附带修复（用户报「这个参考和文字重叠了」）**：同卡「Primary Definition 1/2」灰标签紧贴例句下沿（视觉拥挤）。根因=upstream Lapis 1.7.0 模板 `.def-info` 无上边距；仅在**桌面布局**（`--sentence-position: above`，例句在释义框上方）+**多辞典命中**（MainDefinition=第一辞典 与 Glossary=全辞典 不被 `cleanUpDefinitions` 去重 → 2 释义块 → 标签出现）时触发。手机端（`--mobile-sentence-position: below`，例句移到框下）与单辞典（去重后无标签）都不出现——故「别人 Lapis 没问题」。修法（用户选「加间距」）=**不碰 vendored 原文**：新增独立常量 `LapisNoteType.hibikiCssOverride`（`.def-info{margin-top:0.6em}`），`template.css` 拼成 `css + override`，追加在末尾按源码顺序覆盖 upstream（再 vendor 不被冲掉）。headless（Edge）复现+验证修复前后对比间距已分开。守卫 `lapis_note_type_test.dart`（vendored css 不含 delta、override 含 `.def-info`/`margin-top`、override 在 upstream `.def-info` 规则之后）+ `ankiconnect_create_test.dart` 改断言 `template.css`；`test/anki/` 全 33 例绿。**注**：已建过 Lapis 卡型的用户需经 Anki 设置重新「一键创建/更新 Lapis」或手动同步卡型 CSS 才生效（createNoteType 只在建型时 push）。
