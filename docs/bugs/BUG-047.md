## BUG-047 · 安卓端谷歌云盘「检查账户」显示「未登录」/「已登录」却没有账户名（移动端从不 signInSilently 恢复会话）
- **报告**：2026-06-05（用户：「正在检查账户的时候，为什么会说未登录，而且已登录底下没有我登录的账户名」）。经 AskUserQuestion 确认 = **安卓手机 + 谷歌云盘**。采番注：动手前查 `docs/BUGS.md` 当前最大号为 BUG-046，故本条取 047。
- **真实性**：✅ **真 bug（移动端会话恢复路径是空操作，安卓/iOS 专有）**。沿真实代码路径定位：设置「账户」行 = `_SyncAccountWidget`（`sync_settings_schema.dart:309`），`initState` 跑 `_checkAuth()`（:339）→ `restoreAuth(repo)` → 读 `isAuthenticated`（决定「已登录/未登录」）→ 读 `currentEmail`（决定上行显示**邮箱账户名**还是回退成「已登录」四字，:383）。后端 `GoogleDriveSyncBackend.restoreAuth`（`google_drive_sync_backend.dart:55`）在 `useMobileAuth` 时直接 `return false`——**空操作，从不调用 `signInSilently()`**，而 google_sign_in 6.3.0 冷启动后不会自动恢复用户：`currentUser` 为 null、`isSignedIn()` 不可靠。于是 `GoogleDriveAuth.isAuthenticated`（旧 `_signIn.isSignedIn()`）冷启可返回 false → 「未登录」；即便 `isSignedIn()` 偶返 true，`currentEmail`（旧 `currentUser?.email ?? signInSilently()?.email`）的 `currentUser` 为 null、兜底 silent 没跑成 → 账户名为空 → 「已登录」无邮箱。同一道闸门还卡住自动同步：`sync_auto_trigger.dart:120-121` 也是 `restoreAuth` 后 `if (!await isAuthenticated) return;`，故安卓冷启后自动同步大概率根本不触发。与桌面端有真正的 `restoreDesktopAuth`（刷新 token + 取邮箱）形成不对称——移动端缺这条恢复路径就是根因。根因点 = `google_drive_sync_backend.dart:56`（移动端 `return false` 空操作）+ `google_drive_auth.dart` 的 `isAuthenticated`/`currentEmail` 依赖未恢复的 `isSignedIn()`/`currentUser`。
- **[x] ① 已修复** — `69ae16b48`：根因修——消除「移动端没有会话恢复」这个平台不对称（非补丁、非加延迟/重试）。① `google_drive_auth.dart` 缓存 `GoogleSignInAccount? _mobileUser`，新增 `Future<bool> restoreMobileAuth()`（`signInSilently()` 显式复活会话并缓存账户，镜像 `restoreDesktopAuth`）；`isAuthenticated`（移动端）改为 `_mobileUser != null`（不再赌不可靠的 `isSignedIn()`）、`currentEmail` 改为 `_mobileUser?.email`；`authenticate`/`signOut`/`refreshAuth` 同步维护 `_mobileUser`。② `google_drive_sync_backend.dart` 移动端 `restoreAuth` 调 `_auth.restoreMobileAuth()` 取代 `return false`。③ `sync_compare_dialog.dart` 的 `showSyncCompareDialog` 在读 `isAuthenticated` 前补 `await backend.restoreAuth(repo)`，统一「先恢复再判断」契约（顺带修桌面端重启后直接开对比对话框会误报「先设置同步」的潜伏 bug）。`flutter analyze` 改动 4 文件 0 issue、`dart format` 通过。
- **[x] ② 已加自动化测试** — `69ae16b48` · `hibiki/test/sync/google_drive_mobile_auth_test.dart`：与 BUG-034 桌面修复同范式（host 上 `Platform.isAndroid==false`、google_sign_in 插件与 `GoogleSignInAccount` 均不可注入/构造，移动分支无法真实跑）。① 平台闸门运行时断言：`restoreMobileAuth()` 在 host（非移动）即 `false` 且不碰插件。② 源码扫描守卫：`google_drive_auth.dart` 含 `restoreMobileAuth()` + `signInSilently()` + `_mobileUser?.email` 且**不含** `_signIn.isSignedIn()`；`google_drive_sync_backend.dart` 含 `return _auth.restoreMobileAuth();` 且**不含**旧 `if (GoogleDriveAuth.useMobileAuth) return false;`；`sync_compare_dialog.dart` 的 `restoreAuth` 出现在 `isAuthenticated` 闸门之前。`flutter test test/sync/` **276 绿**（含本 5 例 + 桌面 BUG-034 用例无回归）。
- **备注**：同步/认证/google_sign_in/安卓类。代码 + 单测绿、`flutter analyze` 0 issue。**真机肉眼复测原始失败路径待用户**（安卓 + 谷歌云盘）：曾登录谷歌云盘 → 杀进程冷启 app → 进设置「同步与备份」，账户行应显示真实邮箱 + 「已登录」（不再「未登录」或空账户名）；并确认自动同步在冷启后能正常触发。
