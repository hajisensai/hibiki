## BUG-412 · 视频Shift鼠标悬停不查词(自绘overlay未接reader的shift-hover)
- **报告**：2026-06-23（用户：） · TODO-756a
- **真实性**：✅ 真 bug。阅读器有 Shift+鼠标悬停查词，视频从未接。根因=视频字幕是 Flutter 自绘 overlay（`hibiki/lib/src/media/video/video_subtitle_overlay.dart`，非 WebView），阅读器那套 JS `mousemove + e.shiftKey → onShiftHover → 查词`（`hibiki/lib/src/pages/implementations/reader_hibiki/webview.part.dart:866-873` 发，`:1129-1137` 收 → `_selectTextAt`）在视频根本不存在。视频字幕唯一查词触发是**点击**（`video_subtitle_overlay.dart:300` onTapUp → `_charHitTest` → `onCharTap` → `_handleSubtitleLookupTap`（`video_hibiki_page.dart:2200`）→ `_lookupAt`）。视频虽有 `MouseRegion`（`video_subtitle_overlay.dart` 的 `onEnter/onExit`）但只唤光标/续命控制条（BUG-283/284），不检测 Shift 也不查词。
- **[x] ① 已修复** — commit `e16e662e7`。给 `VideoSubtitleOverlay` 加 `onCharHover` 回调 + `MouseRegion.onHover`：按住 Shift（`HardwareKeyboard.instance.isShiftPressed`）且非模糊态时复用现有 `_charHitTest`（全局坐标命中字符）→ 经 `onCharHover` 通知页面。节流参照阅读器 8px 阈值（`_kShiftHoverThresholdPx=8`，平方比较）+ 同一 grapheme 不重复触发（记 `_lastShiftHoverPos`/`_lastShiftHoverGrapheme`，松开 Shift / `onExit` 复位）。`needHover` 加入 `onCharHover != null` 保证 MouseRegion 挂载。页面侧 `layout.part.dart` 把 `onCharHover` 接到与点击查词**同一条链路** `_handleSubtitleLookupTap`（内部 `_immersiveAllowsLookup` 门控一致）→ `_lookupAt`，零重写。桌面有效、移动端无 hover 自然不触发；现有 `onEnter/onExit` 唤光标/续命控制条/沉浸显形（BUG-198/283/284）全保留，点击查词不变。
  - 文件：`hibiki/lib/src/media/video/video_subtitle_overlay.dart`（onCharHover 字段 + _handleShiftHover + onHover 接线 + needHover）、`hibiki/lib/src/pages/implementations/video_hibiki/layout.part.dart`（onCharHover: _handleSubtitleLookupTap）。
- **[x] ② 已加自动化测试** — `hibiki/test/media/video/video_subtitle_shift_hover_lookup_test.dart`（6 例，widget 行为 + 焦点/键盘事件驱动）：按住 Shift hover 命中字符触发 onCharHover（同点击三元组）/ 非 Shift hover 不查词 / 节流（同字符小幅移动不重复、命中新字符立即换词）/ 模糊态不查词 / needHover 恰多挂一个 MouseRegion / Shift-hover 与 tap 命中同一字符给出完全相同 (句, grapheme, 矩形)（锁「同链路零重写」契约）。`flutter analyze` 0 issues；`test/media/video/` 913 绿 + `test/pages/` 1261 绿（含原 26 例 overlay 测试无回归）。
