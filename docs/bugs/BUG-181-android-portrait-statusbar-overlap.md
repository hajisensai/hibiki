## BUG-181 · 手机竖屏常驻状态栏挤压首页右上角图标（TODO-097）
- **报告**：2026-06-11（用户：手机竖屏时系统状态栏一直显示，会挡住 Hibiki 的图标，比较难点。隐藏掉状态栏或者取消状态栏的位置？）
- **真实性**：⚠️ 半真——首页 body 早已包 `SafeArea`（`home_page.dart:489/453/428`），右上角动作图标**不会被裁切**（widget 实测落在状态栏 inset 之下，icon top == 48 == 状态栏高）。但首页/菜单壳在 Android 上跑 `edgeToEdge`（`main.dart` 启动 + `app_model.dart` closeMedia），系统状态栏（OS 时钟/电量条）**常驻可见且紧贴**在 `HibikiPageHeader` 右上角图标正上方，挤压触控区、视觉打架，用户体感「挡住、难点」。打开书/视频走 `immersiveSticky`（两栏全隐），所以仅首页壳有此问题。
  - 根因（系统 UI 模式，非布局缺口）：首页默认模式 = `SystemUiMode.edgeToEdge`（状态栏画出来）。设置点：`hibiki/lib/main.dart` 启动分支 + `hibiki/lib/src/models/app_model.dart` `closeMedia`（退出媒体回首页）。
- **[x] ① 已修复** — 新增纯 helper `setHomeShellSystemUiMode()`（`hibiki/lib/src/utils/misc/platform_utils.dart`）：Android 走 `SystemUiMode.manual` 且 `overlays:[SystemUiOverlay.bottom]`——**隐藏状态栏、保留导航/手势栏**；iOS/桌面回退 `edgeToEdge`（iOS 状态栏靠 SafeArea 处理，桌面无系统栏）。启动（`main.dart`，原裸 `edgeToEdge` → `unawaited(setHomeShellSystemUiMode())`）与 `closeMedia`（`app_model.dart`，原裸 `edgeToEdge` → `await setHomeShellSystemUiMode()`）改走 helper。**不动阅读器 `reader_hibiki_page.dart:2440` 的 edgeToEdge**（阅读器自管 chrome + `_stableTopInset`，越界即改阅读器行为）。提交：见本轮 commit。
- **[x] ② 已加自动化测试** — `hibiki/test/utils/home_shell_system_ui_mode_test.dart`：①行为测试（platform channel 拦截）断言 helper 恰发一次 `SystemChrome.setEnabledSystemUIMode`；②源码守卫锁定 Android 分支用 `manual`+仅 `bottom` overlay（不含 `SystemUiOverlay.top`）、非 Android 回退 `edgeToEdge`、启动与 `closeMedia` 两处都走 helper 而非裸 `edgeToEdge`（任一退回 → 红）。host 无 Android（`Platform.isAndroid==false`）跑不到真机隐藏路径，故 Android 具体模式靠源码守卫钉死。
- **备注**：真机未验——需 Android 手机竖屏复测「首页状态栏消失、右上角图标可点、从书/视频退出回首页状态栏仍隐藏」。权衡：首页壳隐藏状态栏后浏览书库时看不到系统时钟/电量（用户明确要求隐藏，取舍可接受）；如需保留时钟可回退为「只加顶部 padding 不隐藏」。范围：仅首页/菜单壳，阅读器/视频不变。
