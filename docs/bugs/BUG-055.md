## BUG-055 · 调整界面大小后，视频内查词弹窗的字发糊（查词浮层在缩放小画布栅格化再被拉大）
- **报告**：2026-06-05（用户：「调整界面大小以后查词弹窗的字糊」）。经 AskUserQuestion 确认 = **视频里面的查词**（用户「视频哪有正文」→ 非阅读器路径）。采番注：原在 video worktree 取 051，但合并进 develop 时 051 已被 develop 占用（嵌套弹窗横滑），且 develop 已用到 054，故重编号为 **055**（与 develop 的 054 不冲突）。
- **真实性**：✅ **真 bug（视频查词浮层逃出阅读器那套「按页中和」、在缩放画布里栅格化 WebView 再被拉大）**。沿真实代码路径定位：视频查词浮层渲染在**根 Overlay**（`video_hibiki_page.dart:537` `Overlay.of(context, rootOverlay: true)`，目的是浮在 media_kit 全屏路由之上）。根 Overlay 在全局 `HibikiAppUiScale`（`app_ui_scale.dart`，浏览器式整体缩放，用 `FittedBox(BoxFit.fill)` 把整树渲染进 `canvas=view/s` 画布再拉满屏）的 `FittedBox` **之内**——故浮层坐标空间是**缩放后的小画布 `view/s`**，浮层的词典 WebView（`DictionaryPopupWebView`，平台视图＝纹理，按 `逻辑尺寸×dpi` 栅格化）在小画布尺寸栅格化后被外层 `FittedBox` 拉大 `s` 倍 → 丢分辨率＝字糊。这与 BUG-039（阅读器正文/弹窗发糊）**同源**；BUG-039 的修复 `HibikiAppUiScaleNeutralizer` 只接到了阅读器两处构造点（`reader_hibiki_source.dart` / `reader_hibiki_history_page.dart`），视频是后加的新媒体类型、且查词浮层挂根 Overlay（逃出按页中和），所以没被覆盖。原 `_lookupAt` 用 `scaledRectToCanvas(charRect ÷s)` 把字符屏幕 rect 降到 canvas 空间、配 `screen=canvas` 定位，定位自洽但 WebView 仍在小画布栅格化＝糊。根因点 = `video_hibiki_page.dart` 的 `_buildPopupOverlay`（浮层子树未中和缩放）。
- **[x] ① 已修复** — 根因修，与 BUG-039 同范式（跨平台，非补丁、非 Windows 专修）：① `video_hibiki_page.dart` 的 `_buildPopupOverlay` 把整棵浮层子树用 `HibikiAppUiScaleNeutralizer` 包裹，中和回**真实视口尺寸、净缩放=1**，词典 WebView 按原生像素密度渲染＝清晰（与中和后阅读器弹窗清晰同理）。② 浮层坐标系随之统一翻到真实屏幕空间（净变换=1，与 `localToGlobal` 的字符 rect 同系），`_lookupAt` 删掉 `scaledRectToCanvas` 直接用屏幕 `charRect` 定位（二者是逆操作，必须二选一）。③ `app_ui_scale.dart` 移除中和后已无人用的 `scaledRectToCanvas`（消除「手动 ÷s 换算」这个特例，不留死代码）。`flutter analyze` 改动 4 文件 0 issue、`dart format` 0 改动。
- **[x] ② 已加自动化测试** — `hibiki/test/pages/video_lookup_popup_overlay_test.dart`（重写）：① **行为守卫**：复刻生产层级（`HibikiAppUiScale` 外、根 Overlay 内），中和后的浮层用字符屏幕 rect 定位，断言浮层在**屏幕上**紧贴被点字符下方（`popup.top ≈ char.bottom+4`，缩放 1.0/1.5/0.8 都对齐）。② **对照（红）**：去掉中和器、同样直传屏幕 rect，浮层纵向偏离 >50px（证明中和器不可省）。③ **源码守卫**：`video_hibiki_page.dart` 含 `HibikiAppUiScaleNeutralizer(` 且**不含** `scaledRectToCanvas`；`app_ui_scale.dart` 不含 `scaledRectToCanvas`。「字糊」本身是 WebView 纹理分辨率（设备级肉眼项），中和器把子树净缩放归 1（原生密度）已由 `test/utils/app_ui_scale_neutralizer_test.dart` 单测，故本文件守接线 + 坐标自洽。两个测试文件单独跑各 3 绿、`flutter analyze` 0 issue。
- **备注**：视频/查词浮层/界面缩放/WebView 纹理类。**真机肉眼复测原始失败路径待用户**（Windows）：放大「界面大小」→ 打开视频 → 点字幕字符查词，弹窗正文应清晰、且定位贴着被点字符不偏。注：跑全量 `flutter test` 时本 worktree 有**并发 agent 未提交的 `packages/hibiki_core/.../tables.dart`+`database.dart` 改动但未重跑 build_runner**，多测试入口同跑会触发 `database.g.dart` 的 `VideoBooksCompanion` 增量编译崩溃（与本修复无关，单独跑本文件不受影响）。
