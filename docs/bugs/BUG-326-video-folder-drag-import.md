## BUG-326 · 视频拖放扩展名不全 + 书架拖入视频不自动切视频导入
- **报告**：2026-06-19（用户：B02 验收第 1 条 "好像不太行，而且在书架导入的时候，会说切到视频导入，应该自动切才对"）
- **真实性**：✅ 真 bug（两条独立缺口）
  - 子问题①：拖放分类视频白名单 `kDragVideoExtensions` 与文件夹扫描白名单 `kVideoExtensions` 不同步，缺 `mts/vob/ogv/rmvb/rm` → 拖入这些容器格式的视频被分到 `unknown`，识别不出。根因 `hibiki/lib/src/media/drag_drop/drop_classification.dart:31-44`（拖放白名单）vs `hibiki/lib/src/media/video/video_filename_parser.dart:50-68`（扫描白名单，TODO-050/BUG-168 已补齐 18 种）。注：「选文件夹」按钮路径 `listVideoFilesInDirectory`（同文件 222-234）已是 `recursive: true` 递归扫描，不是本 bug。
  - 子问题②：书架（books）tab 拖入视频，`decideDropIntent` 在 books 表面对视频返回 `unsupportedSurface`，书架 drop handler 只弹 SnackBar `drag_drop_unsupported_on_books` 提示「请切到视频页面」，从不自动切。根因 `hibiki/lib/src/media/drag_drop/drop_decision.dart:32-40`（books 分支无视频意图）+ `hibiki/lib/src/pages/implementations/reader_hibiki_history_page.dart:2267-2276`（提示 + 视频意图空 break）。
- **[x] ① 已修复** — ①同步 `kDragVideoExtensions`=`kVideoExtensions`（补 mts/vob/ogv/rmvb/rm，`drop_classification.dart:30-49`）；②`decideDropIntent` books 表面对视频/播放列表返回 importNewVideo/importNewPlaylist（`drop_decision.dart:32-50`，cardHit 时音频/字幕仍优先挂卡）；③书架 drop handler 新增 `_openVideoImportPrefilled`/`_openPlaylistImportPrefilled` 自动打开 VideoImportDialog 带文件（`reader_hibiki_history_page.dart` importNewVideo/importNewPlaylist 分支）。提交 3c223c85f。
- **[x] ② 已加自动化测试** — `test/media/drag_drop/drop_classification_test.dart`（mts/vob/rmvb 分类 + kDragVideoExtensions↔kVideoExtensions 同步守卫）；`test/media/drag_drop/drop_decision_test.dart`（books 表面视频→importNewVideo / m3u8→importNewPlaylist / .mp4 到书卡仍挂卡 / .mp4 空白区切视频）；`test/pages/shelf_video_drop_autoswitch_guard_test.dart`（书架 handler 接线源码守卫）。38+ 项绿。
- **备注**：采番 326（develop 最高 322，323-325 被并发 worktree 占用，遍历所有 worktree 分支取并集后取 326）。`BookImportDialog` 内部文件选择器用白名单过滤，选不到视频，无对话框内检测路径——「书架导入说切到视频」的真实路径只有拖放。
