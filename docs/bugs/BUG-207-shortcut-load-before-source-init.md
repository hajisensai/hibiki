## BUG-207 · 自定义快捷键重启后丢失/不生效(loadShortcutRegistry早于source.initialise)
- **报告**：2026-06-11（用户：TODO-141）
- **真实性**：✅ 真 bug —— 根因 `hibiki/lib/src/models/app_model.dart`（修复前）：`initialise()` 在 `ReaderHibikiSource.readerSettings` 赋值后、那批 `source.initialise()` 的 `Future.wait`（注册 reader 设置/增强/快捷动作/媒体源）**之前**就调用 `loadShortcutRegistry(...)`。
  - 数据流：`loadShortcutRegistry`（`hibiki/lib/src/shortcuts/shortcut_preferences.dart:12`）走 `source.getPreference<String?>(key: 'shortcut_bindings_json')`。但 `MediaSource.initialise()`（`hibiki/lib/src/media/media_source.dart:100`）才会调 `_loadPreferencesFromDb()` 把 DB 偏好装进内存 `_preferences`。早于 initialise 调用时 `_preferences` 为空，`getPreference<String?>` 命中 `null is String?` → 返回 null（`media_source.dart:134-140`，**对 nullable 类型不触发 write-through**）。
  - 后果：`loadShortcutRegistry` 因 json==null 走 `resetToDefaults`（`shortcut_preferences.dart:22` / `shortcut_registry.dart:73`），registry 内存态装成纯默认键 → 用户已保存的自定义键在本次会话整段**不生效**（即"重启后丢失/不生效"的直接表现）。进一步：用户随后在设置里改任意键触发 `saveShortcutRegistry`（`shortcut_preferences.dart:26`）→ `toJsonString()` 把"默认键基底+本次改动"整体写回 DB，**永久覆盖**掉之前保存的自定义 JSON。
  - 与 profile 切换路径（先 `refreshPreferencesFromDb` 刷新 source 缓存、再 `loadShortcutRegistry`，`app_model.dart:1545`）的正确顺序不对称——切换路径是对的，冷启动路径是错的。
- **[x] ① 已修复** —— 把 `loadShortcutRegistry(...)` 调用从 source 初始化前移到那批 `source.initialise()` 的 `Future.wait` **之后**（`hibiki/lib/src/models/app_model.dart:1351`，注释标 `BUG-207`），保证读 `shortcut_bindings_json` 时 source 偏好缓存已就绪、能取到已保存 JSON，不再误触发 resetToDefaults。提交：见本分支 todo-134-141-shortcuts（哈希见提交记录）。
- **[x] ② 已加自动化测试** —— 行为测试 `hibiki/test/shortcuts/shortcut_load_order_test.dart`：用真实 `loadShortcutRegistry` + 假 source（受控 `_preferences` 与 `initialise()`），断言「source 缓存装载前调用 loadShortcutRegistry 拿不到已存 JSON 而退回默认；装载后调用则正确恢复用户自定义键」；源码守卫 `hibiki/test/models/app_model_shortcut_load_order_guard_test.dart` 断言 `app_model.dart` 中 `loadShortcutRegistry(` 出现位置晚于那批 `source.initialise()` 的 `Future.wait`（撤修复=把调用移回前面则守卫红）。
