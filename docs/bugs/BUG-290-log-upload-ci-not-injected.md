## BUG-290 · 错误日志上传按钮在 CI 构建版（含 Windows）全平台不显示
- **报告**：2026-06-15（用户：qqbotxiaoxiao）
- **真实性**：✅ 真 bug。门控本身无平台判断：`hibiki/lib/src/utils/misc/log_upload_config.dart:19` `bool get showUploadLogAction => isLogUploadConfigured(kLogUploadEndpoint);`，仅按「端点是否非空 http(s)」决定显隐（按钮接线见 `debug_log_page.dart:66` / `error_log_page.dart:47`）。真值密钥 `hibiki/lib/src/utils/misc/log_upload_secret.dart` 是 tracked + `--skip-worktree`，入库为空占位（`kLogUploadEndpoint = ''`）。**根因**：所有出包 workflow 都注入了 `google_oauth_secret`（`build-multiplatform.yml` android/linux/macos/windows 四 job + `release.yml` + `release-desktop.yml`），却**从未注入 `log_upload_secret`**（grep `LOG_UPLOAD` 在 `.github/` 零命中）→ CI checkout 拿到空占位 → 端点空 → `showUploadLogAction=false` → 按钮在**所有平台**（不只 Windows）永久隐藏。这是构建/密钥注入缺口，不是平台门控误排除。
- **[x] ① 已修复** — 在三个出包 workflow 的每个 `Provide gitignored Google OAuth secret stub` 步骤后，对称插入 `Provide gitignored log upload secret stub` 步骤（共 6 处：build-multiplatform 4 + release 1 + release-desktop 1），照搬 google_oauth 范式：配了 `LOG_UPLOAD_ENDPOINT`/`LOG_UPLOAD_TOKEN` repo secret 时把真值写进 `log_upload_secret.dart`（按钮显示且可用），未配置则保留入库空占位（按钮隐藏、不暴露端点，向后兼容 fresh CI）。**是否真显示按钮取决于仓库管理员在 GitHub 配置这两个 secret**（端点不能硬编码入库）。提交见分支 `todo-385-win-log-upload` HEAD。文件：`.github/workflows/build-multiplatform.yml` / `release.yml` / `release-desktop.yml`。
- **[x] ② 已加自动化测试** — `hibiki/test/build/log_upload_workflow_injection_guard_test.dart`：钉死「每个注入 google_oauth 的 workflow 必须对称注入 log_upload」（数量对称 + 引用真 secret + 写两个常量 + 回退 example），防将来加新平台 job 时漏注入复发。配合既有 `hibiki/test/pages/log_upload_button_test.dart`（门控接线）。
- **备注**：HTTP 上传 `log_uploader.dart` 桌面可用（用 `http` 包，无平台限制），放开密钥即可在 Windows 真用。真机/CI 验证项：在 GitHub 配置 `LOG_UPLOAD_ENDPOINT`/`LOG_UPLOAD_TOKEN` 后跑一次 Windows 构建，确认按钮出现且上传 200。次要瑕疵（不在本 bug 范围）：`log_upload_secret.example.dart` 注释称该文件「被 .gitignore 忽略」，实际是 tracked + skip-worktree（与 google_oauth 同机制），注释与实现不符。
