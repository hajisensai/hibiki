## BUG-511 · TODO-1066 app 外查词（桌面全局查词）的快捷键没办法设置
- **报告**：2026-07-01（用户：）
- **真实性**：✅ 真 bug（仅桌面 Windows 命中；Android 侧是**正确**行为，见下）。根因：桌面「app 外全局查词」触发热键把 Ctrl+Alt+D **硬编码**直接注册到 `hotkey_manager`，绕过整个快捷键注册表 `HibikiShortcutRegistry`。`GlobalLookupController.start`（`hibiki/lib/src/lookup/global_lookup_controller.dart:117-128`，旧版）写死 `HotKey(key: PhysicalKeyboardKey.keyD, modifiers: [control, alt], scope: system)` 并直接 `hotKeyManager.register`，与注册表零耦合（不经 `resolveKeyboard`、不读 `_bindings`、不在 `shortcut_defaults`）。而快捷键设置页 `_buildScopeSections`（`hibiki/lib/src/pages/implementations/shortcut_settings_page.dart`）严格遍历 `ShortcutScope.values × ShortcutAction.actionsForScope`，只渲染注册表里登记的 action；可绑动作真相源 `ShortcutAction`（`hibiki/lib/src/shortcuts/shortcut_action.dart`）里**根本没有「app 外查词」动作** → 设置页无行、无分配入口 → 用户无从改键。**平台真伪区分**：(A) Android 的 app 外查词 = 纯系统 PROCESS_TEXT（`AndroidManifest.xml`）/ 分享 / 悬浮球，系统触发；Android 不允许第三方注册系统级文本选择快捷键，本无 app 内快捷键概念 —— 设置页缺该项**是正确的**，只需文案说明。(B) 桌面 Windows 全局查词才是本条真缺配置入口。
- **[x] ① 已修复** — commit 见下。根因向修复，无补丁式绕过：
  - 新增可绑动作 `ShortcutAction.globalExternalLookup`，落在**独立**作用域 `ShortcutScope.globalExternal`（自成 co-active 组，冲突检测只扫自己，绝不与 global/home 等页面 scope 牵连；因为它是系统级、跨页面常驻、不参与页面 `resolveKeyboard`）。`shortcut_action.dart`。
  - `shortcut_defaults.dart`：桌面（Windows/Linux）默认 `Ctrl+Alt+D`（与旧硬编码一致，行为不变），macOS 表自动把 Ctrl 换成 Meta；移动端（Android/iOS）该 scope 无绑定（系统级触发，无 app 内热键）。
  - `shortcut_registry.dart`：`kShortcutSchemaVersion` 3→4，登记 v3→v4 迁移说明。因是**全新 action**，老快照没有其 key，`loadDefaults` 已播种平台默认、`_loadFromJson` 只覆盖显式出现的 key，故老用户升级后天然拿到默认热键，不误伤已有绑定。
  - `global_lookup_controller.dart`：`start` 改为 `_registry = appModel.shortcutRegistry; _registry.addListener(_onRegistryChanged); await _registerHotKeyFromRegistry()`。新增 `_registerHotKeyFromRegistry`（先 unregister 旧热键再 register 新键，从 `registry.bindingsFor(globalExternalLookup)` 读第一个键盘绑定）、`_onRegistryChanged`（改键 / profile 切换后重注册）、`_hotKeyFromBinding`（logical→physical 复用 `InputBinding.physicalKey`，ModifierKey→HotKeyModifier 映射）。热键的键/修饰符自此来自 binding，非硬编码。
  - 设置页 `_actionLabel`/`_scopeLabel` 补 case + i18n（经 i18n_sync）；移动端对 `globalExternal` scope 渲染系统级说明文案行（`shortcut_scope_global_external_mobile_note`）而非可改键行，桌面渲染正常 Ctrl+Alt+D 可改键行。
  - 改动文件：`hibiki/lib/src/shortcuts/shortcut_action.dart`、`shortcut_defaults.dart`、`shortcut_registry.dart`、`hibiki/lib/src/lookup/global_lookup_controller.dart`、`hibiki/lib/src/pages/implementations/shortcut_settings_page.dart`、i18n 17 文件 + `strings.g.dart`。
- **[x] ② 已加自动化测试** — `hibiki/test/shortcuts/global_external_lookup_hotkey_test.dart`：断言 `ShortcutAction` 含 `globalExternalLookup`、scope=globalExternal 且自成 co-active 组；桌面默认 == Ctrl+Alt+D / macOS == Meta+Alt+D / 移动端空；三平台默认表都登记（wiring guard 不变式）；schema bump ≥4；老快照（无该 key）升级后取到默认且不误伤用户已改的旧 action；用户清空后升级不被回填；**controller 热键来源守卫**（源码级钉死热键来自 `registry.bindingsFor(globalExternalLookup)` + `addListener(_onRegistryChanged)`，且不再硬编码 `key: PhysicalKeyboardKey.keyD` / `HotKeyModifier.control, alt` 常量 —— 填补现有 `global_lookup_controller_stack_test` / `m1a_guard` 只覆盖 overlay/stack 的盲区）；设置页 `_actionLabel`/`_scopeLabel` exhaustiveness（含 globalExternalLookup/globalExternal case）。另在 `shortcut_action_wiring_guard_test.dart` 的 executor 文件清单加入 `global_lookup_controller.dart`，让「每个 action 至少被一个执行体按枚举名派发引用」的不变式覆盖新 action。
