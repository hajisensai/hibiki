## BUG-089 · 制卡（Anki 导出）失败时原因全丢：toast 只显示通用「导出卡片失败」、错误日志页也查不到
- **报告**：2026-06-06（用户：制卡失败时既看不到原因、错误日志页也搜不到）。采番注：本 worktree 基线最大号 BUG-088，取 089（仍可能被并发抢，以提交为准）。
- **真实性**：✅ **真 bug（三层都把原因丢掉，沿真实代码路径定位）**。① **数据结构层**：`MineResult` 是纯 enum（`success`/`duplicate`/`notConfigured`/`error`，`anki_models.dart:524`），`error` 不携带任何附加字段。② **后端层**：两后端（`ankiconnect_repository.dart` / `ankidroid/anki_repository.dart`）在所有失败点（解析 payload、重复检测、addNote、空字段、顶层兜底）都只 `debugPrint` 真实异常后返回裸 `MineResult.error`；而 `debugPrint` 仅在用户手动开启 Debug 日志（默认关）时才进 app 日志，**根本不经 `ErrorLogService`**（那才是默认就记录、写 `error_log.txt`、显示在错误日志页的日志）。③ **UI 层**：5 处调用点（`dictionary_page_mixin` / `reader_hibiki_page` / `floating_dict_page` / `video_hibiki_page` / `app_model`）全把 `error` 映射成同一句 `t.card_export_failed`。⇒ 用户看不到原因、错误日志页也查不到。
- **[x] ① 已修复** — 本提交（自含修复+测试+本条目）。**根因修（修数据结构 + 把原因作为返回值带出，非补丁）**：新增 `MineOutcome`（`anki_models.dart`）= `result`（保留 `MineResult` enum，5 个 switch 调用点据此分支不破坏）+ `errorDetail`（简短人类可读原因 → toast）+ `error`/`stackTrace`（完整诊断 → 错误日志）。`mineEntry` 返回类型 `MineResult` → `MineOutcome`（base + 两后端 + 5 调用点 + 测试 fake 全同步）；两后端各失败点改返回 `MineOutcome.failure(简短原因, error:, stackTrace:)`（AnkiConnect 顶层/解析/重复检测/addNote、AnkiDroid 同位；AnkiConnect 把 `AnkiConnectException.message`、AnkiDroid 把 `PlatformException.message` 带出）。`hibiki_anki` 是独立包不依赖主 app 的 `ErrorLogService`，故原因作为返回值带出，由主 app 层负责记日志 + 展示。UI 层新增单一真相源 `logMineFailure(MineOutcome)`（`error_log_service.dart`）：① 把 `error ?? errorDetail` + `stackTrace` 写进 `ErrorLogService`（错误日志页可查完整 stack），② 返回 toast 文案（有 `errorDetail` 走新 i18n `card_export_failed_detail`，无则降级到通用 `card_export_failed`）；5 个 error 分支统一调它，避免「记日志+文案」复制 5 份漂移。i18n 经 `tool/i18n_sync.dart --add card_export_failed_detail`（17 语言完整，`$reason` 占位）+ `dart run slang` 重生成。
- **[x] ② 已加自动化测试** — `packages/hibiki_anki/test/mine_entry_never_throws_test.dart`（扩展：顶层 catch 路径断言 `outcome.error is StateError` + `stackTrace != null` + `errorDetail` 非空 = 原因被带回；`MineOutcome` 构造契约）+ `hibiki/test/anki/mine_failure_surfacing_test.dart`（新增：`logMineFailure` 行为——把完整 error+stack 写进 ErrorLogService、有/无 detail 的 toast 文案分支；源码守卫——5 个调用点 error 分支都走 `logMineFailure(outcome)`，不再用裸 `t.card_export_failed`）。验证：hibiki_anki 包 40 绿、app 全量 **2789 绿 0 失败**（~2 跳过），`flutter analyze lib` + 包 analyze 均 0。
- **备注**：Anki 制卡类。代码正确 + 无回归；**真机复测待用户**：制造一次制卡失败（如 AnkiConnect 未开/牌组未配/重复卡），确认 ① toast 显示具体原因（如 "AnkiConnect: cannot create note because it is a duplicate"）② 错误日志页出现一条 `Anki.mineEntry` 含完整 stack。简短原因为技术英文（多为 AnkiConnect/AnkiDroid 自身英文报错），按用户「toast 只简短原因 + 写错误日志」的决定，前缀经 i18n 本地化、原因保留技术原文以利诊断。
