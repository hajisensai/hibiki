## BUG-477 · 查词弹窗右键同时弹WebView2原生菜单与自定义菜单(双菜单·BUG-468同根·弹窗WebView漏修)
- **报告**：2026-06-30（用户：TODO-1018「右键的时候会出现这种清空」，附图为查词弹窗内右键弹出 WebView2 原生菜单 返回/刷新/另存为/打印/更多工具 + 自定义 搜索/复制 两个菜单）
- **真实性**：✅ 真 bug — 根因 `hibiki/lib/src/pages/implementations/dictionary_popup_webview.dart:702`（`initialSettings: InAppWebViewSettings(...)` 缺 `disableContextMenu`）。这是 BUG-468 的**同根遗留**：BUG-468 给阅读器主 WebView（`reader_hibiki/webview.part.dart`）加了 `disableContextMenu: isWindowsPlatform`，但查词弹窗是**独立的第二个 WebView**（`dictionary_popup_webview.dart`），其设置从未压制原生菜单。弹窗里虽有 `ContextMenu(settings: ContextMenuSettings(hideDefaultSystemContextMenuItems: isWindowsPlatform))`（:670）+ Windows 下 `_showWindowsContextMenu` 的 Flutter showMenu 接管（:1202），但 `hideDefaultSystemContextMenuItems` 是跨平台 ContextMenu API，在 flutter_inappwebview_windows fork 上**并不接到**原生菜单开关；fork 里压制 WebView2 原生菜单的唯一真值是 `InAppWebViewSettings.disableContextMenu`→`put_AreDefaultContextMenusEnabled(!disableContextMenu)`（`packages/flutter_inappwebview_windows/windows/in_app_webview/in_app_webview.cpp:231`）。弹窗 `initialSettings` 从未设 `disableContextMenu`（默认 false）→ 原生菜单一直开着，与 Flutter 自定义菜单重叠双弹（用户感知「右键出现清空」=原生菜单 刷新/另存为 等项盖在弹窗上）。
- **[x] ① 已修复** — `dictionary_popup_webview.dart` 的 `initialSettings: InAppWebViewSettings(...)` 加 `disableContextMenu: isWindowsPlatform`：Windows 关掉弹窗 WebView2 原生菜单只留 Flutter 选区菜单；移动端值为 false 不动原生 `ContextMenu`（查词项），不回归。与 BUG-468 同一外科范式，仅作用于弹窗 WebView。提交哈希：见 integration owner 落地。
- **[x] ② 已加自动化测试** — 源码守卫 `hibiki/test/pages/popup_webview_disable_context_menu_guard_test.dart`：断言弹窗 WebView 的 `InAppWebViewSettings` 块里 `disableContextMenu: isWindowsPlatform` 在位、且**不是**无条件 `true`（防移动端丢菜单）。镜像 BUG-468 的 reader 守卫 `reader_webview_disable_context_menu_guard_test.dart`。
- **备注**：与 BUG-468 同根（跨平台 ContextMenu hide 标志 ≠ WebView2 原生菜单开关），区别在受影响的 WebView 表面：468=阅读器主 WebView，477=查词弹窗独立 WebView。两处均需 `disableContextMenu`。真机待验：Windows 查词弹窗内右键应只出 Flutter 搜索/复制菜单，不再弹原生 返回/刷新/另存为/打印/更多工具。
