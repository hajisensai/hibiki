## BUG-058 · 词典管理某类型 tab 无该类型词典时的空状态样式与「完全无词典」不一致（左对齐灰卡 vs 居中带图标提示）
- **报告**：2026-06-05（用户：截图 2「汉字词典」tab 显示「请先导入词典以便使用」，「这个请先导入词典的显示也有问题」，经追问确认指「样式不统一/难看」）。采番注：与 BUG-057 同轮，取 058（编号可能被并发 agent 抢，以提交为准）。
- **真实性**：✅ **真 bug（同一空状态两套外观）**。`dictionary_dialog_page.dart`：完全无任何词典走 `buildEmptyMessage()`（`HibikiPlaceholderMessage`，居中、带词典图标、`vertical: card+gap` 内边距）；而某类型 tab（如「汉字词典」）无该类型词典、但其它类型词典存在时走 `_buildEmptyCategoryRow()`——一张**左对齐**的 `HibikiCard` 内裹裸 `Text`，与前者外观/对齐/留白都不同。在 tab 间切换到恰好为空的类型时，空状态忽而居中带图标、忽而左对齐灰卡，观感割裂。
- **[x] ① 已修复** — `_buildEmptyCategoryRow()` 改为复用与 `buildEmptyMessage` 同款的 `HibikiPlaceholderMessage`（同 `DictionaryMediaType.instance.outlinedIcon` 图标、同 `t.dictionaries_menu_empty` 文案、同 `vertical: card+gap` 居中留白），删掉左对齐 `HibikiCard`+`Text`。`dart format` 通过、改动文件 `flutter analyze` 0 issue。
- **[x] ② 已加自动化测试** — `hibiki/test/pages/dictionary_dialog_layout_static_test.dart`（源码扫描守卫，最强可落地层——`DictionaryDialogPage` 需完整 `appModel` 难在单测 pump，沿用本文件既有静态守卫范式）新增「per-category empty state matches the all-empty placeholder (BUG-058)」：截取 `_buildEmptyCategoryRow` 函数体，断言含 `HibikiPlaceholderMessage` + `DictionaryMediaType.instance.outlinedIcon`、且**不含** `HibikiCard` / `child: Text(`（防回退到左对齐灰卡）。该文件 11 例全绿。
- **备注**：词典管理/空状态/MD3 一致性类。纯原生组件、缩放下不糊，**真机肉眼复测**（任意平台：词典管理→切到无词典的类型 tab，应见居中带图标的空提示）**待用户**。
