## BUG-041 · 「Local vs Remote」对比对话框点 Apply 不下载远端独有书
- **报告**：2026-06-04（计划 `docs/specs/2026-06-04-sync-gdrive-oom-and-compare-download-plan.md` Part B；用户：谷歌云盘同步「没下载书籍」）。采番注：计划暂定 BUG-037，动手前查 `docs/BUGS.md` 当前最大号为 BUG-040（036/037/038/039/040 已被并发 agent 占用），故本条取 041。
- **真实性**：✅ **真 bug（remote-only 书被设计性排除），确定性**。沿真实代码路径逐行验实：对比对话框（`sync_compare_dialog.dart`）远端独有书 `bookId==null`（`SyncCompareEntry.bookId = local?.id`，本机没有则 null），被 `_load` 的 `if (e.bookId == null) choices=skip` 强制跳过、被 `_applyChoices` 的 `actionable` 过滤（`e.bookId != null`）与 `_actionableCount` 同条件排除、`_buildEntry` 也不给它渲染任何选择控件 → 这本远端书在对话框里永远无选项、永不计入 Apply、永不下载。唯一能下载远端独有书的路径是 `sync_orchestrator.importRemoteBooks`，被 `if (syncContent)` 门控、只由全量 `run()` 调用，对话框从不走它。根因点 = `sync_compare_dialog.dart` 的 `_load`/`_applyChoices`/`_actionableCount`/`_buildEntry` 全部以 `bookId != null` 排除 remote-only 书，且唯一下载路径 `importRemoteBooks` 不被对话框调用。
- **[x] ① 已修复** — `276f485af`（抽出可复用单书下载导入）+ `1110aeb5c`（对话框接入下载）：根因修——消除「remote-only 书没有任何对账动作」这个特殊情况，而非补丁。① `sync_orchestrator.dart` 抽出顶层 `importRemoteBookFolder({db,backend,folderId,tempDir})`（下载文件夹内 `.epub` → `EpubImporter.importFromPath`，临时文件用后即删），`importRemoteBooks` 改为循环调用它（orchestrator 行为完全不变，仅去重）。② 对话框：`_load` 对 `bookId==null` 且有 `remoteFolderId` 的书默认勾选「下载」（`SyncChoice.useRemote`，匹配「点 Apply 就把云端书拉下来」的用户直觉，且下载是显式用户动作不受 `syncContent` 门控）；新增 `_isActionable` 助手统一 `_applyChoices`/`_actionableCount` 的参与判定（非 skip 且 `bookId!=null || remoteFolderId!=null`）；`_applyChoices` 加 remote-only 分支调 `importRemoteBookFolder`，catch `DuplicateImportCancelledException` 当良性跳过；`_buildEntry` 给 remote-only 书渲染 `_downloadRow`（Checkbox 切 skip↔useRemote）；select-all 菜单兼顾 remote-only（只在 useRemote/skip 间切，忽略 useLocal）；构造器 + `showSyncCompareDialog` 加可选 `Directory? tempDir`。`dart analyze` 0 issue、`dart format` 0 改动。
- **[x] ② 已加自动化测试** — `a2201575e` · `hibiki/test/sync/sync_compare_download_test.dart`：行为级——fake `SyncBackend` 远端有一本本机没有的书（`listChildren` 返回一个 `.epub`，`getAsset` 写一个真实可解析的最小 EPUB：mimetype+container.xml+content.opf+xhtml），`pumpWidget(SyncCompareDialog(db, backend, tempDir))` 后断言 remote-only 条目渲染出 `t.sync_compare_download` 下载控件且 Checkbox **默认勾选**、Apply 按钮计数为 `Apply (1)`；点 Apply（`runAsync` 内驱动真实后台 isolate 导入 + 真文件 IO，因 `EpubImporter.compute` 与 Apply 转圈动画使 `pumpAndSettle` 不收敛，改 runAsync 轮询直到落库）后断言 `getAsset` 确被调用 + `db.getAllEpubBooks()` 多出该书（证明真把远端书下载导入了，非仅渲染控件）。`flutter test test/sync/` 全量 **267 绿**无回归。
- **备注**：同步/对比对话框/下载类。代码 + 单测绿、`dart analyze` 0 issue、`dart format` 0 改动。**真机复测原始失败路径待用户**：设置→同步→「Local vs Remote」对比 → 远端独有书应默认勾「下载」、点 Apply 后本机书架出现该书；或手机谷歌云盘「立刻同步」远端独有书也下载到本机。计划见 `docs/specs/2026-06-04-sync-gdrive-oom-and-compare-download-plan.md`（Part B）。
