## BUG-094 · 视频 tab 页头标题字号 / 动作按钮位置与书架、词典不统一
- **报告**：2026-06-06（用户：「视频的标题大小和书架的标题大小不统一，还有按钮的位置也是」）。采番注：初记为 092，rebase 到 develop 时发现 090/091/092/093 均已占用，改取 094。
- **真实性**：✅ **真 bug（结构性不一致，沿真实代码路径定位）**。根因=三个首页 tab 的页头**不同构**。书架（`reader_hibiki_history_page.dart _buildPageHeader`）与词典（`home_dictionary_page.dart _buildPageHeader`）都是「无 Scaffold 的纯 Column + `HibikiPageHeader`（MD3 大标题）+ `HibikiIconButton` 动作」，由 `HomePage` 提供的唯一外层 Scaffold 承载（`home_page.dart` mobile/desktop layout）；而视频页（`home_video_page.dart` build）**自带一个嵌套 `Scaffold` + `adaptiveAppBar`**——标题走 `Text(t.nav_video)`（AppBar 小号 titleLarge）、动作走裸 `IconButton(onPressed:)`（落在 AppBar 右侧、splash/尺寸与 HibikiIconButton 不同）。⇒ 同为首页 tab，视频标题明显比书架小、按钮位置/样式也对不齐。
- **[x] ① 已修复** — 本提交（自含修复+测试+本条目）。**根因修（让视频页与书架/词典同构，而非微调字号）**：删掉视频页自带的 `Scaffold` + `adaptiveAppBar`，改成 `DesktopContentLayout(kind: readerShelf)` + `Column`，首项 `if (!isCupertinoPlatform) _buildPageHeader(canImport)`；新 `_buildPageHeader` 用 `HibikiPageHeader(title: t.nav_video, actions: [HibikiIconButton(统计), if(canImport) HibikiIconButton(导入)])`——标题字号、动作控件/位置、桌面内容宽度三项均与书架一致。外层 Scaffold 由 HomePage 提供（`ScaffoldMessenger.of` 仍可用）。
- **[x] ② 已加自动化测试** — `hibiki/test/pages/video_experimental_markers_guard_test.dart`「视频页页头与书架/词典统一」组（源码守卫：用 `HibikiPageHeader` 不用 `adaptiveAppBar`、动作用 `HibikiIconButton(onTap:)` 不用 `IconButton(onPressed:)`、`DesktopContentKind.readerShelf` 约束宽度、页头同门控仅非 Cupertino）+ 更新 `home_video_page_menu_test.dart` 把页面 pump 在 `Scaffold` 内（页面不再自带 Scaffold，反映真实 HomePage 装配）。
- **备注**：UI 布局类。代码正确 + 无回归；**真机/桌面肉眼复测待用户**：视频 tab 与书架/词典切换时标题字号一致、统计/导入按钮与书架右上动作位置对齐。本次同时把视频从「实验性功能」设置开关毕业为常驻 tab（底栏图标加实验性小圆点徽标 + 视频页头下方实验性提示横幅），页头统一与这部分改动同提交。
