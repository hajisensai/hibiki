## BUG-251 · 导入对话框点文字标题没反应，只有右边图标可点
- **报告**：2026-06-14（TODO-307：导入界面点文字没反应，必须点右边 icon）
- **真实性**：真 bug。`hibiki/lib/src/utils/components/hibiki_material_components.dart:1863-1900` 的 `HibikiFilePickerRow` 提供了可选 `onTap`（`:1870`）并喂给 `HibikiListItem.onTap`（`:1888`），整行可点；但调用方 `hibiki/lib/src/media/audiobook/book_import_dialog.dart` 的四行（`_epubRow:223` / `_subtitleRow:239` / `_audioRow:267` / `_coverRow:455`）全部只把选择动作放进 `actions` 的 `HibikiIconButton.onTap`，从不传 `onTap` → `HibikiListItem.onTap` 为 null → 标题/副标题整行无手势，用户只能精准点击右侧图标。同款问题：`hibiki/lib/src/media/audiobook/audiobook_import_dialog.dart` 的 `_alignmentRow:432`（单一选择动作）也留空 `onTap`。
- **根因**：`HibikiFilePickerRow` 的整行点击契约靠调用方显式传 `onTap` 接通，而上述导入行只接了 trailing 图标按钮、漏传 `onTap`，使「点行 = 触发该行选择」这一直觉交互断链。`video_import_dialog.dart` 不使用 `HibikiFilePickerRow`，不受影响。`audiobook_import_dialog._audioSourceRow`（选目录/选文件两个等权动作）和已附加书的只读展示行（无 `actions`）无单一明确主动作，保持图标驱动。
- **[x] ① 已修复**：给单一主选择动作的导入行接通整行 `onTap`：`book_import_dialog.dart` 四行分别 `onTap: () => _pickEpub()` / `_pickSubtitle` / `_pickAudio` / `_pickCover`（`_pickEpub` 带可选具名参数，用闭包包成 `VoidCallback`）；`audiobook_import_dialog.dart` 的 `_alignmentRow` 接 `onTap: _pickAlignment`。trailing 的 close/clear/选择图标各自吃手势，不向行冒泡；已存在的 `_pickerActive` 重入保护让整行点和图标点都安全幂等。
- **[x] ② 已加自动化测试**：`hibiki/test/widgets/hibiki_file_picker_row_tap_test.dart` 守 `HibikiFilePickerRow` 组件契约——点击标题文字区触发 `onTap`、`enabled:false` 时不触发、trailing 图标点击只触发图标自身回调不冒泡到行 `onTap`；同文件源码扫描守卫断言 `book_import_dialog.dart` 四行与 `audiobook_import_dialog._alignmentRow` 均传了 `onTap`，防回归到「只有图标可点」。
- **备注**：纯交互/几何修复，逻辑路径未改。导入对话框走真机/桌面肉眼复测「点标题文字即弹文件选择器」仍待用户在设备上验证（CLAUDE.md 导入类验证纪律）。
