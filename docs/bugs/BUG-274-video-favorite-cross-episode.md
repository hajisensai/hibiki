## BUG-274 · 视频收藏句子面板跨集/跨视频污染（缺 bookKey 过滤）
- **报告**：2026-06-14（用户：row223 — 「本集收藏句子会把其他集的也放进去。我目前一共看的两集，都在里面了。（而且我是单集）」）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/media/video/video_favorite_sentences_panel.dart:21-27`（修前）：`VideoFavoriteSentencesPanel.build` 的过滤只判 `source == kFavoriteSentenceSourceVideo && sectionIndex == currentEpisode`，**没有按视频身份键 `bookKey` 过滤**。收藏写入时记的是 `bookKey = widget.bookUid` + `sectionIndex = _currentEpisode`（`video_hibiki_page.dart:2034-2035` / `2101-2102`），数据维度齐全；丢维度的是「本集收藏」面板的展示 query（`_buildFavoriteSentencesSidePanel` 在 `video_hibiki_page.dart:4355` 把 `getAll()` 全量喂给面板）。单集视频 `_episodes` 为空、`_currentEpisode` 恒 0（`video_hibiki_page.dart:656` 默认 + `825-852` 单视频路径不动它），于是每个单集视频的收藏都落在 `sectionIndex == 0`，面板仅按 `sectionIndex == 0` 过滤 → 把别的单集视频的收藏一并拉进当前视频的面板，复现用户「两集都在里面 + 我是单集」。对照：阅读器/有声书的收藏读取正确按 `bookKey` 隔离（`reader_hibiki_page.dart:2758-2759` 与 `2780-2781`），所以本 bug 仅限视频面板。
- **[x] ① 已修复** — 给 `VideoFavoriteSentencesPanel` 增 `currentBookKey` 必填参数，过滤增加 `sentence.bookKey == currentBookKey`（`video_favorite_sentences_panel.dart:6/22/30`）；调用点传 `currentBookKey: widget.bookUid`（`video_hibiki_page.dart:4364`）。属根因修复——补回写入时已有、展示时丢掉的视频身份维度，不靠前端额外过滤掩盖错误存储；无 schema/迁移改动（数据本身正确，旧条目 `bookKey` 早已写入）。提交：见提交哈希。
- **[x] ② 已加自动化测试** — `hibiki/test/media/video/video_side_panel_test.dart` 新增「isolates by bookKey across single videos (BUG-274)」widget 行为测试（两个 `bookKey` 不同、`sectionIndex` 同为 0 的单集视频收藏，断言另一个视频的句子不出现），实测撤掉过滤行 → 红，恢复 → 绿；并更新原「shows only the current episode」用例补 `bookKey`/`currentBookKey`。源码守卫：`hibiki/test/pages/sentence_favorites_todo047_guard_test.dart` 新增「BUG-274」组守 `sentence.bookKey == currentBookKey` 与调用点 `currentBookKey: widget.bookUid` 不回归。
- **备注**：`flutter analyze`（4 改动文件）No issues；`flutter test test/media/video test/media/audiobook/favorite_sentence_source_test.dart` 596 passed / 2 skipped(baseline)。真机复验：建议用户用两个独立单集视频各收藏一句后，分别打开各自「收藏夹/本集收藏」面板确认互不串入（host widget 测试已覆盖过滤逻辑，真机仅验证 UI 接线）。
