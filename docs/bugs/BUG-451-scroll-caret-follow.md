## BUG-451 · 连续模式滚动焦点环不跟随可视区
- **报告**：2026-06-28（用户：「滚动模式，焦点不会跟着页面动」）
- **真实性**：✅ 真 bug — 根因 `hibiki/lib/src/pages/implementations/reader_hibiki/navigation.part.dart` 的 `_handleReaderScroll` → `_refreshProgressFromScroll`：连续/滚动模式手动滚动回调只刷阅读进度并去抖落库，**从不重定位字符级焦点环（caret）**。caret 是阅读器正文焦点的唯一真值（JS `window.hoshiCaret`，Dart 经 `_caretRefresh()`/`_caretReanchor()` 驱动），**没有 JS scroll 监听**，只在 Dart 显式调用时移动。分页模式翻页走 `chrome.part.dart` 的 `_caretReanchor(direction)` 焦点跟随；连续模式手动滚动整链不碰 caret → 焦点环钉死在旧字符、随内容滚出视口。`refresh()` JS 原语（`hibiki/lib/src/reader/reader_caret_scripts.dart:83`）已能在锚字符离开视口时 `_firstVisibleStop()` 重锚首个可见字符，只是从未在手动滚动路径被调用。
- **[x] ① 已修复** — 在 `_refreshProgressFromScroll` 进度刷新落地的同一 50ms 节流相位（对齐 hoshi 安卓 `CONTINUOUS_PROGRESS_THROTTLE_MS`，高频滚动只刷一次 + 停止后尾沿补一发），经新增纯函数门控 `readerScrollCaretFollowAllowed`（`reader_hibiki_page.dart`）追加一次 `_caretRefresh()`。严格在既有 `readerScrollProgressRefreshAllowed`（恢复期/重锚 settle/歌词/未就绪统一抑制）门控之后；仅 `连续模式 + caretActive + caretOnReader`（正文、非弹窗非歌词）触发。分页模式 / caret 未激活 / 弹窗滚动零影响。改动文件：`reader_hibiki_page.dart`（新增纯谓词）、`reader_hibiki/navigation.part.dart`（门控调用点）。提交：见下。
- **[x] ② 已加自动化测试** — `hibiki/test/reader/scroll_caret_follow_test.dart`：① `readerScrollCaretFollowAllowed` 门控真值表（连续+caret正文→true；分页/未激活/非正文→false）；② 源码守卫锁定 `_refreshProgressFromScroll` 内经该纯函数门控调 `_caretRefresh()`、且落在穿过 50ms 节流闸门之后（与进度刷新同相去抖，非另起 Future.delayed）。red→green：临时移除调用点 → 两源码守卫转红；恢复 → 全绿。
- **备注**：真机未验，留用户验：连续/滚动模式下用键盘/手柄激活焦点环后滚动正文，焦点环应跟随可视区重锚到首个可见字符。
