- **报告**：2026-06-19（用户：B03 验收第 3 条 / TODO-566）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/pages/implementations/video_hibiki_page.dart`：打开字幕列表 `_toggleSubtitleJumpList()`（open 分支，原 `:2001`）里 `unawaited(_refreshFavoritedCueCache())` 在面板已可见时再发一次异步整表查 DB。
  - 数据流：点字幕列表入口 → `_toggleSubtitleJumpList` 把 `_subtitleListVisible.value = true`（面板**同帧立即渲染**，行内星标 `isCueFavorited` 读缓存 `_favoritedVideoSentences`）→ 紧接着 `unawaited(_refreshFavoritedCueCache())` → 内部 `await FavoriteSentenceRepository.getAll()`（`packages/hibiki_audio/.../favorite_sentence_repository.dart:89`：读 `preferences` JSON → `jsonDecode` 整个收藏列表 → 排序）→ 返回后 `setState` 整体替换缓存重建。
  - 本质：收藏缓存 `_favoritedVideoSentences` 本就是单一真相源——视频 load 时 `_applyLoad` 内（`:1762`，`_seedWarmPopup(); … _refreshFavoritedCueCache()`）已预填一次，之后列表行 toggle（`_toggleFavoriteCueForVideo`）、查词浮层 toggle（`_toggleFavorite…`）都增量维护它。打开面板时再异步整表重查纯属冗余：面板先以缓存渲染、DB 往返（IO + 整表 jsonDecode + 排序）后才 `setState` 重建，已收藏行的实心星标因此「等一会」才从空心刷成实心。视频 load 阶段（demux 字幕 / 预热 WebView / 加载弹幕）IO 繁忙时这次往返被进一步推迟，延迟更明显。
