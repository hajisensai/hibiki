## BUG-029 · 「管理音频来源」本地库行「调整」按钮与开关列错位 + 缩放后长按拖拽排序飞出屏幕
- **报告**：2026-06-04（用户，附两张截图：① 本地库 `android.db` 行的「调整(tune)」按钮夹在开关与删除之间、各行开关/删除列对不齐，建议把调整按钮放开关左边；② 把界面大小调小后，长按拖动某行排序时被拖的行飞到屏幕右下角）。
- **真实性**：✅ **两个真 bug，同一处 `AudioSourcesDialog._buildSourceRow`（`dictionary_settings_dialog_page.dart`）**。
  - **① 列错位**：trailing 是右贴边的 `Row(MainAxisSize.min)`，原顺序 `[Switch][↑][↓][tune?][delete]`。`tune` 仅在本地库行出现（`isLocal && onEditLocalSources!=null && path 非空`），夹在 `↓` 与 `delete` 之间。min-size 右贴边下 `delete` 对齐、向左 `↓/↑/Switch` 依次排；本地行多出的 `tune` 把 `Switch/↑/↓` 整体往左推一个按钮宽 → 本地行开关比远端行开关偏左 = 跨行开关列错位（截图1 实证 `android.db` 开关明显偏左）。根因 = 把「仅本地行才有的 tune」放在公共列中间，挤动公共列。
  - **② 拖拽飞出**：界面大小是 `HibikiAppUiScale`（`main.dart:501`，在 `MaterialApp.builder` 里包住 Navigator/Overlay）用 `FittedBox` 等价 `Transform.scale` 缩放整棵树，**根 Overlay 也在缩放内**。该列表用 `ReorderableListView` + `ReorderableDelayedDragStartListener`（整行长按拖拽）。Flutter SDK 的拖拽代理（`reorderable_list.dart` 的 `_DragItemProxy`）用「全局坐标 − overlay 原点」的**纯平移**把代理放进 Overlay 本地坐标系，不认祖先缩放变换；当 Overlay 在 `scale=s` 下时，代理实际落点按 `(1−s)×(指针到 overlay 原点距离)` 漂移，缩小（s<1）时一拖即飞出屏幕（截图2）。根因 = SDK 对 `Transform` 内 Draggable/Reorderable 的已知坐标限制（外部/平台层缺陷，app 内无法改 SDK 数学）；该对话框已有 `↑/↓` 箭头按钮作为**完整等价重排路径**（注释明确标为「无障碍/手柄重排路径」）。
- **[x] ① 已修复** — (a) 列错位：把 `tune` 块移到 trailing `Row` 的**最前**（开关左侧），公共列 `Switch/↑/↓/delete` 在所有行右贴边对齐，`tune` 只在本地行向左凸出、不再挤动公共列——改布局结构消除「公共列被本地专属按钮挤动」的特殊情况，非补丁。(b) 拖拽飞出：**初版用「非 1.0 缩放禁拖拽 + 箭头替代」的门控（提交 `272fb19ed`），用户拒绝——禁用是阉割功能、非根因修。改用真修法**：新增自实现组件 `HibikiReorderableColumn`（`lib/src/utils/components/hibiki_reorderable_column.dart`，~210 行），两个对话框（`AudioSourcesDialog._buildSourceList` + `LocalAudioSourcesDialog`）改用它，彻底弃用 `ReorderableListView`。核心：**不借助任何 Overlay**——拖拽中的浮层复制就渲染在本列表自身的 `Stack`（`Positioned`，列表本地坐标），随整棵子树被祖先 `Transform.scale` 统一缩放 → 视觉与其余缩放界面**完全一致**；所有指针坐标都用本列表 `RenderBox.globalToLocal(globalPosition)` 转成本地坐标再定位浮层，`globalToLocal` 自动消掉祖先任意缩放/平移 → 浮层在任意缩放系数下**精确跟手、零偏移**（绕开 SDK `_DragItemProxy` 的全局↔本地空间错配）。被拖行原位 `Opacity(0)` 保留高度充当随实时重排移动的空位；行身份 `Key` 用 `keyForIndex` 保证拖拽中 `GestureDetector` 元素与活跃长按识别器跨重排不被销毁。长按用默认 `kLongPressTimeout`（快速滑动仍交外层滚动）。`↑/↓` 箭头照常工作。`flutter analyze` 0 issue、`dart format` 已跑。
- **[x] ② 已加自动化测试** — ① `test/widgets/hibiki_reorderable_column_test.dart`（3 条，组件级真手势）：默认缩放下长按拖拽重排；**`HibikiAppUiScale(scale:0.5)` 下长按拖拽仍真实重排、无飞出/异常**（正是 SDK `ReorderableListView` 失败之处）；不按住直接滑不触发重排（长按门控）。② `test/pages/audio_sources_dialog_page_test.dart`：本地/远端行开关列对齐 + tune 在开关左侧（BUG-029 ①）；断言对话框用 `HibikiReorderableColumn` 而非 `ReorderableListView`；**0.5 缩放下在真对话框上长按拖拽 A 行越过 B 行、关闭后 `onSave` 顺序确为 A 落到 B 之后**。③ `test/pages/local_audio_sources_dialog_test.dart`：子来源对话框同款（用 `HibikiReorderableColumn` + 0.5 缩放下拖拽 `nhk16` 越过 `forvo`、`onApply` 顺序生效）。`test/pages` + `test/widgets` 全量 364 绿无回归。
