## BUG-031 · 有声书音量调完退出重开书后不保存（恒回 1.0）
- **报告**：2026-06-04（用户：「有声书的音频、倍速都没保存，修复」。倍速经核查配线正确，见下「未复现」）。
- **真实性**：✅ **音量真 bug（持久化原语完全缺失）**。有声书的 delay / speed / imagePause 三项都有完整的「`onXxxPersist` 回调 + `load(initialXxx:)` 初值 + repo `readXxx`/`updateXxx`」三件套；唯独**音量没有任何持久化**：`AudiobookPlayerController.setVolume`（`audiobook_controller.dart:191`，修复前）只 `_player.setVolume` + `notifyListeners`，没有 persist 回调；`load()` 没有 `initialVolume` 参数；`AudiobookRepository`（`audiobook_repository.dart`）没有 volume 读写键。所以每次开书音量都从 just_audio 默认 1.0 起、用户调的值进不了 DB → 退出重开恒回 1.0。根因 = 音量漏配了与 speed 同型的持久化链。**倍速（speed）经核查未复现**：`onSpeedPersist`（`reader_hibiki_page.dart` 两条控制器路径都装）+ `load(initialSpeed: repo.readSpeed(...))`（`audiobook_controller.dart:378` 应用）+ `updateSpeed`/`readSpeed`（`audiobook_repository.dart:139/133`）三件套齐全且正确，代码路径上倍速按书持久化成立，未发现回归点。
- **[x] ① 已修复** — `ddabc0b50`：给音量补齐与 speed 完全同型的持久化链（消除「唯独音量没有持久化」这个特殊情况，非补丁）：(a) `AudiobookPlayerController` 加 `onVolumePersist` 回调，`setVolume` 钳值后比对 prev、值变化才落 persist（与 `setSpeed` 同避免拖动写库风暴），`load()` 加 `initialVolume` 参数、非默认值才下发（load 不触发 persist，与 speed/delay 同——载入非用户操作）；(b) `AudiobookRepository` 加 `readVolume`/`updateVolume`（`audiobook_volume_<uid>` 前缀，string 存储，镜像 speed）；(c) `reader_hibiki_page` 两条控制器初始化路径（audiobook / srt）各接 `repo.readVolume` → `initialVolume:` → `onVolumePersist`。弹窗音量滑条 `_buildVolumeSection` 的 `ctrl.setVolume(v)` 无需改，现已内部落库。`flutter analyze` 0 issue、`dart format` 0 改动。
- **[x] ② 已加自动化测试** — 三层守卫：① `hibiki/test/media/audiobook/audiobook_volume_persist_test.dart`——repo 往复（真 `HibikiDatabase` 内存库）：默认回退 1.0、按书写入互不串味、覆写读回最新；② `hibiki/test/media/audiobook/audio_persist_wiring_static_test.dart`——源码守卫钉 reader 两条路径都接了 `onVolumePersist`/`readVolume(`/`initialVolume:`（各 ≥2 次），并钉 speed 既有接线不被回归删；③ `packages/hibiki_audio/test/audiobook/audiobook_volume_persist_wiring_test.dart`——源码守卫钉控制器侧 `onVolumePersist`/`initialVolume`/`setVolume` 体内落 persist。`setVolume` 运行期需真实 just_audio player、单测不便驱动，故运行期用源码守卫兜底（与 BUG-026/028 同范式）。`hibiki_audio` 全量 11 绿。
