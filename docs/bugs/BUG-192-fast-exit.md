## BUG-192 · 桌面 app 退出慢（几秒~十几秒）
- **报告**：2026-06-11（用户：关闭要好久）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/main.dart:401`（修复前）`await windowManager.destroy()` 在桌面 `onWindowClose`（`setPreventClose(true)` 拦截 WM_CLOSE 后）触发原生 WM_DESTROY → `windows/runner/flutter_window.cpp` 的 `flutter_controller_ = nullptr` **同步销毁整个 Flutter 引擎 + 所有插件原生资源**，逐个串行拆：WebView2（`flutter_inappwebview_windows`，阅读器主 WebView + 常驻 warm-slot 弹窗 WebView 全会话存活从不显式 dispose）、WGC 捕获、libmpv（media_kit）。每个原生 teardown 几百 ms~秒级，串行叠加成几秒~十几秒卡死 UI 线程（=「关闭要好久」）。次因（仅开 LAN 同步主机用户）：`main.dart:419` `shutdownForExit().timeout(3s)`，Bonsoir mDNS 原生 stop 不归吃满 3s。
  - 数据完整性事实：`ProviderContainer` 从不 dispose（`main.dart` 无 `container.dispose()`），退出路径**不调用** reader/video 页面的 flush，也不 close database；且 `setPreventClose(true)` 下点 X **不触发**页面 `dispose()` / `paused`/`inactive` 生命周期。退出时 reader 的位置（500ms debounce）/ 阅读统计、video 的位置 / 观看时长（60s 周期）这些经后台 isolate 异步写 Drift 的 pending 写，过去全靠 `destroy()` 拖的几秒里**碰巧** fsync 完成（运气，非保证）。改 exit(0) 快杀必须先确定性 flush 这些写，否则丢进度/统计。
- **[x] ① 已修复** — 新增 `ExitFlushRegistry`（`hibiki/lib/src/startup/exit_flush_registry.dart`）：活跃页面 `initState` 登记退出 flush 回调、`dispose` 注销。桌面 `onWindowClose` → `_flushAndExitForWindowClose()`（`main.dart`）：①`shutdownForExitFast()`（同步 cancel Bonsoir 事件订阅防 TODO-036 崩，原生 stop 后台化，超时 3s→1.5s）→②`ExitFlushRegistry.flushAll()`（await 所有页面缓存值落库，**不碰退出期正在拆的 WebView**）→③`appModel.closeDatabase()`（WAL checkpoint 排空后台 isolate pending 写）→④`exit(0)`（`desktop_lifecycle_service.exitApp()`），跳过 `destroy()` 同步插件拆除，毫秒级返回。reader `_flushAllForProcessExit()` 用 `_flushPosition`（debounce 缓存值，不依赖 `_syncPositionFromWebViewProgress` 的 WebView eval）+ `_flushReadingStats`（改 `Future<void>` 可 await）+ 有声书 flush。video `_flushAllForProcessExit()` = `flushPosition` + `_watchTracker.stop()`（`VideoWatchTracker.addStat` 改 `FutureOr<void>`、`_flush`/`stop` 改 `Future` 可 await 统计写）。提交：见本轮 commit。
- **[x] ② 已加自动化测试** — `hibiki/test/startup/exit_flush_test.dart`（ExitFlushRegistry 行为：flushAll await 全部回调 / unregister 不跑 / 抛异常隔离 / 卡住回调被 timeout 不阻塞退出 / flushAll 清空；main.dart 源码守卫：无 `windowManager.destroy()` 字面、退出顺序 flush→closeDB→exit(0)、fast 路径 1.5s；reader/video initState 注册退出 flush 且退出 flush 不依赖 WebView eval）；`hibiki/test/media/video/video_watch_tracker_test.dart` 加退出 await 回归（异步 addStat：撤掉 `_flush/stop` 的 await 转红）；`hibiki/test/sync/server_exit_teardown_test.dart` 更新旧 destroy 守卫为新快杀契约 + 加 `shutdownForExitFast` 切断事件源测试。3 文件 39 用例 + 全套 sync/video/database/reader/audiobook/pages 绿。
- **备注**：真机 Windows 退出计时（点 X 到进程消失）+ 退出不丢进度/统计需用户验证（host 无法跑桌面真实退出）。移动端退出走 FlutterExitApp/moveTaskToBack 未动；`detached` 生命周期兜底（`_shutdownSyncSources`）保留幂等、超时同收紧到 1.5s。冲突组 app-lifecycle，高冲突文件 main.dart。
