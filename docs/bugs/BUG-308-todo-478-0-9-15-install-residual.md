## BUG-308 · TODO-478: 0.9.15 installer can leave old running build installed
- **报告**：2026-06-17（TODO-478：TODO-445 在 0.9.15 负向反馈的取证与必要修复）。
- **真实性**：✅ **真取证问题（安装残留 / 运行中旧包），本轮未确认 reader/audiobook 运行时代码在 0.9.15 实包内仍失败**。实包链路先固化：`v0.9.15-debug.5094+214910d` 指向 `214910da6e954a15d248fbf18081e0e42f042198`；GitHub release 的 Windows asset `hibiki-0.9.15-debug.5094-windows-setup.exe` digest 为 `sha256:8028cb8ed46f536189e983eea00794996d8c963fb33f349d17702e6d9c98fde7`，本机下载后 `Get-FileHash` 同值。安装前本机 registry / exe 均为 `0.9.17-debug.5110`，且 `D:\APP\Hibiki\hibiki.exe` 正在运行；此时执行 0.9.15 silent setup 返回 exit code 0，但安装日志显示 `Defaulting to Cancel for suppressed message box ... Hibiki is currently running` 后 `Got EAbort exception`，安装后 registry / exe 仍是 `0.9.17-debug.5110+5110`。关闭 Hibiki 后重跑同一 setup，日志记录复制 `D:\APP\Hibiki\hibiki.exe` 并 `Installation process succeeded`，registry 变为 `0.9.15-debug.5094`，`Hibiki.exe` `ProductVersion/FileVersion` 变为 `0.9.15-debug.5094+5094`。`package_info_plus` Windows 实现读取 exe `ProductVersion` 并按 `+` 拆成 `version/buildNumber`，所以本机安装后的 PackageInfo 证据对应 `version=0.9.15-debug.5094`、`buildNumber=5094`。这解释了“看似执行了 0.9.15 installer 但实际仍在旧 build”的残留路径；不能用源码已有修复关单，也不能把第一次 silent install 的 exit code 0 当作安装成功。
- **[x] ① 已处置（不改 reader/audiobook 代码）** — 本轮确认 0.9.15 tag 已包含连续滚轮和 cue 高亮相关旧修复：`86be62bef`（横排连续滚轮放行原生滚动）、`ca37839d8`（竖排连续 wheel 投到横向 `scrollBy`）、`7aaa5eadd`（reader sasayaki payload 带 cue text）、`28255d146`（收藏/音频高亮可见性）均为 `214910da` 祖先；且 `214910da..137e502` 在 `reader_hibiki_page.dart` / `highlight_bridge.dart` / `reader_content_styles.dart` / 对应测试上无 diff。因此没有证据支持在本任务内对 reader/audiobook/WebView 做无依据改动。
- **[x] ② 已加取证/验证记录** — 证据文件位于 `.codex-test/todo-478/`：`release/release-v0.9.15.json`、`release/windows-setup-sha256.txt`、`release/install-0.9.15.log`（运行中 silent setup 被 EAbort）、`release/install-0.9.15-after-close.log`（关闭进程后安装成功）、`release/installed-before.txt`、`release/installed-after-close-0.9.15.txt`、`release/versioninfo-after-close-0.9.15.txt`、`release/db-current-media-audiobook-schema-correct.txt`。targeted tests：`flutter test test/reader/swipe_page_turn_no_animation_test.dart test/pages/reader_hibiki_vertical_test.dart test/reader/favorite_audio_highlight_overlap_test.dart test/reader/ruby_highlight_guard_test.dart test/reader/reader_content_styles_test.dart --reporter expanded --no-pub`，77 tests passed。尝试跑 `integration_test/image_pause_realreader_test.dart` 的 Windows WebView 证据时，旧测试命中现有书库的 `かがみの孤城` 且 JS hook `Uncaught`，最后超时；该失败记录在 `.codex-test/todo-478/windows-image-pause-realreader.log`，不作为 0.9.15 reader/audiobook 负向结论。
- **备注**：用户复测 0.9.15 时应先确认无 `hibiki.exe` 进程，再安装 `hibiki-0.9.15-debug.5094-windows-setup.exe`，安装后用 registry 或 `D:\APP\Hibiki\Hibiki.exe` 的 `ProductVersion=0.9.15-debug.5094+5094` 作为包版本锚点；随后打开已导入的有声书（本机 DB 有 `謎解きはディナーのあとで` 9436 cues、`転生王女と天才令嬢の魔法革命` 9804 cues），分别验证竖排连续滚轮、横排连续滚轮和 cue 高亮。若在上述版本锚点确认后仍失败，再以该路径继续定位 reader/audiobook 小缺陷。
