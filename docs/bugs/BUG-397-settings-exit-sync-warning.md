## BUG-397 · 设置页退出100%弹同步进行中
- **报告**：2026-06-22（用户：）
- **真实性**：✅ 真 bug — `hibiki/lib/src/pages/implementations/home_page.dart:370`（顶层同步 PopScope `onPopInvokedWithResult`）+ `:635`（设置 tab 内层 PopScope `canPop:false`，BUG-236 修复引入）。
- **[x] ① 根因修复** — home 同一路由挂两个 PopScope：顶层同步 `PopScope(canPop:!syncing)`（syncing 时回调弹 `_SyncExitWarningDialog`）+ 内层设置 tab `PopScope(canPop:false)`（拦截返回切回来源 tab）。Flutter 3.44 同 route 多 PopScope 进无序 Set、popDisposition 按 OR 聚合（任一 `canPop:false` 即 route 不 pop），但 dispatch 时**所有** PopScope 的 `onPopInvokedWithResult` 都被遍历调用（`didPop=false`）。故设置 tab + app-open 同步同时进行时按返回：route 被内层否决（正确），但顶层回调也触发 → 误弹「同步进行中」告警。修复：顶层 `onPopInvokedWithResult` 在 `if(didPop) return;` 之后加 `if (_visibleTab == HomeTab.settings) return;`（`home_page.dart:382`），并抽纯函数 `bool shouldWarnOnExit({required bool syncing, required bool isSettingsTab}) => syncing && !isSettingsTab;`（`home_page.dart:80`）供单测与可读性；不动顶层 `canPop:!syncing`、不动内层 B、修正 `:621` 处错误注释（原称「内层先于顶层收事件」，实为全遍历 + OR-veto）。提交：本修复提交（分支 fix-698-settings-exit-sync，合并后由 integration owner 记录最终 SHA）。
- **[x] ② 加自动化测试** — `hibiki/test/pages/home_settings_exit_sync_warning_test.dart`：`shouldWarnOnExit` 纯函数 3 例（syncing×isSettingsTab 真值表）+ 与生产同构两层 PopScope（外层同步收窄 + 内层设置 `canPop:false`）的 widget 行为测试 3 例：设置 tab + 同步中按返回不弹告警（OR-veto 遍历仍被收窄拦下）/ 非设置 tab + 同步中仍弹（回归保护）/ 未同步不弹。撤掉收窄（`shouldWarnOnExit` 退回 `syncing` only）验证转红。BUG-236 守卫 `home_settings_back_intercept_test.dart` 三条仍绿。提交：本修复提交（分支 fix-698-settings-exit-sync，合并后由 integration owner 记录最终 SHA）。
- **备注**：纯 PopScope 回调收窄，未触平台原生层；Android 大屏（设置 tab + 后台同步进行时按系统返回 / 手势返回）真机复测原始失败路径待用户/集成测试确认。
