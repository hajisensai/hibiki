## BUG-040 · 点「编辑书籍 CSS」画面卡好久才出来（initState 在 UI 线程同步递归遍历整个解压书目录）
- **报告**：2026-06-04（用户：「点击编辑书籍 css，要把画面卡好久才出来，解决一下」）。采番注：动手前查 `docs/BUGS.md` 当前最大号为 BUG-039（038/039 已被并发 agent 占用），故本条取 040。
- **真实性**：✅ **真 bug（UI 线程同步递归遍历 + 同步读盘）**。沿真实代码路径定位：阅读器快捷设置「编辑书籍 CSS」行 `Navigator.push` 到 `BookCssEditorPage`（`reader_quick_settings_sheet.dart:467`）；该页 `initState`（`book_css_editor_page.dart:24`）同步调用 `_reload()`，里面 ① `BookCssRepository.discoverCssFiles()` 对**整个解压书目录**做 `dir.listSync(recursive: true)`（`book_css_repository.dart`），EPUB 解压目录含全部图片/字体/xhtml，动辄上千条目；② 再对每个 CSS 文件 `readCssSync()`（`File.readAsStringSync`）同步读盘。这一整串阻塞式文件 IO 全部跑在 UI isolate，正好卡在页面 push 转场动画期间 → 画面冻结到 `_reload` 跑完才显示。与 BUG-036 同源（同步 IO/重活跑 UI 线程）。根因点 = `book_css_editor_page.dart` 的 `initState`/`_reload` 在 UI isolate 同步执行 `discoverCssFiles()`（`listSync(recursive:true)`）+ 逐文件 `readCssSync()`。
- **[x] ① 已修复** — `1941da339`：根因修，把发现 + 读盘整体搬离 UI 线程（消除「初始化时在 UI isolate 同步阻塞遍历整树」这个特殊情况，非补丁）。① `BookCssRepository.loadSnapshots()`：用异步 `dart:io`——`dir.list(recursive:true)` 流式遍历 + `readCss()` 异步读，阻塞 syscall 跑在 Dart IO 线程池、结果经事件循环投递，UI isolate 全程不被阻塞、帧持续渲染（同步 `discoverCssFiles` 与异步路径共享纯函数 `_entriesFromCssPaths`，发现契约——排序、排除 `.original`/`.xhtml`——完全一致）。② 编辑器 `_reload()` 改 `async`，加载期间显示 `CircularProgressIndicator`，`build` 先走 `_loading` 分支。③ 顺手删 `_doSave`/`_doResetCurrent` 里**冗余的**同步 `discoverCssFiles()` re-walk——编辑/重置 CSS 内容不会改变 .css 文件集合，修改标记（`isDifferentFromOriginal`）本就实时读盘计算，那次 re-walk 只会在保存/重置时再次冻结 UI 线程，纯属浪费。**为何不用 `Isolate.run`**：初版用 `Isolate.run` 搬运，但 `Isolate.run` 的完成走真实事件循环 ReceivePort 消息，与 `flutter_test` 的伪时钟 `pumpAndSettle` 冲突（饿死超时，且无法在测试中驱动）；异步 `dart:io` 同样把 syscall 放 IO 线程池（真正离开 UI isolate），却能在测试中以 `runAsync`/`pump` 交替驱动，是更相称且可测的根因修。`flutter analyze` 0 issue、`dart format` 0 改动。
- **[x] ② 已加自动化测试** — `1941da339`：① 行为级 `hibiki/test/pages/book_css_editor_page_test.dart` 新增「opens off the UI thread」——`pumpWidget` 后**仅 pump 一帧**即断言显示 `CircularProgressIndicator` 且 tab（`a.css`）**尚未出现**（证明 populate 不再同步发生 = 卡死的源头已除），再交替 `runAsync`（让真实文件 IO 在真实事件循环推进）+ `pump`（排空伪时钟 zone 里 `_reload` 的 await 续延，二者缺一不可——这正是修复后异步路径的判别特征）驱动到 spinner 清除后断言 tab + 编辑器内容（`aaa`）渲染；原有 3 个 widget 测试改用同款 `pumpEditor` 助手（修复前同步 populate 时它们靠 `pumpAndSettle`，现在初始加载异步必须 `runAsync` 驱动，故 3 测试若退回同步路径会失败 = 防回归）。② `hibiki/test/epub/book_css_repository_test.dart` 新增 `loadSnapshots` group：断言异步路径与 `discoverCssFiles` 同序、排除 xhtml、每条已携带磁盘内容，且不存在目录返回空。`flutter test test/epub test/pages` 全量 **263 绿**无回归（含 reader_quick_settings/MD3 静态守卫）。
