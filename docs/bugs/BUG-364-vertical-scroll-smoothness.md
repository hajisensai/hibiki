## BUG-364 · 竖排连续(滚动)模式刷新率低/一格一格跳不顺 (TODO-629 ②)
- **报告**：2026-06-21（用户：阅读器竖排连续滚动模式「刷新率低，看着难受」）
- **真实性**：✅ 真 bug（体验缺陷，非崩溃）。根因 `hibiki/lib/src/pages/implementations/reader_hibiki_page.dart` 旧 :2544 —— 竖排(vertical-rl)连续模式滚轮被全部 preventDefault，逐 wheel 事件手动 `window.scrollBy({left: wheelDelta*sign, top:0, behavior:'auto'})`。竖排内容轴是横向(overflow-x 可滚、overflow-y:hidden)，桌面滚轮只产 deltaY、不投影到横向轴，故须手动投影；但 `behavior:'auto'` = 瞬时离散跳，每事件一次 deltaY 离散颗粒，丢弃浏览器原生平滑/惯性 → 看着像低刷新率/一格一格跳。横排(轴=纵向与 deltaY 同轴)走 `if(!vertical)return` 放行原生平滑，相对顺滑，不受影响。
- **[x] ① 已修复** — 竖排投影从逐事件 `scrollBy(behavior:'auto')` 改为 rAF 缓动：wheel 事件只把投影后的主 delta 累积进目标 `_vScrollTarget`（clamp 到可滚区间，方向随 vertical-rl/lr 的 sign），由 `requestAnimationFrame(_vScrollEaseStep)` 每帧从当前 `scrollLeft` 指数逼近目标（factor 0.18 / snap 0.5），消除离散颗粒。缓动核心抽成纯函数 `ReaderPaginationScripts.smoothScrollStep(current, target, factor, snap)`（单调收敛、不超调、收尾吸附）。rAF 每帧只写 `scrollLeft`、不读 scrollWidth，边界判定仍只在 wheel 事件读一次，避免每帧 reflow 风暴。不破坏：TODO-627 跨章边界判定 `continuousWheelBoundaryDirection`（:2545-2557）原样保留；横排放行原生滚动不变；`_onReaderScrollEvent`（reader_pagination_scripts.dart 的 rAF+200ms debounce 进度回传）不动；drag-to-pan `_hoshiReaderMouseDragScrollBy` 的 1:1 跟手 `behavior:'auto'` 不动。提交 `3382f72bd`，file:line `reader_hibiki_page.dart:2496-2514`（rAF 缓动状态/步进）+ `:2558-2578`（wheel 投影累积）+ `reader_pagination_scripts.dart` `smoothScrollStep`。
