## BUG-215 · 移动端进度条触摸热区太小难命中
- **报告**：2026-06-12（用户：）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/pages/implementations/video_hibiki_page.dart` `_mobileControlsTheme`：从未给 `MaterialVideoControlsThemeData` 传 `seekBarContainerHeight` / `seekBarThumbSize` / `seekBarHeight`，全部回退 media_kit 默认（`media_kit_video-2.0.1` 的 `lib/media_kit_video_controls/src/controls/material.dart`：`seekBarContainerHeight=36` / `seekBarThumbSize=12.8` / `seekBarHeight=2.4`）。手机触摸下这套默认细轨道 + 小滑块 + 浅触摸热区难以命中，进度条「滑不到 / 拖不动」。
- **[x] ① 已修复** — 新增三组基线常量 + getter（随 `_videoUiScale` 缩放）：`_videoSeekBarContainerHeightBase=52`→`_videoSeekBarContainerHeight`（触摸热区，>默认36）、`_videoSeekBarThumbSizeBase=18`→`_videoSeekBarThumbSize`（滑块，>默认12.8）、`_videoSeekBarTrackHeightBase=5`→`_videoSeekBarTrackHeight`（轨道，>默认2.4），并把这三个 getter **接进** `_mobileControlsTheme` 的 `seekBarContainerHeight` / `seekBarThumbSize` / `seekBarHeight` 字段。注意：前置提交只定义了 getter 但**漏接进主题**（死代码、热区实际没放大），本次补接才让放大真正生效。热区向上长（靠 BUG-214 把进度条抬到按钮条上方腾出竖直空间），不向下侵入系统边缘手势区。提交见 git log（TODO-157）。
- **[x] ② 已加自动化测试** — `hibiki/test/pages/video_mobile_controls_margin_test.dart`：新增用例①断言三字段确实接进 `_mobileControlsTheme`（`seekBarContainerHeight: _videoSeekBarContainerHeight` 等，专门防「定义了 getter 却没接」的死代码回归）；②断言三个基线常量分别 >36 / >12.8 / >2.4（抬高于 media_kit 默认）；③断言四个尺寸 getter 都乘 `_videoUiScale`（随界面缩放）。
- **备注**：media_kit 跑不了 headless、触摸命中依赖真实控制条渲染，用源码守卫钉住参数；实际「好不好滑」由用户真机确认（needsDevice）。
