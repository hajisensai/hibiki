## BUG-468 · Windows 阅读器右键同时弹原生与自定义两个菜单
- **报告**：2026-06-30（用户：）
- **真实性**：✅ 真 bug — 根因 `hibiki/lib/src/pages/implementations/reader_hibiki/webview.part.dart:1160`（`initialSettings: InAppWebViewSettings(...)` 缺 `disableContextMenu`）。桌面端（Windows，flutter_inappwebview_windows fork 渲染 EPUB）阅读器右键时同时弹两个菜单：左侧是 Hibiki 自定义的 Flutter 选区菜单（`_showReaderTextContextMenu`，经 `GestureDetector.onSecondaryTapDown` 触发，见 `webview.part.dart:1581`），右侧是 WebView2/Edge 原生菜单（复制/打印/更多工具）。代码里虽然给 `contextMenu: ContextMenu(settings: ContextMenuSettings(hideDefaultSystemContextMenuItems: true), menuItems: const [])`，但 `hideDefaultSystemContextMenuItems` 是跨平台 ContextMenu API，在 WebView2 fork 上**并不接到原生菜单开关**；fork 里压制 WebView2 原生菜单的唯一真值是 `InAppWebViewSettings.disableContextMenu`→`put_AreDefaultContextMenusEnabled(!disableContextMenu)`（`packages/flutter_inappwebview_windows/windows/in_app_webview/in_app_webview.cpp:231`、`:1594`）。reader 的 `initialSettings` 从未设 `disableContextMenu`（默认 false）→ 原生菜单一直开着，与 Flutter 菜单重叠双弹。
- **[x] ① 已修复** — `webview.part.dart` 的 `initialSettings: InAppWebViewSettings(...)` 加 `disableContextMenu: isWindowsPlatform`：Windows 关掉 WebView2 原生菜单只留 Flutter 选区菜单；移动端值为 false 不动原生 `ContextMenu`（查词+导出），不回归。提交哈希：见 integration owner 落地。
- **[x] ② 已加自动化测试** — 源码守卫 `hibiki/test/pages/reader_webview_disable_context_menu_guard_test.dart`：断言 `_buildWebView()` 的 `InAppWebViewSettings` 块里 `disableContextMenu: isWindowsPlatform` 在位、且**不是**无条件 `true`（防移动端丢菜单）。
- **备注**：根因是「跨平台 ContextMenu API 的 hide 标志在 WebView2 fork 上不等于原生菜单开关」这一映射缺口；既有守卫 `reader_text_context_menu_scale_guard_test.dart` 只校验了 `hideDefaultSystemContextMenuItems: true`（无效标志）故没拦住本 bug。真机待验：Windows 阅读器右键应只出 Flutter 菜单。
