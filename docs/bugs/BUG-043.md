## BUG-043 · 阅读器内按 Esc / 手柄 B 不退出书籍，反而切换底栏（Esc 被「切底栏」抢占）
- **报告**：2026-06-04（用户：「win 按 esc 为什么不是退出书而是翻页」→ 追问澄清「当前有底栏，按 esc 应退出到上一级，凭什么隐藏底栏」「焦点在底栏和在阅读正文按 esc 都应退出，手柄 B 也是退出，而不是关底栏」）。经多轮澄清确认目标 = Esc / 手柄 B 在任何焦点都「向上一级」（有词典弹窗关弹窗，否则退出书），永不切底栏。采番注：动手前查 `docs/BUGS.md` 当前最大号为 BUG-042，故本条取 043。
- **真实性**：✅ **真 bug（键盘键静默双绑 + 枚举顺序抢占），可复现**。沿真实代码路径定位：桌面默认绑定（`shortcut_defaults.dart` `_desktop`）把 **Esc 同时绑给 `readerToggleChrome`（切底栏）和 `readerDismissDict`（关词典）**；`resolveKeyboard` 按 `ShortcutAction.actionsForScope` 的**枚举声明顺序**返回第一个命中者（`shortcut_registry.dart:92-99`），而枚举里 `readerToggleChrome` 排在 `readerDismissDict` 前（`shortcut_action.dart:30-31`）→ Esc 永远解析成 `readerToggleChrome`，`_executeShortcutAction` 调 `_toggleChrome`（`reader_hibiki_page.dart:3702-3708`）翻转 `_showChrome` 并 `_applyChromeInsets` 改 WebView inset → 重新分页/重锚，正文位移被误认成「翻页」；`return handled` 也使其**永不冒泡到 `Navigator.pop`**，故退不出书。退出之所以「看起来时有时无」取决于底栏是怎么唤出的：键盘/手柄唤出（`moveFocusToChrome:true`）焦点进底栏 → 此时 Esc 走 `_chromeFocusScope` 分支 `maybePop` 退出（`3469-3472`）；鼠标点击唤出（`1781`/`1807` 不移焦点）焦点留正文 → Esc 落「切底栏」。手柄 B 绑 `readerDismissDict`（`_gB`），无弹窗时 `_executeShortcutAction` 返回 ignored→`_handleGamepadButton` 返回 false→GamepadService 的 global-back 兜底退出，故 **B 本就退出**，跑偏的只有键盘 Esc。`globalBack` 默认是 Alt+←（`shortcut_defaults.dart:103-105`），从来不是 Esc。根因点 = `shortcut_defaults.dart` 把 Esc 双绑 `readerToggleChrome`+`readerDismissDict`，枚举顺序让「切底栏」赢、「返回」永远轮不到。
- **[x] ① 已修复** — `4eec6eb19`：根因修——拆掉双绑，让 Esc 成为 reader 唯一「返回」键（消除「同一键被两动作占用、靠枚举顺序隐式裁决」这个特殊情况，非补丁）。① `shortcut_defaults.dart`：`readerToggleChrome` 键盘键 Esc→`KeyM`（手柄仍 Y），`readerDismissDict` 单独保留 Esc（手柄仍 B）。② `reader_hibiki_page.dart` `_executeShortcutAction` 的 `readerDismissDict`：无词典弹窗时不再 `return ignored`，改 `Navigator.of(context).maybePop()` 退出（与同文件 `_chromeFocusScope`/popup 分支的 B/Esc `maybePop` 对齐）。效果：键盘 Esc（正文/底栏焦点皆然）、手柄 B 统一「有弹窗关弹窗，否则退出书」，绝不切底栏；底栏开关交给 M/Y/点击。`_macOS`/`_mobile` 默认均派生自 `_desktop`，一处改全平台一致。`flutter analyze` 0 issue、`dart format` 0 改动。
- **[x] ② 已加自动化测试** — `4eec6eb19`：① 回归 `test/shortcuts/shortcut_registry_test.dart`——把原本**锁死 buggy 行为**的用例（「Escape returns first matching action by enum order」断言 `readerToggleChrome`，正是这个 bug 的化石）改为断言 Esc 解析成 `readerDismissDict`；新增 `KeyM→readerToggleChrome` 与「Escape 在 reader 组只被一个动作拥有（无双绑）」用例。② 通用守卫 `test/shortcuts/shortcut_defaults_test.dart`——新增「reader 组无键盘键被多动作占用」（镜像既有手柄版，按完整 InputBinding key+modifiers 去重，Space vs Shift+Space 不误伤），以及「底栏切换在 M/Y、Esc 是返回键不是切底栏」断言；任何人把 Esc 再绑回 toggle 即红。③ 修正 `shortcut_binding_edit_dialog_test.dart` 冲突用例的期望动作（Esc 冲突现指向 `readerDismissDict`）。`flutter test test/shortcuts/` 全量 **158 绿**。
- **备注**：reader/快捷键/焦点路由类。代码 + 单测绿、`flutter analyze` 0 issue、`dart format` 0 改动。全量 `flutter test` 另有 2 条**非本改动**预存失败（`audio_lifecycle_flush_wiring_static_test`、`dictionary_dialog_layout_static_test`=BUG-042 并发 agent 半成品），与快捷键无关。**真机肉眼复测原始失败路径待用户**（Windows/桌面）：进书 → 点击唤出底栏 → 按 Esc 应直接退出书籍（不再只是隐藏底栏/正文位移）；按 M（或手柄 Y）才切底栏；有词典弹窗时 Esc/B 先关弹窗。**小遗留**：`readerDismissDict` 的 i18n 标签仍是「关闭词典」，现在也兼任「退出书」，设置页/冲突提示文案略不全（改需 `i18n_sync` 跨 17 文件，未做）。
