## BUG-423 · 调试日志框选拖拽未响应卡死
- **报告**：2026-06-25（用户：「我在框选调试日志的时候未响应卡死了」）
- **真实性**：✅ 真 bug。同族 TODO-762/BUG-119/BUG-413/TODO-806 延续。根因 `hibiki/lib/src/utils/components/hibiki_material_components.dart`（`_HibikiLogPanelState`/`logSelectionScrollDecision` 边缘自动滚动放行分支，旧 :2366-2369）。
  - 链路：日志正文（调试/错误日志页共用 `HibikiLogPanel`）TODO-762 起改为 `SelectionArea` + `ListView.builder` 懒构造行。拖拽框选贴边时，旧判据放行「边缘自动滚动」→ `ListView` 持续构造视口外新行 → `SelectionArea` 注册的 `Selectable` 集合随拖拽时长**单调膨胀** → 每次 `onPointerMove`（:2282）对全部已注册 `Selectable` 做命中/边界重算占满 UI 线程 → 未响应卡死。
  - 放大器：日志行 `Text(softWrap: true)`（旧 :2305）把一行拆成多视觉行，`SelectionArea` 单行选区命中要对每段 wrap 子矩形逐一求交，命中成本随行长放大（也是 TODO-806 框选坐标错位来源）。
- **[x] ① 已修复** — 根因修复：拖拽选区激活且有实质位移时一律拦掉程序化滚动，**删掉「边缘自动滚动」放行分支**，`Selectable` 集合钉在视口内有限行，膨胀链路从根上断掉；判据 `logSelectionScrollDecision` 简化为仅 `pointerSelectionActive`/`delta` 两参（消除按指针位置/方向的特殊情况）。`_LogSelectionScrollController` 同步删除不再使用的指针几何 / 手动滚动状态（`_pointerY`/`_viewportHeight`/`_userScrolledDuringSelection`/`_markUserScroll`），`_LogSelectionScrollPosition` 删 `applyUserOffset`/`pointerScroll` override（手动滚动本就不该拦，走默认）。日志行 `softWrap: true → false`（降低单行命中成本，兼修 TODO-806 坐标错位）。提交 673084dbf。
  - **BUG-119 行为契约取舍**：拖拽期边缘自动滚动被禁用。Never break userspace 的边界——滚到视口外去框选本就不可靠（`SelectionArea` 不为滚出视口的行保留选区），「复制整段日志去排障」这一真实用途已由常驻「复制全部」按钮（绕开 `SelectionArea` 走 `widget.log` 全量，:2217/:2314）完整覆盖，禁用边缘自动滚动不损失任何可用能力。BUG-119「拖拽选区不被程序化 jumpTo/animateTo 拽回」的不变式仍由同一 gate 守住（拖拽期一律拦，是旧拦截规则的超集）。手动滚动（拖滚动条 / 滚轮）不经判据，不受影响。
- **[x] ② 已加自动化测试** — `hibiki/test/widgets/log_panel_scroll_select_guard_test.dart`：①纯函数真值表 `logSelectionScrollDecision`（拖拽未激活→放行；位移可忽略→放行；拖拽激活 + 实质位移上/下/小位移→一律 BLOCK，钉死「边缘自动滚动放行分支不复活」）；②源码守卫追加 `softWrap: false` 在场、`softWrap: true` 不在场断言。提交 673084dbf。
- **备注**：需真机/真渲染复测原始失败路径——在真实运行的 app「调试日志」页对大日志按住鼠标拖拽框选（含拖到顶/底边缘）不再卡死、UI 持续响应；「复制全部」仍能拿整段日志。headless 难以稳定复现真实选区+滚动几何，故在最强可落地层（纯函数真值表 + 源码守卫 + widget 结构）守住不变式。
