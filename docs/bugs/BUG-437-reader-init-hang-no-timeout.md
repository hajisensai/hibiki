## BUG-437 · 打开书籍偶发永久卡加载不恢复
- **报告**：2026-06-27（用户：）
- **真实性**：✅ 真 bug —— 根因 `hibiki/lib/src/pages/implementations/reader_hibiki_page.dart:1058 _initBook`。整函数无 top-level try/catch，其内多处 DB await（`_resolveProfileAndSettings` / `_locateBookOnDisk→db.getEpubBook` / `_resolveAudioSlot` / `repo.findByBookKey`）任一抛异常即逃逸出 async 链 → `_book` / `_audioSlotResolved` 永不置好 → 尾部 `setState`（约 :1188）永不执行 → `_buildBody`（约 :1732）永远返回 spinner → WebView 从不构造 → 唯一兜底超时 `_startContentReadyTimeout`（navigation.part.dart:21，只在 onWebViewCreated 启动）从不触发 → 永久卡加载、无恢复。双实例放大器：Windows runner（windows/runner/main.cpp:12 附近）只 CreateMutexW 不检查返回值=无真单实例，两进程共享 WAL DB（busy_timeout=5000，packages/hibiki_core/.../database.dart:47），第二实例持写锁超 5s → 另一实例 DB 调用抛 SQLITE_BUSY → 正中无 try/catch 死路（解释「偶发 + 同开两个就会」）。
