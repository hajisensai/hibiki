## BUG-085 · Hibiki 互联服务端：切出「同步与备份」界面就把服务端关掉了
- **报告**：2026-06-06（用户：「为什么 hibiki 互联服务端切出同步与备份界面会把服务端关掉」）。采番注：本轮 077 已占，取 078。
- **真实性**：✅ **真 bug（生命周期归属错误，沿真实代码路径定位）**。`HibikiSyncServer` 与 LAN 广播 `LanBroadcastService` 由**设置页内的 widget `_ServerModeWidgetState` 持有**（`sync_settings_schema.dart` 旧 `_server`/`_broadcast` 字段），其 `dispose()` 调 `_server?.stop()` + `_broadcast?.stop()`——所以一离开「同步与备份」页，widget 销毁就把服务端连同广播一起停了。`lan_discovery_service.dart:174` 注释本就承认「server 生命周期归设置 UI 管（a later task）」是个未完成的临时归属。根因：服务端实例的**所有者应是 app 级长生命周期对象，而非随路由进出销毁的设置页 widget**。
- **[x] ① 已修复** — 本提交（自含修复+测试+本条目）。**根因修（所有权上移，非补丁）**：新增 `lib/src/sync/hibiki_server_controller.dart` 的 `HibikiSyncServerController extends ChangeNotifier`，封装 `HibikiSyncServer` + `LanBroadcastService` + 配对单飞 + 启停/重启逻辑 + 配对批准弹窗（经全局 `navigatorKey`，故在任意界面都能弹）。由 `AppModel` 持有（`late final syncServerController`，镜像既有 `syncConflictPrompter` 范式），`initialise()` 末尾 `unawaited(syncServerController.startIfEnabled())` 在启用了 host 时**随 app 启动常驻**（bind 失败自禁用 + 记日志，绝不阻断 init）。`_ServerModeWidgetState` 退化为薄视图：`dispose()` **不再停服务端**（只摘监听 + dispose 控件），开关/端口/令牌/运行态全部委托/读取 `appModel.syncServerController`（`start()` 返回 `HibikiServerStartOutcome` sealed：started / portInUse(port) / error(msg)，UI 据此弹 snackbar）。`stop({persistDisabled})` 仅在用户显式关开关时落 `serverEnabled=false`，app 退出/瞬时停不动该标志（下次启动恢复 host）。
- **[x] ② 已加自动化测试** — `hibiki/test/sync/server_lifecycle_appmodel_guard_test.dart`：① 行为测试——禁用 host 时 `startIfEnabled()` 在构造服务端**之前**短路（用会抛异常的 lookup 工厂证明从不 bind，`isRunning` 恒 false，确定性无 socket/UDP）；② 源码守卫——设置页 schema 不再含 `HibikiSyncServer? _server` / `_server?.stop()` / `_broadcast?.stop()` 且改调 `appModel.syncServerController`；③ AppModel 持有 `syncServerController` 且 init 调 `startIfEnabled()`；④ 控制器 `stop` 默认不清 enabled 标志。真实 bind/广播/配对依赖 socket+UDP 多播，host 测试不可靠，故走源码守卫 + 禁用路径行为测试（同平台门控范式）。`test/sync/` + `test/settings/` + `test/models/` 全量 642 绿，analyze 0。
