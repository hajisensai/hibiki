## BUG-119 · 日志页（错误日志/调试日志）按住鼠标选区想上滑复制时视口被拽回

- **报告**：2026-06-08（用户：「鼠标按住错误日志里面的页面的时候，想要滚动上滑复制，会被拽回来」）。
- **真实性**：✅ **真 bug（沿真实代码路径定位）**。错误日志页 `error_log_page.dart:77` / 调试日志页 `debug_log_page.dart` 的正文都用共享组件 `HibikiLogPanel`（`hibiki/lib/src/utils/components/hibiki_material_components.dart:1940`）。旧实现把整段 log 渲染成一个**非滚动**的 `SelectableText`（`maxLines` 默认 null = 全高），外层再套 `SingleChildScrollView` 提供滚动。鼠标拖拽选区时，`SelectableText` 内部的 `EditableText` 会对祖先 `Scrollable`（这个 `SingleChildScrollView`）调 `bringIntoView`，把视口强行拉回选区 extent/caret 位置 → 用户按住选区往上拖/滚轮上滑想选屏幕外文本时，视口被一次次「拽回」到选区光标处。根因 = 嵌套滚动器冲突（外层 `SingleChildScrollView` ⇔ 内层 `EditableText` 选区自动追随），根因 `file:line`：`hibiki/lib/src/utils/components/hibiki_material_components.dart:1959`（`SingleChildScrollView` + `SelectableText` 旧结构）。
- **[x] ① 已修复** — 本提交。根因修（消除嵌套滚动器，非补丁）：把 `HibikiLogPanel` 从 `StatelessWidget` 改为 `StatefulWidget`，正文改用**只读 `TextField`**（`readOnly: true` / `maxLines: null` / `expands: true`），让 `EditableText` 自己当**唯一**滚动器——选区拖拽的边缘自动滚动走它自己的 scroll controller，没有外层 `SingleChildScrollView` 来抢/拽，结构上消除冲突。镜像同文件已有的 `HibikiEditorPanel`（只读差异）。自定义 `selectionControls`（复制/分享/全选，禁剪切/粘贴）完整保留，未破坏选区工具栏与分享。`State` 持有并 `dispose` 的 `TextEditingController`，`didUpdateWidget` 在 log 变更时同步文本。
- **[x] ② 已加自动化测试** — `hibiki/test/widgets/log_panel_scroll_select_guard_test.dart`：①widget 行为——面板渲染一个 `readOnly` 的 `TextField`（controller 文本 == log、selectionControls 为 `HibikiTextSelectionControls`），且 `HibikiLogPanel` 子树里**不再有** `SelectableText` / 包内容的 `SingleChildScrollView`；②源码守卫——面板源码含 `readOnly: true` 且不再出现 `SelectableText(` / `SingleChildScrollView(` 构造调用，防止改回旧结构。真实的「选区拖拽 + 滚动几何」需设备/真 `EditableText` 拖拽，headless 难稳定复现，故守在最强可落地的结构不变式层。
- **备注**：布局/交互类，桌面（鼠标）为主。`flutter analyze` 改动文件 0 issue；`flutter test` 守卫 + 相邻（`hibiki_material_components_test` / `log_pages_static_test` / `log_pages_export_test` / `md3_design_system_static_test`）共 84 绿无回归。**桌面真机肉眼复测待用户**：打开错误日志/调试日志页 → 按住鼠标向上拖选屏幕外文本、或选区激活时滚轮上滑 → 视口不再被拽回，可正常选取并复制。
