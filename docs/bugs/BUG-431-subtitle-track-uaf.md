## BUG-431 · selectSubtitleTrack libmpv UAF (回退/关字幕闪退)
- **报告**：2026-06-26（用户：）
- **真实性**：✅ 真 bug — 根因 `hibiki/lib/src/media/video/video_player_controller.dart:480-485`（修前）。`selectSubtitleTrack` 是切/关字幕路径上唯一一处 `await` libmpv 原生下发（`setSubtitleTrack`）却**不带 loadToken 守卫**的方法。await 期间用户退出页面/换集触发 `dispose()`（:2070 `_loadToken++` → :2089 `unawaited(_player?.dispose())` 异步释放底层 libmpv NativePlayer → :2090 `_player = null`），await 完成后旧引用向已释放/被换的 native handle 下发 = 原生 use-after-free（访问违例，AMD 更敏感）。关字幕（`SubtitleTrack.no()`，经 `_selectSubtitleOff`）是高频路径，故症状表现为「回退字幕闪退」。兄弟方法 `selectEmbeddedGraphicTrack`（:495-534）每个 await 后都用 `_isCurrentLoad(player, loadToken)` 双判据（`_player == player && _loadToken == loadToken`，:925-926）重校验，本方法漏了。
- **[x] ① 已修复** — `selectSubtitleTrack` 镜像 `selectEmbeddedGraphicTrack`：①同步图形标志复位（`_graphicSubtitleActive=false`）保持在 player 空判/await **之前**无条件执行（纯 Dart 状态，未 load 时关字幕仍需复位，防回归 BUG-301）②`_player` 提为局部 `player` + 空判早退 ③await **前**捕获 `final int loadToken = _loadToken;` ④`await player.setSubtitleTrack(track)` 后 `if (!_isCurrentLoad(player, loadToken)) return;` 过期立即放弃下发，不触野 handle。只改这一个方法，不动成功路径语义/调用方签名。提交：见分支 todo-409-subtitle-uaf。
- **[x] ② 已加自动化测试** — `hibiki/test/media/video/video_lifecycle_static_test.dart` 新增 `selectSubtitleTrack` 源码扫描守卫两条：①await 前捕获 loadToken、await 后重校验 `_isCurrentLoad(player, loadToken)`、过期 `return` ②`_player` 提局部 + 禁止回退到无守卫的 `await _player?.setSubtitleTrack`。钉死「每个 libmpv 异步下发都带 loadToken 守卫」不变量。libmpv UAF 本身无法纯单测（需真实 player），与既有 `selectEmbeddedGraphicTrack` / `load()` 守卫同范式。回归保护：`test/media/video/video_player_controller_test.dart` BUG-301「关字幕复位图形标志」用例验证未 load 时标志仍复位。
- **备注**：TODO-409 修2（只做 Dart UAF 守卫，不碰修1 原生 C++ 层）。真机门禁：桌面「开视频→选字幕→关字幕/回退反复 + 退页面」无闪退，可与修1 AMD 门合并一轮验证。
