  - **窗口模式音量 HUD（桌面已有）**：HUD overlay `_buildLevelHudOverlay()` 由共享 controls builder 挂载（见下）**无平台门控**；所有改音量入口（滑条 / 桌面滚轮 `_onVolumeWheel`→`_adjustVolume`→`_applyUserVideoVolume`、键盘音量键绑定、静音、移动竖滑）统一走 `_applyUserVideoVolume`，其 `if (mounted) _showVolumeOsd(clamped);` 同样无平台门控 → 桌面窗口调音量（滚轮/快捷键）本就弹右侧音量百分比 HUD。
  - **全屏模式音量/亮度 HUD（同样已有，原判定错误）**：全屏 `Video` 设 `controls: params.controls`（`hibiki/lib/src/pages/implementations/video_hibiki_page.dart:3525`），与窗口侧共享同一 controls builder `_buildVideoControls`（窗口侧赋值 `:7497` `controls: (state) => _buildVideoControls(state, controller)`）。fork 的 media_kit `Video.build` 在 `controls != null` 时无条件挂 controls（`third_party/media_kit_video/lib/src/video/video_texture.dart:461`，无窗口/全屏门控）。`_buildVideoControls` 返回 `VideoControlsFocusGate(child: _buildVideoControlsInner(...))`（`:7497`）；`_buildVideoControlsInner` **无门控**挂载 `_buildOsdOverlay()` / `_buildLevelHudOverlay()`（`:7699-7700`）。`VideoControlsFocusGate.build`（`hibiki/lib/src/media/video/video_controls_focus_gate.dart:47-54`）**只在窗口侧**卸载 controls（`fullscreenRouteActive && !inFullscreenRoute` 才返回 `SizedBox.shrink()`），全屏侧 `inFullscreenRoute == true` → 返回 child → 全屏 controls 照常渲染那对 overlay。**结论：全屏路由的 controls 子树本就渲染 OSD + Level HUD，全屏 HUD 已经存在。**
  - **亮度桌面诚实不可用（保留现状）**：`ScreenBrightnessController.canControl => isMobilePlatform`（`hibiki/lib/src/platform/screen_brightness_controller.dart:29`），桌面无统一窗口级背光 API，亮度手势/回调/快照三处都被 `canControl` 门控；桌面只调音量、不调亮度，HUD 亮度侧仅移动出现，属平台限制不强行造假。
