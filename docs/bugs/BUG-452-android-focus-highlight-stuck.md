## BUG-452 · Android 焦点高亮手柄/滑动消不掉
- **报告**：2026-06-28（用户：）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/shortcuts/gamepad_service.dart`（原 `start()` 第 271-272 行 `if (!isSupportedPlatform) return;` 在 Android 早退）。Android 上 `FocusManager.highlightStrategy` 恒为 Flutter 默认 `automatic`，无人接管：手柄/D-pad 经引擎当 `gameButton*` key event 投递把 automatic 推成 traditional 点亮 Material 焦点环；触摸点击（PointerDown）能让 automatic 翻回 touch（用户证实点击能消），但滑动（PointerMove）automatic 不复位 → 滑动消不掉；手柄全是 key event 永不翻回 touch → 手柄消不掉；首页/视频是普通 Material 控件故常驻，阅读器正文落 WebView 非可高亮控件故无框。`start()` 早退导致 Android 既不装 pointer 全局路由也不 seed 显式 strategy（BUG-398 的 `_onKey` 收紧 / `resetHighlightForScreenSwitch` / `HighlightResetNavigatorObserver` 全走该 service，Android no-op 故从未覆盖）。
- **[x] ① 已修复** — `gamepad_service.dart` 拆分 `start()`：新增平台无关 `_installHighlightTracking()`（seed `alwaysTouch` + 装 `_onPointerGlobal` 全局 pointer 路由 + 装 `_onKey`），在所有平台（含 Android）安装；`isSupportedPlatform` 改名 `needsGamepadPoller` 只 gate `gamepads` 轮询（拆出 `_startGamepadPoller()`，Android 不调）。`_onPointerGlobal` 早已含 `PointerMoveEvent`→`alwaysTouch`，Android 接上同一路由后滑动即复位。桌面/Apple 行为不变（仍先装 tracking 再起轮询，与原顺序等价）。提交哈希：见提交。
- **[x] ② 已加自动化测试** — `hibiki/test/shortcuts/android_focus_highlight_reset_test.dart`：①源码契约守卫（`start()` 必在 poller gate 前调 `_installHighlightTracking`、pointer 路由/key handler/strategy seed 在平台无关 installer 内、`_onPointerGlobal` 含 `PointerMoveEvent`→`alwaysTouch`）②行为测试（装好 pointer 路由后：方向键→`alwaysTraditional`，触摸滑动 PointerMove→`alwaysTouch`，即 Android「滑动消不掉」的直接复现解）。red 证据：对 `origin/develop` 原始源码跑源码契约组失败（`needsGamepadPoller`/`_installHighlightTracking` 缺失），实现后转绿。既有 `focus_highlight_residue_test.dart`（桌面零回归）仍绿。
- **备注**：Android 真机未验（手柄点亮焦点框→触摸点击/滑动复位、首页/视频不再常驻；桌面行为不变）。
