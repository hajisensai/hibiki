## BUG-396 · 默认(system)主题下阅读器sasayaki/选区/链接色不吃强调色(落硬编码默认)
- **报告**：2026-06-22（用户：修了 sasayaki 高亮后追问「默认主题(桌面强调色生成的)高亮吃主题吗」+「还有没有别的不吃主题的」）
- **真实性**：✅ 真 bug。默认主题是 `system-theme`（从 Windows 桌面强调色 `getAccentColor`→`fromSeed` 生成 ColorScheme），但阅读器正文的三个角色色（跟读高亮 sasayaki / 选区高亮 selection / 链接 link）在 system/light 主题下落硬编码默认，不随强调色变。
- **根因**：reader 的高亮/选区/链接色有两套源且对 system-theme 不一致。
  - `ReaderContentStyles.css`（`hibiki/lib/src/reader/reader_content_styles.dart`）的 `_themeColors` default 分支只覆盖 `bg`/`fg`（吃 ColorScheme），**不设** selection/sasayaki/link → 落 `_ThemeColors` 构造默认：sasayaki=`rgba(135,206,235,.40)`(天蓝)/selection=`rgba(160,160,160,.40)`(灰)/link=`#426cf5`(蓝)。
  - 页面 `reader_hibiki_page.dart:_computeStyleTag`/`_applyStylesLive`（旧）只在 `_isCustomTheme` 时传这三色，system-theme 传 null → 用上面硬编码默认。
  - 而 `_readerThemeColors`（经纯函数 `resolveReaderThemeColors`，reader_hibiki_page.dart:199）对 system-theme **已**派生 `sasayaki: scheme.primary.withValues(alpha)`（强调色）——只是页面没把它用到 `::highlight`/选区/链接的 CSS 上。两源对 system-theme 打架（`.hoshi-active` 走 `_readerThemeColors.sasayaki` 吃强调色，`::highlight(hoshi-sasayaki)` 走硬编码天蓝）。
- **[x] ① 已修复** — 把 `ReaderThemeColors` record 扩成五角色（+`selection`+`link`），`resolveReaderThemeColors` 作单一真相源：custom→用户色、preset→`_themeMap` 透传（值与 css switch 逐一相等，零变化）、system/light→从 `scheme` 派生（sasayaki=primary、selection=tertiary 与跟读错开色相、link=primary）。页面 `_computeStyleTag`/`_applyStylesLive` 统一用 `_readerThemeColors.{selection,sasayaki,link}`，删掉 `_isCustomTheme?:null` 旧分叉（`_isCustomTheme` getter 随之删除）。文件：`reader_hibiki_page.dart`(typedef+resolver+两 css 调用)、`reader_hibiki/chrome.part.dart`(_themeMap+_customReaderThemeColors)。提交：本轮 BUG-396（code-review 待跑）
- **[x] ② 已加自动化测试** — `hibiki/test/pages/reader_theme_background_follows_theme_test.dart` 扩展：新增「BUG-396 高亮/选区/链接色跟随主题」组（system-theme 三色=scheme.primary/tertiary 派生且≠旧硬编码默认；暗色 alpha 档；preset 透传零变化）；record 字段新增同步更新该文件全部 `ReaderThemeColors` 字面量。`reader_lyrics_progress_bottom_reserve_static_test.dart` 的 `_buildBody` 切片结束标记由已删的 `_isCustomTheme` 改为 `_buildStyleTag`。
- **备注**：preset 主题（ecru/water/gray/dark/black）的 selection/sasayaki/link 仍是手调专色，零变化（`_themeMap` 新增值与 `_themeColors` switch ARGB 逐一相等）。preset link 由 css hex 串改走 rgba 串（同色不同串，渲染一致）。真机视觉待用户在桌面强调色主题下复看；headless 无真 WebView 看不到上色。
