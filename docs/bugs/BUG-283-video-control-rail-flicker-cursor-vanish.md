## BUG-283 · 视频右侧控制按钮闪烁 + 鼠标放字幕上光标消失 (TODO-373)
- **报告**：2026-06-15（用户：①鼠标放到视频右边的几个按钮上，按钮会闪烁；②鼠标放到字幕上，鼠标会消失。要求根本修复）
- **真实性**：✅ 真 bug（两症状同一根本原因）。`hibiki` 把 opaque 的 chrome 叠在 media_kit 桌面控制条那个**全画面** hover-tracking `MouseRegion` 之上，而该 MouseRegion 的 `onExit` 会**立即**把控制条 `visible` 置 false（媒体库 `media_kit_video-2.0.1/.../material_desktop.dart:509-515` 的 `onExit`；`onEnter` 才置 true + 2s timer；其 cursor=`hideMouseOnControlsRemoval && !mount ? none : basic`，line 658-665）。
  - **症状①（右侧按钮闪烁）根因**：`hibiki/lib/src/pages/implementations/video_hibiki_page.dart` 的 `_buildVideoSideActionRail`（右/左浮动学习按钮 rail，默认 `subtitleList` 等落 `screenRight`，见 `video_control_customization.dart:357`）由 `_videoControlsVisible` **整体显隐**（`SizedBox.shrink`），rail 按钮是 opaque `IconButton`。鼠标 hover 进按钮 → MouseTracker 把最顶命中切到按钮 → media_kit `onExit` 立即 `visible=false` → `_videoControlsVisible=false` → rail 消失 → 鼠标位置下方重新是 media_kit region → `onEnter` `visible=true` → rail 重现 → 鼠标又落按钮上 → 每帧级别快速闪烁。
  - **症状②（鼠标放字幕上消失）根因**：字幕盒（底部居中，覆盖在控制条之上）。鼠标停字幕上读字/查词时，media_kit 控制条 2s 自动隐藏 → `mount=false` → media_kit 自身 `cursor: none`（`hideMouseOnControlsRemoval`），叠加 hibiki 顶层 `_cursorHidden` 胜出层（BUG-258），光标在字幕上消失。
  - **与 ef7ee819b（BUG-198/199）关系**：那次只处理了「字幕逐字符 opaque 吞 hover」，把字幕改 translucent；本 bug 是**另一对** opaque chrome（右侧 rail）+ 「字幕区静止时光标被自动隐藏」，ef7ee819b 未覆盖。非回退、保留其 translucent 纪律。
- **[x] ① 已修复** — 根因修复（消除特殊情况，非去抖/延迟掩盖）：
  - 症状①：新增 `_railHovered` ValueNotifier 单一真相源；rail 显隐判据从 `!controlsVisible` 收紧为 `(!controlsVisible && !railHovered)`（hover 期间永不被 media_kit 瞬时 visible 抖动收走 → 振荡根除），rail 按钮列外层挂 `_railHoverKeepAlive`（`MouseRegion(opaque:false)`：进 rail 置 `_railHovered=true` + `_pokeControlsVisible()` 续命 media_kit 控制条＝其自身设计的续命路径；出 rail 置 false 回落到 `_videoControlsVisible`）。`opaque:false` 不阻断按钮点击/画面下探（沿 BUG-198 纪律）。
  - 症状②：`VideoSubtitleOverlay` 新增 `onHoverChanged(bool)` 回调，仅在字幕盒（非整片画面）hover 进出时触发（与 blur 显形共用一个 `MouseRegion(opaque:false)`，未注册且非 blur 时透传 box、外观零变化）；页面 `_handleSubtitleHover(true)` → `_setCursorHidden(false)` 唤回光标 + `_pokeControlsVisible()` 续命控制条。
  - 文件：`hibiki/lib/src/pages/implementations/video_hibiki_page.dart`（`_railHovered` 字段/dispose、`_buildVideoSideActionRail` 判据、`_railHoverKeepAlive`、`_buildVideoSideRailFor` 包裹、`_handleSubtitleHover`、字幕 overlay 接 `onHoverChanged`）、`hibiki/lib/src/media/video/video_subtitle_overlay.dart`（`onHoverChanged` 参数 + 合并 hover MouseRegion）。提交哈希见末行。
- **[x] ② 已加自动化测试** — `hibiki/test/pages/video_rail_hover_flicker_guard_test.dart`（源码守卫：`_railHovered` 单一真相源 + dispose；rail 判据含 `(!controlsVisible && !railHovered)` 且监听两 notifier；`_railHoverKeepAlive` opaque:false + 进 rail 续命；keep-alive 真正包按钮列而非整片 fill；`_handleSubtitleHover` 唤回光标+续命且接到 overlay）；`hibiki/test/media/video/video_subtitle_overlay_test.dart` BUG-283 组（widget pointer hover：进/出字幕盒回调 true/false；注册 `onHoverChanged` 才多挂一个字幕盒 hover MouseRegion＝外观对照）。`media_kit` controls 跑不了 headless，故 rail 闪烁用源码守卫钉不变量；字幕 hover 用真 widget pointer 测试。
