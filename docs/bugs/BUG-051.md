## BUG-051 · app 外查词弹窗的「下钻子弹窗好小」+「横滑关闭子弹窗会把整张卡片一起平移」（嵌套层复用阅读器小浮卡 + 外层 Listener 横滑冒泡）
- **报告**：2026-06-05（用户截图：安卓 app 外查词弹窗，点 Pixiv 词条里的链接词下钻出「職業」子弹窗——「app 外的查词弹窗的子弹窗好小，并且滑动关闭子弹窗会把所有弹窗都滑动」）。采番注：动手前查 `docs/BUGS.md` 当前最大号为 BUG-050，故本条取 051。
- **真实性**：✅ **真 bug（两症状同源于 `PopupDictionaryPage` 嵌套层渲染，app 外查词窗口专有）**。沿真实代码路径定位（`hibiki/lib/src/pages/implementations/popup_dictionary_page.dart`）：① **子弹窗好小** —— `_buildLayer` 对 base 层（index 0）用 `Positioned.fill` 满卡渲染，但对嵌套层（index>0）复用共享 mixin 的 `buildNestedPopupLayer` → `calcPopupPosition(maxHeight:360, maxWidth:popupMaxWidth)`。这套「贴选区的小浮卡」定位是给**全屏阅读器/HomeDictionaryPage**（小卡浮在整屏内容之上）设计的；而 app 外查词窗口本身已是一张约 480 宽的约束卡片，再在里面浮一张更小的卡 → 被压成小窗（截图里「職業」子弹窗明显比外卡小一圈）。② **横滑连带整卡** —— `SwipeDismissWrapper` 基于 `Listener`（`swipe_dismiss_wrapper.dart`：`onPointerMove`/`onPointerUp` + `Transform.translate`），而 Flutter 的指针事件会派发给 hit-test 路径上的**所有祖先 Listener**（Listener 不参与手势竞技场、无法阻止祖先接收）。`_buildCard` 把**整张卡片**（搜索栏 + 嵌套栈）包了一层外层 `SwipeDismissWrapper(onDismiss:_close)`，而每个嵌套层各自又带一层 `SwipeDismissWrapper`。于是横滑嵌套层时，嵌套层自身 wrapper 与外层卡片 wrapper **同时被同一指针流驱动** → 两者一起 `Transform.translate` → 「滑动子弹窗把整张卡片一起滑」。app 内阅读器（`base_source_page.dart`）没这毛病，因为它没有一个套住整个嵌套栈的祖先 wrapper。根因点 = `popup_dictionary_page.dart` 嵌套层错用阅读器小浮卡语义 + 外层卡片横滑 wrapper 套住了带自横滑的嵌套层。
- **[x] ① 已修复** — `97ed3cf3c`：根因修，一处重构同时消两症状（Linus 式：消除特例而非打补丁）。① `_buildLayer` 统一为**所有层满卡渲染**（`Positioned.fill`）：base 层透明（露出卡片表面）、横滑交给整卡外层；嵌套层不透明（`overrideDictionaryColor ?? tokens.surfaces.page`）铺满盖住下层、自带横滑 `onDismiss:()=>_popAt(index)` 仅返回上一层。彻底不再用 `buildNestedPopupLayer`/`calcPopupPosition`（那是阅读器全屏语义），下钻 = 满卡导航栈、逐层覆盖、逐层返回。截断逻辑改按层 index：`_stack.removeRange(index + 1, _stack.length)`。② `_buildCard` 把外层 `SwipeDismissWrapper` **闸到 base 层**（`if (_stack.length > 1) return card;`）：栈深 1 时整卡横滑关闭窗口（base 层 `swipeDismissible:false`，单层平移，保留「整卡滑走」手感）；一旦下钻（栈深>1）外层 wrapper 不渲染 → 祖先 Listener 消失 → 横滑嵌套层只驱动嵌套层自身 wrapper（返回上一层），不再连带整卡。`buildNestedPopupLayer` 仍留在 mixin 供 `home_dictionary_page.dart`（全屏语义）使用，未动。`dart format` 通过、`flutter analyze`（改动文件 + 两测试）**0 issue**。
- **[x] ② 已加自动化测试** — `97ed3cf3c`：分两层守护。**行为（最强可落地层，机制 + 修复模式真跑）** `hibiki/test/widgets/swipe_dismiss_wrapper_test.dart` 新增 `nesting hazard (BUG-051)` 组 2 例：外层 wrapper 在场时横滑内层 → 内外 `onDismiss` **双触发**（锁住「Listener 冒泡连带整卡」的根因机制）；移除外层 wrapper（镜像 `_buildCard` 栈深>1 不渲染外层）后同样横滑 → **只内层触发**（锁住修复模式）。**源码守卫** `hibiki/test/pages/popup_dictionary_page_nested_test.dart`：断言嵌套层满卡渲染（`Positioned.fill`、`swipeDismissible: !isBase`、base 透明）、**不再含** `buildNestedPopupLayer(`/`calcPopupPosition`、外层横滑闸门 `if (_stack.length > 1) return card;` + 嵌套返回 `() => _popAt(index)`、按层截断 `_stack.removeRange(index + 1, _stack.length)`。`flutter test test/widgets/swipe_dismiss_wrapper_test.dart test/pages/popup_dictionary_page_nested_test.dart` **12 绿**。注：`popup_dictionary_page_test.dart`/`dictionary_popup_layer_test.dart` 因 develop 在飞的 v16 book-key 迁移（`sync_manager.dart`/`sync_auto_trigger.dart` 的 `book.id`/`getAudiobookByBookUid`/`getEpubBook(int)` 编译错，**非本改动**，仅 `popup_dictionary_page.dart` 在我改动集内）暂无法加载，待迁移收口后回归。
