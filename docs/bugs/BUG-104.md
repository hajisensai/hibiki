## BUG-104 · 大容器（27GB BluRay REMUX）切换内嵌字幕「点了没切换过去」
- **报告**：2026-06-07（用户：「`...TrueHD.5.1-ManualMux.mkv` 这个的内嵌字幕点击以后没切换过去」）。采番注：初记 103，rebase 到 develop 时 103 已被并发提交「视频统计纵坐标 0m」占用，改取 104。
- **真实性**：✅ **真 bug（沿真实代码路径 + 真实文件实测定位）**。先排除「图形字幕」假设：`ffmpeg -i` 实测该 mkv 的 8 条内嵌字幕**全是 `subrip (srt)` 文本字幕**（含 `#0:6 jpn`），`subtitleFormatForCodec('subrip')→srt`，本应可解析；我实测抽 jpn 轨得到合法日文 srt（138KB）。真根因是**性能+零反馈+固定超时**：① 点击走 `video_hibiki_page.dart:_selectSubtitleSource` **同步 await** `loadCuesForSource`→`_loadEmbeddedCues`→`extractEmbeddedSubtitleViaFfmpeg`（`ffmpeg -map 0:s:N`）。② 27GB 是**交错容器**，抽**任一条**字幕都要 ffmpeg 读穿整个时间轴——实测**单条 ~19–20s**（纯 I/O，CPU≈0）。③ 点击后底栏菜单 `Navigator.pop` 立即关闭，**整段 ~20s 无任何加载提示**，画面字幕不变 → 用户读作「没切换过去」。④ `desktop_audio_clipper.dart:316` 写死 **30s 超时**；播放时 media_kit 正读同一文件，I/O 争用把 20s 推过 30s → 返回 null → cue 空 → 黑底「加载失败」吐司（易错过）→ 仍「没切换」。
- **[x] ① 已修复** — 本提交。根因修（消除重复读盘的低效本质 + 补诚实反馈 + 放宽错误超时，非补丁）：
  - **单趟抽全部 + 缓存**（消除「每次切换都重抽 20s」）：新增 `buildFfmpegMultiSubtitleArgs`/`extractEmbeddedSubtitlesViaFfmpeg`（一个 `-i` + 多对 `-map 0:s:N out`，**单次 demux 抽出全部文本字幕轨**——实测抽 4 条仍 ~20s，与抽 1 条同）。`_loadEmbeddedCues` 改为先查 `embeddedSubtitleCacheDir`（按 base+size+mtime 键，原地换文件即失效）下的缓存文件，未命中才触发一次性全轨抽取（in-flight 去重防并发重抽），命中即瞬时读盘。同一视频在 8 条轨间切换只付一次 ~20s。
  - **加载遮罩**：`_selectSubtitleSource` 抽取前后包不可关 dialog（`_showSubtitleLoadingOverlay`/`_hideSubtitleLoadingOverlay`，`try/finally` 保证任何路径收起），消除「点了没反应」错觉。
  - **超时按体积放宽**：`subtitleExtractTimeoutForBytes`（60s + 8s/GB，clamp[60,1200]）取代固定 30s；27GB→276s 裕量，根治大文件/IO 争用下的静默超时失败。
- **[x] ② 已加自动化测试**：`hibiki/test/utils/desktop_audio_clipper_test.dart`（纯函数 `buildFfmpegMultiSubtitleArgs` 升序单 `-i` 多 `-map` + 真 ffmpeg 单趟抽 2 轨 e2e，ffmpeg 缺失自跳过）；`hibiki/test/media/video/video_subtitle_source_test.dart`（`embeddedSubtitleCacheKey` 命中/原地替换失效/非法字符折叠 + `subtitleExtractTimeoutForBytes` 下限/27GB=276s/上限/单调）；`hibiki/test/pages/video_subtitle_loading_overlay_guard_test.dart`（源码守卫：遮罩 show 在抽取前、hide 在 finally，media_kit headless 不可跑视频 widget）。
- **备注**：播放/性能/UI 类。`flutter analyze` 改动文件 0 issue + 全量 flutter test 2881 绿 0 失败。**真机/桌面肉眼复测待用户**：播该 REMUX，切内嵌字幕应先见加载圈、~20s 后字幕切到目标轨（jpn 等），**再切其它轨瞬时**；首次抽取期间不再「无反应」。
