## BUG-250 · 书架/视频标签多选模式按返回键直接退出 App（TODO-306）

- **报告**：2026-06-14（TODO-306：标签多选界面按返回退 App）
- **真实性**：真 bug。书架多选态 `_selectionMode` 活在 tab 内容 `_ReaderHibikiHistoryPageState`（`hibiki/lib/src/pages/implementations/reader_hibiki_history_page.dart:129` 状态字段、`:222`-`:234` toggle/exit）里，不是独立 route。HomePage 顶层 `PopScope`（`hibiki/lib/src/pages/implementations/home_page.dart:345`，`canPop: !syncing`）对多选态无感 → 返回键 `didPop=true` 弹掉 root route → 退出 App。对照正解：查词 tab 已用嵌套 `PopScope`（`hibiki/lib/src/pages/implementations/home_dictionary_page.dart:177`-`:186`）拦截活动查询/弹窗。视频 tab 同样问题：`home_video_page.dart` 的 `_selectionMode`（`:86` 状态、`:158`-`:170` toggle/exit）也没有 PopScope。
- **根因**：书架与视频两个首页 tab 在开启批量选择模式时，build 根都没有嵌套 `PopScope` 把返回手势消费掉。系统返回手势冒泡到 HomePage 顶层 PopScope，该 PopScope 只看同步状态，于是允许 pop 整个应用根 route。多选态是 tab 内部状态、不入路由栈，所以「返回应先退出多选态」这个语义从来没有被任何 PopScope 实现。
- **[x] ① 已修复**：书架 build 根（`reader_hibiki_history_page.dart` build 起始）包一层 `PopScope(canPop: !_selectionMode, onPopInvokedWithResult: (didPop, _) { if (!didPop && _selectionMode) _exitSelectionMode(); })`；视频 build 根（`home_video_page.dart` build 起始）同款 PopScope 调 `_exitSelectionMode()`。多选态下返回被消费、只退出多选态，不再弹 root route 退出 App；非多选态 `canPop=true`，返回行为不变。用 Flutter 3.44 的 `PopScope` / `onPopInvokedWithResult`。
- **[x] ② 已加自动化测试**：`hibiki/test/pages/shelf_video_selection_back_intercept_test.dart` 真 pump `HomeVideoPage`（内存 DB 注入、与 `home_video_page_menu_test.dart` 同一 harness），进入多选模式后经 `WidgetsBinding.handlePopRoute()` 派发系统返回，断言：多选态下返回被消费（页面不退、退出多选模式回到普通态），非多选态返回不被本页消费（`handlePopRoute` 返回 false，交给外层）。书架对称行为用源码守卫断言其 build 根包了 `PopScope(canPop: !_selectionMode ...)` 调 `_exitSelectionMode()`（书架页 pump 依赖大量 reader/webview provider，行为测试成本高，按「最强可落地层」用源码守卫 + 视频行为测试的等价 wiring 兜底）。
- **备注**：与 wave-1 集成分支（未合 develop）改同文件的 TODO-284（卡片布局）/ TODO-293（长按对话框 extraActions）由 PM 做 3-way 合并；本修复只动 build 根 + 批量标签段，不碰卡片布局/长按对话框区。真机返回键/手柄 B 复测待用户。
