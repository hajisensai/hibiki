## BUG-300 · 视频「下一句」跳到当前句（应排除当前句）
- **报告**：2026-06-15（用户：）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/media/video/video_player_controller.dart`（修复前 `nextCueIndexFor` ~1073 / `prevCueIndexFor` ~1095）：`_currentCueIndex` 由 125ms tick 的 `updateCueForPosition` 异步更新（`_tick` ~497），存在滞后窗口。`nextCueIndexFor`/`prevCueIndexFor` 在 `currentCueIndex` 合法时**只信任成员变量、忽略实时 `positionMs`**。用户 seek 进某句后 tick 尚未追平时，`_currentCueIndex` 仍停在上一句：`next = 上一句 + 1` 恰好等于当前句本身（「下一句跳到当前句」），`prev = 上一句 - 1` 乱跳 / no-op。
- **[x] ① 已修复** — 见本轮 commit。`nextCueIndexFor`/`prevCueIndexFor`/`prevSeekDecisionFor` 彻底移除 `currentCueIndex` 形参（消除两个真相源），改以实时 `positionMs` 为权威：`nextCueIndexFor` 用 `_floorCueIndexByPosition(positionMs)` 定位当前/最近句 floor，下一句 = `floor + 1`（严格排除当前句；早于首句→0；末句→null）。`prevCueIndexFor` 用 `JsonAlignmentParser.findCueIndex(positionMs)` 区分语义（方案 B）：句中命中 `hit>=0` → 上一句 = `hit - 1`（排除当前句，与 next 对称，hit==0 返 null）；gap / 早于首句（findCueIndex 返回 -1）→ 二分回退到 gap 之前刚播完那条（保留 TODO-085/119 转场回退参照）。同步更新 4 处内部调用点（`skipToNextCue` / `skipToPrevCue` / `skipToPrevCueOrSeekBack` / `skipToNextCueOrSeekForward`）。保留 `skipToCue` / `cueSeekTargetMs` 的 preroll 下界钏制（BUG-259）、`skipToPrevCue` / `_replayCurrentCueAndPokeControls` 重播本句独立语义、BUG-176/189（OP gap/早于首句→0/末句→null）、BUG-074（gap 清空字幕条）不变。
- **[x] ② 已加自动化测试** — `hibiki/test/media/video/video_player_controller_test.dart` 新增 group `TODO-410 stale currentCueIndex`：相邻无 gap 三 cue，实时 `positionMs=14000` 落在 cue1 内、成员变量陈旧停在 0，断言 `nextCueIndexFor→2`（不是 1）、`prevCueIndexFor→0`（不乱跳），并含真实控制器 tick 滞后场景。撤修复（回到读 stale 索引）实测红：next 算成 1（=当前句）、prev 算成 null。现有命中 / gap / 边界 / TODO-073 / TODO-085 / TODO-119 用例全绿（全 video 套件 641 通过，2 skip 为 media_kit headless smoke）。
- **备注**：本 bug 仅作用于 `VideoPlayerController`；有声书的 `AudiobookPlayerController`（`packages/hibiki_audio`）有独立的 `nextCueIndexFor`/`prevCueIndexFor` 和「正文跟随高亮」语义，不在本次范围。
