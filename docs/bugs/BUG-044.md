## BUG-044 · 词典管理页长按拖拽重排时拖拽浮层向右下漂移、飞离原位（界面缩放下 SDK ReorderableListView 坐标错配）
- **报告**：2026-06-04（用户截图：放大「界面大小」后，词典管理页长按托选某词典，拖拽的「长框」位置向下偏移、且不在原本位置而是漂到右下方）。采番注：动手前查 `docs/BUGS.md` 当前最大号为 BUG-043（042/043 已被并发 agent 占用），故本条取 044。
- **真实性**：✅ **真 bug（祖先 Transform.scale 下 SDK Overlay 拖拽代理坐标错配），确定性**。沿真实代码路径定位：词典管理页 `dictionary_dialog_page.dart` 的 `_buildDictionaryList` 用 SDK `ReorderableListView.builder` + `_buildDictionaryTile` 的 `ReorderableDelayedDragStartListener` 渲染可拖拽列表。整个 app 被全局 `HibikiAppUiScale`（浏览器式整体 `Transform.scale`）缩放；SDK 拖拽代理（`reorderable_list.dart` 的 `_DragItemProxy`）把拖拽浮层放进 Overlay 时用「全局坐标 − overlay 原点」的**纯平移**定位、不认祖先缩放变换 → 缩放系数 s≠1 时浮层实际落点按 `(1−s)×(指针到 overlay 原点距离)` 漂移，正是用户截图里「长框漂到右下方、不在原位」的症状（与音频来源对话框 BUG-027②、`HibikiAppUiScale` 备忘里记录的同一 SDK 坐标缺陷同源）。根因点 = `dictionary_dialog_page.dart` 的 `_buildDictionaryList`/`_buildDictionaryTile` 用 SDK `ReorderableListView`/`ReorderableDelayedDragStartListener`，其 Overlay 拖拽代理在祖先 `Transform.scale` 下坐标错配，app 内改不了 SDK 那段数学。
- **[x] ① 已修复** — `95ea0c558`：根因修——改用仓库已有的 `HibikiReorderableColumn`（`lib/src/utils/components/hibiki_reorderable_column.dart`），消除「拖拽浮层走 Overlay 全局坐标、不认祖先缩放」这个特殊情况，而非补丁。① `_buildDictionaryList` 从 `ReorderableListView.builder` 改为 `HibikiReorderableColumn`：后者不借助 Overlay，拖拽浮层复制就渲染在列表自身 `Stack` 里（`Positioned`，列表本地坐标），随整棵子树被祖先 `Transform.scale` 统一缩放 → 视觉与其余缩放界面一致；所有指针坐标都用列表 `RenderBox.globalToLocal` 转本地坐标定位浮层，自动消掉祖先任意缩放/平移 → 任意缩放下精确跟手、零偏移。音频来源对话框（`dictionary_settings_dialog_page`/`local_audio_sources_dialog`/`audio_sources_dialog`）早已用它，本条把唯一遗漏的「实际在用的」词典管理页补齐。② `_buildDictionaryTile` 去掉 `ReorderableDelayedDragStartListener` 包装（长按拖拽由 `HibikiReorderableColumn` 内部 `onLongPress*` + `HitTestBehavior.translucent` 统一接管，行内 Switch/可见性按钮/上下箭头/3点菜单普通点击照常）。③ 顺带把 `_reorderDictionaries` 从 SDK 的「插入前下标」语义（`if(newIndex>oldIndex)newIndex--`）重构为「最终下标」语义：上箭头 `index-1`、下箭头 `index+1`（旧为 `index+2`+SDK 自减，等价）、拖拽 `onReorder:(from,to)` 直传 `to`——三条路径统一 `removeAt(from); insert(to)`，消除 SDK 的特例分支（好品味：消除特殊情况）。order 重排 + `appModel.updateDictionaryOrder` 持久化不变。`flutter analyze`（改动文件）0 issue、`dart format` 0 改动。
- **[x] ② 已加自动化测试** — `95ea0c558` · `hibiki/test/pages/dictionary_dialog_layout_static_test.dart`：源码扫描守卫（该页 widget 挂载需完整 appModel/DB，与既有 `custom_fonts` reorder 守卫同范式取最强可落地层）。新增「dictionary list uses HibikiReorderableColumn (UI-scale safe), not SDK ReorderableListView」：断言页源码含 `HibikiReorderableColumn(`、**不含** SDK 拖拽控件的构造调用（`ReorderableListView.builder(`/`ReorderableListView(`/`ReorderableDelayedDragStartListener(`/`ReorderableDragStartListener(`——说明性注释可提及其名字，故只禁构造调用形）、且保留上下箭头图标（无障碍/手柄重排路径）。任何人把词典页退回 SDK 重排控件即红。真红→绿验证过（修复前断言 `HibikiReorderableColumn(` 缺失 + `ReorderableListView.builder(` 存在 → 红；修复后 → 绿）。`flutter test test/pages/dictionary_dialog_layout_static_test.dart` 10 绿、`md3_design_system_static_test` 无回归。
