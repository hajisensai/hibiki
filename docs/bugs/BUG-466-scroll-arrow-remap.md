## BUG-466 · 滚动模式方向键改绑有声书句子无效仍翻页
- **报告**：2026-06-30（用户：TODO-992）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/shortcuts/reader_space_override.dart:resolveReaderArrowPageTurn`（修前）+ 调用点 `hibiki/lib/src/pages/implementations/reader_hibiki/caret.part.dart` `_handleKeyEvent`。用户在快捷键设置里把裸 ArrowLeft/ArrowRight 从「阅读器翻页」(`readerPageForward`/`readerPageBackward`) 解绑、改绑「有声书上一句/下一句」(`audiobookPrevSentence`/`audiobookNextSentence`)，但方向键仍只翻页、不动有声书。
- **根因详述**：阅读器唯一键盘入口 `_handleKeyEvent` 里，`action` 解析顺序为 `spaceOverride ?? arrowOverride ?? directReaderAction ?? registry(reader) ?? registry(audiobook)`。其中 `arrowOverride = resolveReaderArrowPageTurn(...)`（修前）**无条件**把裸 Left/Right 映射成翻页（仅按 RTL/reverse 决定前进/后退方向），完全不查用户当前绑定，**优先级高于** `directReaderAction` 与注册表。于是用户改绑的有声书句子动作永远没机会执行。该覆写本是为 BUG-098/099（RTL 书翻页方向跟随阅读方向）+ TODO-120（反转开关）服务，但「写死翻页、忽略改键」是设计缺陷。分页(paged)与连续滚动(continuous)两模式**共用同一条** `_handleKeyEvent` 路径（差异只在被解析之后的 `_paginate` 内部 `chrome.part.dart:62` 执行分叉，连续走 JS 滚动、分页走翻页），JS 侧无 keydown 监听、WebView 外无 Flutter Scrollable，故方向键消费唯此一处。用户感知「分页生效、滚动不生效」是观感差异；代码层两模式行为一致，本修复对两模式同时生效。
- **[x] ① 已修复** — 给 `resolveReaderArrowPageTurn` 增加 `required ShortcutAction? boundAction` 入参（调用方传入该裸键在 reader+audiobook co-active 组解析出的真实绑定）：仅当 `boundAction` 仍是翻页动作（`readerPageForward`/`readerPageBackward`）时才做阅读方向校正；否则（改绑别的动作 / 解绑为 null）返回 null 让出，交回注册表解析用户真实绑定。默认绑定下裸 Left/Right 仍解析为翻页，行为不变（Never break userspace）。改动文件：`hibiki/lib/src/shortcuts/reader_space_override.dart`（gate + 文档）、`hibiki/lib/src/pages/implementations/reader_hibiki/caret.part.dart`（`_handleKeyEvent` 计算 `bareArrowBinding` 并传入）。提交：见本分支 commit。
- **[x] ② 已加自动化测试** — `hibiki/test/reader/reader_space_pause_test.dart` 新增 group「resolveReaderArrowPageTurn (TODO-992 改键后让出，尊重用户绑定)」：改绑有声书上/下句 → 覆写让出(null)（rtl × reverse 全组合）；解绑(boundAction=null) → 让出；仍绑翻页(默认) → 照常方向校正（默认行为不变）；IME process+物理键但已改绑非翻页 → 仍让出。同时既有 BUG-098/099/TODO-120/TODO-847 用例补传 `boundAction: readerPageForward` 保持覆盖。提交：见本分支 commit。
- **备注**：连续滚动模式按方向键移动有声书句子需 EPUB+真实音频+cue，headless 自动化照不到，留 integration owner / 用户真机验证。纯函数 gate + 解析顺序为确定性逻辑，已被上述单测覆盖。
