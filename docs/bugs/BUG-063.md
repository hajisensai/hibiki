## BUG-063 · Google Drive 同步在 access token 过期约 1 小时后报 `同步错误：Access was denied (... error="invalid_token")`，自动刷新永不触发，需手动重登
- **报告**：2026-06-05（用户截图：同步错误 `Access was denied (www-authenticate header was: Bearer realm="https://accounts.google.com/", error="invalid_token").`）。采番注：动手前查当前最大号 BUG-059，本条取 063（原 worktree 内编号 060，合并时与并发占用的 BUG-060 撞号，落 develop 时改 063）。
- **真实性**：✅ **真 bug（401 刷新-重试分支认错异常类型，过期 token 的最常见表现完全绕过自动刷新）**。沿真实代码路径定位：① 桌面/移动端 `GoogleDriveAuth.getAuthClient()`（`google_drive_auth.dart:112`）返回的是 `auth.authenticatedClient(...)` —— **普通客户端，不自动刷新**。Google access token 约 1 小时过期。② token 过期后再调 Drive，`googleapis_auth-1.6.0/lib/src/auth_http_utils.dart:43` 的 `authenticatedClient.send()` 一见响应带 `www-authenticate` 头就 `throw AccessDeniedException('Access was denied (www-authenticate header was: $wwwAuthenticate).')`——**与截图逐字吻合**，且在 googleapis 把 401 转成 `DetailedApiRequestError` **之前**就抛了。③ 根因点 `google_drive_handler.dart:_call`（旧）：401 刷新-重试只 `on drive.DetailedApiRequestError catch (e) { if (e.status == 401) ... }`，而第②步抛的 `auth.AccessDeniedException` **不是** `DetailedApiRequestError`、也不是 `GoogleDriveAuthError` → 两个 catch 都不命中 → 异常裸冒泡到同步层 → 用户看到 `invalid_token` 硬失败。`refreshAuth()`（`:302`）本身完好（BUG-034 已确保桌面有 refresh token），只是**触发刷新的那道闸认错了类型**，导致每隔约一小时同步必失败一次、得手动重登才恢复。上传/下载/进度 json 全走 `_call`，故所有同步动作同病。
- **[x] ① 已修复** — 根因修（Linus 式：消除「401 只有一种异常形状」的特例，统一成一个分类器）。提交 `f8dc8515e`。新增 `@visibleForTesting bool googleDriveErrorIsUnauthorized(Object)`（`google_drive_handler.dart`）：把 `DetailedApiRequestError(status==401)`、`auth.AccessDeniedException`、`auth.ServerRequestFailedException(statusCode==401)` 统一判为「可刷新的未授权错误」；`_call` 重构为 `catch (e)` + 该分类器 → 清缓存 client → `refreshAuth()` → 重试一次。等价性说明（opus 审 W-1）：保留 `on GoogleDriveAuthError { rethrow; }` 不被包装、非 401 的 `DetailedApiRequestError` 仍包装成 `GoogleDriveError(message, statusCode)`、其它异常原样 rethrow——这些与原版等价；**唯一有意的行为改变**：重试后仍属 unauthorized（如又抛 `AccessDeniedException`）的情况，原版会裸冒泡那串丑陋异常，新版统一包装成 `GoogleDriveError(retry.toString(), statusCode: 401)`（经 `_wrapErrors` 归一为 `SyncBackendError(isRetryable:false)`，不会无限重试），是改进不是回归。平台无关：移动端 `_signIn.authenticatedClient()` 同样经 googleapis_auth wrapper、同样受益。`flutter analyze` 0 issue。
- **[x] ② 已加自动化测试** — `hibiki/test/sync/google_drive_unauthorized_classify_test.dart`（行为级守卫，最强可落地层——直接用截图原文构造 `auth.AccessDeniedException` 喂分类器）6 例：截图原文 `AccessDeniedException`→true、`DetailedApiRequestError(401)`→true、`ServerRequestFailedException(401)`→true、`DetailedApiRequestError(404/403)`→false、`ServerRequestFailedException(500)`→false、`SocketException`/`StateError`→false。**red→green 实测**：临时注释掉分类器里 `AccessDeniedException` 分支 → **恰好** BUG-060 那条「截图原文」用例红、其余 5 绿；还原后 6 绿。`test/sync/` 全量 291 绿无回归。
- **备注**：同步/Google Drive/认证类。修复纯 Dart、CI 可测。**测试覆盖边界（opus 审 I-6）**：守卫只覆盖纯函数分类器 `googleDriveErrorIsUnauthorized`，未覆盖 `_call` 的重试**编排**（清缓存→refresh→重试一次→重试仍失败的包装）——因 `_call` 依赖单例 `GoogleDriveAuth.instance` 难注入，分类器层是最强可落地层，编排正确性依赖真机复测。**真机肉眼复测原始失败路径待用户**（桌面或移动）：登录 Google Drive 后保持 app 开着等 access token 过期（约 1 小时）再触发同步，应自动刷新成功、不再弹 `invalid_token`；或人为吊销 token 后同步。**关联**：[[BUG-034]]（桌面拿不到 refresh token，已修「能不能刷」）；本条修「过期时压根没去刷」，两者互补。
