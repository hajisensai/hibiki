## BUG-194 · LateInitializationError: languages map accessed before populateLanguages during init
- **报告**：2026-06-11（用户：旧包 427362e7 启动崩溃 `LateInitializationError: Field 'languages' has not been initialized.`；当前 develop 仍是潜伏 bug）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/models/app_model.dart:474`（`late Map<String, Language> languages;`，唯一初始化在 `populateLanguages()` `:855`，而该调用在 `initialise()` 的 `:1275`/`:1482` 才发生，前面有 ~80 行 await：DB 打开 / BackupService / SyncRepo / 仓库并行加载）。旧 getter `Language get targetLanguage => languages.values.first;`（旧 `:1655`）在 init 早期窗口（`populateLanguages` 之前）被任意旁路 widget 重建访问就抛 `LateInitializationError`。`initialise()` 被包在 try/catch（早期 await 抛的异常被捕获设 `_initError` 切 error 屏），所以崩溃只能落在某个在 init 早期窗口重建、且无 `isInitialised` 守卫、直接读 `targetLanguage` 的 widget（裸访问点：`reader_hibiki_history_page.dart:247` 书架 build 第一行 `ref.watch(hibikiBooksProvider(appModel.targetLanguage))`；次要异步回调 `reader_hibiki_source.dart:112/185`）。
- **[x] ① 已修复** — `hibiki/lib/src/models/app_model.dart:1663`：`targetLanguage` getter 改为直接返回 `JapaneseLanguage.instance`，不再读 `late languages` map，消除整类 init-window race。根因修而非守卫绕过：app 全程只注册一个语言（`populateLanguages` 的 `availableLanguages` 列表硬编码唯一项 `JapaneseLanguage.instance`，`app_model.dart:856-861`），单语言场景下 `late map + values.first` 是过度设计；getter 返回值在生命周期任意时刻都恒等于这同一个实例，正常运行期（振假名 / 查词 / 字体 / 词典隐藏折叠）行为不变。`languages` 字段与 `populateLanguages()` 调用保留（不删，避免动 `initialise()` 流程）。修复 commit：6959f40c00a5b8937f869fe39651c3d430102b02
- **[x] ② 已加自动化测试** — `hibiki/test/models/target_language_late_init_guard_test.dart`：构造未跑 `initialise()` 的 `AppModel`（`languages` 未初始化，正是早期窗口），断言 `targetLanguage` 不抛 `LateInitializationError` 且返 `JapaneseLanguage.instance`；第二例断言 `populateLanguages()` 前后 `targetLanguage` 是同一实例（无回归）。撤修复（getter 改回 `languages.values.first`）实测转红，复现原始 `LateInitializationError: Field 'languages' has not been initialized.`。测试 commit：6959f40c00a5b8937f869fe39651c3d430102b02（同提交）
