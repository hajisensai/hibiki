## BUG-467 · 竖排正文文字溢出到底栏区域
- **报告**：2026-06-30（用户：截图——竖排日文正文 + 蓝色音频高亮，正文画进底部控制栏区域，「文字去到底栏了」）
- **真实性**：✅ 真 bug（TODO-975 回归）。根因 `hibiki/lib/src/pages/implementations/reader_hibiki_page.dart:1148`（`_bottomChromeReserve` getter 门控 `_hasEverLoaded && _showChrome`）+ `hibiki/lib/src/pages/implementations/reader_hibiki/webview.part.dart:351`（初始 HTML 用 `chromeBottomInset: _readerBottomReserve` 求值，此刻 `_hasEverLoaded==false`）。
  - 数据流：初始 WebView HTML 在 `_buildWebView` 里求 `chromeBottomInset = _readerBottomReserve = _bottomChromeReserve + _stableBottomInset`。`_bottomChromeReserve` 经 TODO-975（commit `f03f398d5`）改成门控 `_hasEverLoaded && _showChrome`；但 HTML 构建发生在内容首次就绪之前，`_hasEverLoaded` 仍为 false → `_bottomChromeReserve == 0` → 初始底栏预留只剩系统底 inset（多数桌面/手势导航机为 0），底栏高度被漏掉。
  - TODO-975 之前旧式 `chromeBottomInset: _showChrome ? _readerChromeHeight + _stableBottomInset : _stableBottomInset` 不依赖 `_hasEverLoaded`、`_showChrome` 默认 true，首屏即正确预留。975 把预留改成派生 getter 并塞进 `_hasEverLoaded` 门控，破坏了首屏预留。
  - 内容就绪后（`_hasEverLoaded` 翻 true）**没有任何代码重下 chrome insets**（`_applyChromeInsets` 仅由 `_setChromeVisible`/`_toggleChrome`/`onChromeReanchorLive` 触发，不在 load-complete 路径）→ WebView 永远停在「底栏未预留」。
  - 竖排 vertical-rl 字形沿物理纵轴流到屏底，底栏预留缺失时整列文字直接画进底栏区 → 现象最刺眼；横排同样漏底，但竖排「文字进底栏」最直观，正是用户截图所见。CSS（`reader_content_styles.dart` 的 `verticalColumnWidthCss`/padding-bottom/clip-path）本身正确——它扣 `var(--chrome-bottom-inset)`，问题在喂进来的 inset 值偏小。
- **[x] ① 已修复** — 新增 `_reapplyChromeInsetsAfterFirstLoad()`（`reader_hibiki/chrome.part.dart`），在每个「内容首次就绪、`_hasEverLoaded` 翻 true」落点补一次 `_applyChromeInsets()`，把此刻已正确的 `_readerBottomReserve`（含底栏高）下发给 WebView。挂在 3 处：`navigation.part.dart` 的 `_onRestoreComplete` 正常路径 + 兜底超时路径、`webview.part.dart` 的 `spreadReady`。幂等、零行为变化于 975 语义（悬浮态仍预留 0、关进度仍 0，读同一组派生 getter）；歌词模式由 `_applyChromeInsets` 自身 `_lyricsMode` 早返回挡掉。提交：（见分支 fix-993-vertical-text-bottom-overflow）。
- **[x] ② 已加自动化测试** — `hibiki/test/reader/reader_chrome_floating_test.dart` 加 source-scan 守卫组「first-load chrome inset re-apply (BUG-467 regression)」：钉死①辅助方法存在且调 `_applyChromeInsets`；②三处内容就绪落点（onRestoreComplete×2 + spreadReady）都调 `_reapplyChromeInsetsAfterFirstLoad()`；撤回任一即红。WebView 真值层（初始 HTML inset 偏小）无法 headless 起整页，故钉「就绪后必补发」这条结构不变式。
- **备注**：真机待验——竖排 + 有声书模式开书首屏，正文列底沿应停在底栏之上不侵入；横排同验；切章/退出再进首屏同验。
