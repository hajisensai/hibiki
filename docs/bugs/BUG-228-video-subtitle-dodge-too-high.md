## BUG-228 · 进度条出来字幕往上顶太高（抄B站只让进度条上缘）
- **报告**：2026-06-12（用户：底部进度条出来时把字幕往上顶，顶太上面了很怪，抄 B站）
- **真实性**：✅ 真 bug（TODO-161 后续微调）。根因 `hibiki/lib/src/media/video/video_subtitle_style.dart:20`（修复前）：避让基准量 `kVideoControlsBottomReserve = 42 + 56 = 98` 是**整条底部按钮行**（离底 margin 42 + 按钮行高 56）。控制条可见时 `video_subtitle_overlay.dart:431-444 _paddingFor` 取 `max(bottomPadding, reserve)`，把字幕底缘抬过**整条按钮行**到 98px。但真正遮挡字幕的只是底部控制条最上沿的**进度条（seek bar）那一条**（骑在按钮行上沿，约一个按钮行高处），不需要让出整条按钮行 + 离底 margin。98px 把字幕顶进画面中上部，用户觉得「顶太上面很怪」。B站做法 = 字幕只抬到进度条上方一点点（进度条自身那一条），不含整条按钮行。
- **[x] ① 已修复** — `hibiki/lib/src/media/video/video_subtitle_style.dart:20-30`：`kVideoControlsBottomReserve` 从 `42 + 56 = 98` 降到 `_kButtonBarHeight = 56`（只保留到进度条上缘那一条 = 一个按钮行高，去掉「离底 margin 42」的累加；那 42 是控制条整体离屏幕底边的空白、不是遮字幕的实体，且 Hibiki 实际 new 的桌面主题 `MaterialDesktopVideoControlsThemeData` 本就走构造器默认 `bottomButtonBarMargin` 只有左右、vertical=0，没这 42px）。`_paddingFor` 的取下限逻辑（TODO-161）和 `AnimatedPadding` 动画不动，base 75 仍兜底。同步更新 style.dart line 6-30/71-78、overlay.dart line 109-113 的几何注释为「进度条上缘清空高度」。提交：（见报告 commit）。
- **[x] ② 已加自动化测试** — `hibiki/test/media/video/video_subtitle_style_test.dart`：守卫 `expect(kVideoControlsBottomReserve, 56)` + `isNot(98)` + `lessThan(98)`（断言已减小、不再含整条按钮行 + margin 累加）；反转 089 守卫从过时的 `< reserve` 关系改为 `isNot(75 + reserve)`（reserve 降到 56 < 基线 75 后 `< reserve` 不再成立）。`hibiki/test/media/video/video_subtitle_overlay_test.dart`：避让几何用例改用 `lowBaseline = kVideoControlsBottomReserve - 1`（与具体 reserve 解耦），验证基线低于 reserve 时抬到 reserve、隐藏落回基线、低位用户取下限躲进度条、高位用户尊重原值。`test/pages/video_subtitle_push_up_guard_test.dart`（TODO-161 取下限非加法守卫，符号引用不写死数字）继续绿。改完 `dart run tool/bug.dart reindex`。
- **备注**：reserve 是**单一常量**。桌面进度条用 `Transform.translate(Offset(0,16))` 重叠在按钮行上沿（上缘约 36~56px，默认基线 75 已在其上 → reserve=56 让默认字幕停在 75 恰骑进度条上方一点点 = B站效果，控制条可见时不再被多抬到 98）。移动端 `_mobileControlsTheme` 进度条 bottom = `bottomChromeInset(24+系统inset) + buttonBarHeight(56) + gap(8) ≈ 88+px`，比 reserve 高（这在旧值 98 下也 < 移动端进度条上缘，非本次新引入的回归）。**真机微调项（dev=True）**：① 桌面/移动两端进度条出现时字幕实际停位是否「骑进度条上方一点点」不被遮、不太高；② 若移动端默认 75 仍被高进度条遮，考虑把 reserve 提到移动端进度条上缘量级或按平台分值（需另起，本次先按桌面主诉求降到 56）。**撞号风险**：TODO-152 并发改 `video_subtitle_jump_panel.dart` + `video_hibiki_page.dart` 的 fit，若它同时 `bug.dart new` 可能也取 228（develop 最大 BUG-227），合并时若撞号须重编号本条。
