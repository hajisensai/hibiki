## BUG-129 · 嵌套查词弹窗遮挡被查的词
- **报告**：2026-06-08（用户：「查词弹窗里面，查词的时候，查词弹窗会把查的词挡住」）。采番注：`tool/bug.dart` 在 origin/develop（worktree fresh 基底）上取到空号 124，但多个未 push 的并发 worktree 已占用 124-128（`worktree-android-self-built-ffmpeg` 占 124；`worktree-video-subtitle-fixes` 占 124-128），故本条改号 **129** 规避撞号（见根级 CLAUDE.md 撞号约定）。采番前须 `git ls-tree <各 worktree 分支> docs/bugs/` 查全部未 push 番号。
- **真实性**：✅ **真 bug（坐标系错误，沿真实代码路径定位）**。在弹窗内再查词（嵌套查词）时，子弹窗的定位锚 `childRect` 由 `localRect.shift(Offset(parentPos.left, parentPos.top))` 重建（`base_source_page.dart` onTextSelected/onLinkClick、`dictionary_page_mixin.dart` 同名回调）。`localRect` 是 `getBoundingClientRect()` 给的、相对 **WebView 视口左上角** 的词矩形；但 `parentPos` 是父弹窗 `Positioned` 的左上角——**不是 WebView 的左上角**。`DictionaryPopupLayer._buildContent` 里当 `headerWidget!=null`（index 0 弹窗恒有 `buildPopupAudioControls` 的收藏★/音频控制行，约 48px）时 WebView 被 `Column` 压在 header 之下；外层还有 `HibikiPopupSurface` 边框内缩、以及可能的 `appUiScale` 祖先缩放。于是 `childRect` 比真实词矩形整整高出一个 header 高度，子弹窗按 BUG-098 逻辑贴在 `childRect` 下方时正好盖住真正被查的那个词（`dictionary_popup_layer.dart:8` 的 `calcPopupPosition` 本身没错，错在喂给它的选区矩形）。
- **[x] ① 已修复** — 本提交。**根因修（消除手工重建几何的整类特例，改问框架真实渲染位置）**：新增纯函数 `popupWordScreenRect`（`dictionary_popup_layer.dart`），用 `webViewKey.currentContext.findRenderObject()` 取 WebView 的 `RenderBox`，对 `localRect` 的左上/右下两角各做一次 `localToGlobal` 再 `Rect.fromPoints`——一次性把 header 偏移、surface 边框内缩、任意祖先缩放全部沿真实 transform 链算对（两角分别映射故缩放下的尺寸也正确，不只是平移）。`base_source_page.dart` 与 `dictionary_page_mixin.dart` 的 onTextSelected/onLinkClick 嵌套回调改用它，渲染框不可用时回退原 `selectionRect`。`base_source_page` 删掉为旧重建而多算的 `parentPos`（`_calculatePopupPosition` 仍服务渲染分支，未动）。`localRect==Rect.zero`（系统右键菜单「搜索」无矩形）仍回退父选区，行为不变。
- **[x] ② 已加自动化测试** — `hibiki/test/pages/popup_word_screen_rect_test.dart`：① widget 行为测试镜像 `DictionaryPopupLayer` 结构（`Positioned`→`Column[header(48px), Expanded(WebView 占位)]`），断言 `popupWordScreenRect` 返回的词矩形顶边 = 118（= Positioned.top 50 + header 48 + 局部 y 20），并显式对照旧 `localRect.shift(Positioned.topLeft)` 得 70（在词之上=覆盖）证明已修正；② 渲染框不可用回退用例；③ 源码守卫：两文件必须经 `popupWordScreenRect(` 且不得再出现 `localRect.shift(Offset(`。本测试 + `dictionary_popup_layer_test`/`popup_dictionary_page_nested_test`/`popup_vertical_writing_wiring_test` 共 28 绿，`test/pages` 全量 353 绿，`flutter analyze` 改动文件 0 issue。
- **备注**：弹窗坐标变换类。代码正确 + 单测/全量无回归；**真机复测待用户**：在书内（尤其有声书，index 0 弹窗带音频控制 header）查词后，于弹窗内再选一个词下钻，子弹窗应出现在该词的上/下方且不遮住它；视频/首页查词页/texthooker 同链路一并修正。
