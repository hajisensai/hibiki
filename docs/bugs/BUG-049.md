## BUG-049 · 「本地 vs 远端」对比框总有一条「可下载」却永远下不下来（远端文件夹只剩同步元数据、无 .epub 内容）
- **报告**：2026-06-05（用户：「为什么本地vs远端，总是有一个远端下载的，明明我下载不了，明明我没有这本书，并且云端没有这本书，只有云数据，我却下载不了，能不能修复，可以放弃旧数据，暂时没人用」）。采番注：动手前查 `docs/BUGS.md` 当前最大号为 BUG-048（被并发 agent 占用 = 设置文本框焦点），故本条取 049。
- **真实性**：✅ **真 bug（对比框把「只有同步元数据、没有书内容」的远端文件夹当成可下载书，且永远下不下来）**。沿真实代码路径定位：对比框 `_fetchCompareData`（`sync_compare_dialog.dart`）对 `backend.listBooks(root)` 列出的每个远端文件夹建一条 entry；`_fetchRemoteBookData` 只读 `listSyncFiles`（progress_/statistics_/audioBook_ 三类元数据 JSON）**完全不检查该文件夹有没有真正的 `.epub` 内容资产**。本地没这本书 → `bookId==null`；只要远端有 progress.json → entry 默认选择（旧 `:457-461`）`remoteFolderId!=null ? useRemote(下载) : skip`、`_isActionable`（旧 `:500`）`remoteFolderId!=null` → 被当「可下载远端独有书」。点 Apply → `importRemoteBookFolder` 列子项找不到 `.epub` → `return false` → 没导入、entry 不消失 → 每次开对比框都在、永远拉不下来 = 幽灵。**成因**：源设备开着进度同步但关着内容同步（只推了 progress 元数据没推 EPUB），或本地删了书但远端元数据没删（删除从不传播），留下只剩 progress/stats 的孤儿文件夹。`google_drive_handler.dart:278 listSyncFiles` 其实已列出文件夹全部文件，但只按前缀挑元数据、把 `.epub` 丢弃，故信息本就拿得到。根因点 = `sync_compare_dialog.dart` 把「远端独有可下载」只判 `remoteFolderId != null`，漏判「该文件夹是否真有 `.epub` 内容」。
- **[x] ① 已修复** — `93906e986`：根因修——给「远端独有可下载」补上「确有内容」这一必要条件（非补丁、非吞错）。① 新增 top-level `_remoteFolderHasContent(backend, folderId)`（`listChildren` 扫 `.epub`，镜像 `importRemoteBookFolder` 的内容判定；列举失败时**保守返回 true 不隐藏真书**）。② `_fetchCompareData` 仅对「本地无书 + 远端有文件夹」的候选查一次内容（省网络），结果存入 `SyncCompareEntry.remoteHasContent`（新字段，默认 true）。③ 新增 getter `isDownloadableRemoteOnly = bookId==null && remoteFolderId!=null && remoteHasContent`；默认选择、`_isActionable`、批量「全部用远端」、行内下载控件渲染全部改用它——**无内容的孤儿不再默认勾下载、不计入 Apply、不显示下载勾选框，改显示「仅云端数据，无可下载书籍」提示**。④ **关键：条目仍保留**（不是过滤掉），行内删除菜单（`remoteFolderId!=null` 即显示）照旧 → 用户可一键删除孤儿远端数据清理（契合「可以放弃旧数据」）。中途曾误把孤儿整条 `continue` 过滤 → `sync_compare_delete_test` 3 红（删除入口没了），已纠正为「保留可删 + 不可下载」。`flutter analyze lib` 0 issue、`dart format` 通过。
- **[x] ② 已加自动化测试** — `93906e986` · `hibiki/test/sync/sync_compare_orphan_test.dart`：加 `@visibleForTesting fetchCompareDataForTest` 测试缝直跑真实 `_fetchCompareData`。fake backend 带 `hasEpub` 开关：`hasEpub:false`（孤儿）→ entry **保留**但 `remoteHasContent==false`、`isDownloadableRemoteOnly==false`；`hasEpub:true`（真书）→ `isDownloadableRemoteOnly==true`。红→绿守护（修复前孤儿会 `isDownloadableRemoteOnly==true`）。`flutter test test/sync/` **284 绿**（含本例 + 既有 download/delete/conflict 对比测试无回归）。
