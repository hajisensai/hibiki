- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/media/video/video_player_controller.dart:1467`（`cueSeekTargetMs` 的 `cueStartMs - preRollMs`）与同文件 `_syncCueForPosition` 的纯位置反推（`json_alignment_parser.dart:102` 闭区间 `findCueIndex`）耦合冲突。引入 commit `feccc417b`（BUG-259 加前导余量，2026-06-14）。
  - 数据流：点字幕列表第 N 行 → `video_subtitle_jump_panel.dart` `onTapCue(cue)` → `video_hibiki_page.dart` `_handleSubtitleJumpTap` → `skipToCue(cue_N)`（controller `:1422`）→ seek 落点 = `cue_N.startMs − 180ms`（preRoll，行 `:1467`）→ 125ms timer 读实际位置 ≈ `cue_N.startMs − 180ms` → `updateCueForPosition` → `findCueIndex(落点)` 因落点 `< cue_N.startMs` 二分判到上一句 `N-1`（或 gap `-1`）→ `_currentCueIndex = N-1` → 面板 `rawIndex == currentIndex` 高亮第 N-1 行。
  - 本质：preRoll 是「听感/seek 落点」需求（吸收 media_kit 关键帧吸附避免句首被吃），却污染了「高亮定位」推导。高亮真相源（TODO-410 无状态：实时 position 反推）对被 preRoll 故意偏移的瞬态位置给出了 N-1。
