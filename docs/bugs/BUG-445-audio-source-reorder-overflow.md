## BUG-445 · 管理音频来源排序对话框出框无法滚动且弹窗过小
- **报告**：2026-06-28（用户：）
- **真实性**：✅ 真 bug。根因在 `hibiki/lib/src/pages/implementations/local_audio_sources_dialog.dart:90`——对话框 body 是 `ConstrainedBox(maxHeight = (screenH*0.55).clamp(128, 420))` 直接包**不带任何滚动**的 `HibikiReorderableColumn`（组件本体是 `Stack`+`Column(mainAxisSize.min)`，自身不滚动，见 `hibiki/lib/src/utils/components/hibiki_reorderable_column.dart:215`）。行总高超过受限高度时 RenderFlex 溢出（出框），且无任何滚动视口 → 无法滚动看到下面的行；行少时按内容收缩显得弹窗过小。
- **[x] ① 已根因修复** — 在 `local_audio_sources_dialog.dart:90` 的 body `ConstrainedBox` 内层套 `SingleChildScrollView(child: _buildBody(tokens))`：内容超高时整体滚动而非溢出，行少时仍收缩。与「管理音频来源」主对话框 `dictionary_settings_dialog_page.dart:101` 同款已验证修法。拖拽重排与滚动不冲突——触摸屏走 `DelayedMultiDragGestureRecognizer`（长按起拖，快速滑动归滚动），鼠标走滚轮 `PointerScrollEvent` 滚动 + 按下拖动 `ImmediateMultiDragGestureRecognizer` 起拖，两者各走不同指针序列、不争用手势竞技场（见组件类注释）。未改 `HibikiReorderableColumn`（4 处复用：local_audio / media_sources / dictionary_settings / dictionary_dialog，全部向后兼容）。提交：见本轮 commit。
- **[x] ② 已加自动化测试** — `hibiki/test/pages/local_audio_sources_dialog_test.dart` 新增 `many sources scroll without overflow inside the constrained dialog (BUG-445)`：矮窗口（240px）+ 24 行，断言①无 RenderFlex 溢出异常 ②列表被外层 `Scrollable` 视口包住、组件本体仍不带滚动 ③`maxScrollExtent>0` 且 `jumpTo(max)` 后能看到尾部 `source_23`。已实测红→绿（撤掉 SingleChildScrollView 该用例失败）。
- **备注**：`media_sources_dialog.dart`（TODO-817 媒体来源）有同款结构隐患，但属另一功能链路，本 bug 不动它。
