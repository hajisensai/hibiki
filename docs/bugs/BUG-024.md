## BUG-024 · 阅读器底栏「调整」面板打开慢半拍（开面板前空跑设置同步风暴）
- **报告**：2026-06-04（用户：「打开书籍里面底栏的调整的时候，总是感觉慢半拍，加载好慢」）。
- **真实性**：✅ **真 bug（开面板前做冗余设置全量同步，触发 DB 写放大 + WebView 往返风暴）**。阅读器页 `_settings` 与 `ReaderHibikiSource.readerSettings` 是**同一个对象**：`reader_hibiki_page.dart` initState 段 `if (ReaderHibikiSource.readerSettings == null) ReaderHibikiSource.readerSettings = rs; _settings = ReaderHibikiSource.readerSettings;`（约 :333-338），二者全文件再无其它赋值点；`ReaderHibikiSource` 的 ttu* getter/setter 形如 `readerSettings?.X ?? getPreference(...)`，readerSettings 非 null 时恒走 readerSettings 分支。旧 TTU「全局 prefs ↔ profile prefs」双存储时代的 `_syncSettingsToHive`/`_syncSettingsFromHive` 因此退化成「把对象的值写回它自己」的死桥。`_showAppearanceSheet`（`:4553`）开面板前 `await _syncSettingsToHive()`（`:4566`）经 `ReaderHibikiSource.setTtu*` 把 17 个值写回自身：每个 setter 末尾 `onSettingsChangedLive?.call()`（`reader_hibiki_source.dart:550` 等）→ `_applyStylesLive`（`:1387`，内部又 `_syncSettingsFromHive` 17 次 DB 读 + 重生成 CSS + WebView `evaluateJavascript` 往返 + setState）。结果：点「调整」到面板出现之间空跑约 **17 次 DB 写 + 17×17 次 DB 读 + 17 次 WebView 往返 + 17 次整页 setState**，全是自拷贝冗余。`reader_settings.dart:67` `_set` 每次真写 `_db.setPref`、无相等短路，故每次自拷贝都是真实异步 DB 写。关面板后 `:4737` 的 `_syncSettingsFromHive` 与 `_applyStylesLive:1390` 的 `_syncSettingsFromHive` 同为自拷贝（后者每次 live 编辑还多 17 次冗余 DB 写）。根因 = 共享同一 ReaderSettings 对象后，旧双存储桥退化成自拷贝，却仍带 `setTtu*` 的 `onSettingsChangedLive` 副作用。
