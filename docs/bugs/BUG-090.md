## BUG-090 · Windows 启动时「系统主题」不吃系统主题色（恒落到硬编码 teal，从不跟随 OS 强调色）
- **报告**：2026-06-06（用户：「win 启动的时候颜色不吃主题颜色」）。采番注：本 worktree 基线最大号 BUG-089，取 090（仍可能被并发抢，以提交为准）。
- **真实性**：✅ **真 bug（沿真实代码路径 + 对照 dynamic_color 自身实现定位）**。默认 `appThemeKey` 为空时落到 `'system-theme'`（`theme_notifier.dart:160` `appThemeKey` getter），此模式本应跟随操作系统主题色。但 `refreshSystemPalette()`（`theme_notifier.dart`）**只调 `DynamicColorPlugin.getCorePalette()`**，而 `getCorePalette` 仅 Android 实现（读 `@android:color/system_*`），**Windows / macOS / Linux 恒返回 null**——这些平台的系统色只经 `getAccentColor()` 暴露（对照同版本 dynamic_color 1.8.1 自带的 `DynamicColorBuilder`：先试 `getCorePalette`，null 再回退 `getAccentColor` + `ColorScheme.fromSeed`，正是这套跨平台范式）。Hibiki 缺这条回退，于是 Windows 上 `_systemPalette` 永远是 null，`buildColorScheme` 的 system-theme 分支（旧 `theme_notifier.dart:250` `if (appThemeKey=='system-theme' && _systemPalette!=null)`）从不命中 → 直落 `buildHibikiColorScheme(seedColor: _seedColor)`，而 system-theme 的 `_seedColor` = `themePresets['system-theme']?.seed ?? 0xFF1F4959`（system-theme 不在 presets 里）= **硬编码 teal `0xFF1F4959`**。⇒ Windows 启动恒为固定 teal，从不跟随 OS 强调色。
- **[x] ① 已修复** — 本提交（自含修复+测试+本条目）。**根因修（补 accent 回退，把 system-theme 取色逻辑收敛成纯函数，非补丁）**：① 新增纯函数 `buildSystemThemeColorScheme({brightness, fallbackSeed, palette, accent})`（`theme_notifier.dart`）：有 `palette` 用之；否则 `ColorScheme.fromSeed(seedColor: accent ?? fallbackSeed)`——即正规 dynamic_color 路径。② `refreshSystemPalette` 在 `getCorePalette()` 为 null 时回退 `getAccentColor()`，新字段 `Color? _systemAccentColor` 缓存之；`systemPrimaryColor` getter 同步反映 accent。③ `buildColorScheme` 的 system-theme 分支改为无条件委托该纯函数（`palette: _systemPalette, accent: _systemAccentColor, fallbackSeed: _seedColor`），消除「palette 为 null 就静默落 teal」的旧特例。行为等价：Android（有 palette）不变；OS 无任何系统色时仍回退 teal（行为不变）；Windows/macOS/Linux 现按 OS 强调色取色（修复点）。
- **[x] ② 已加自动化测试** — `hibiki/test/models/theme_system_accent_test.dart`：① 行为——`buildSystemThemeColorScheme` 在 `palette==null` 且有 `accent` 时，`scheme.primary` 等于 `ColorScheme.fromSeed(accent).primary` 且**不等于** teal 回退方案（accent 真正驱动取色，这就是 Windows 失败路径）；② 退化——无 palette 无 accent 时落 `fallbackSeed`；③ brightness 透传；④ 源码守卫——`refreshSystemPalette` 同时含 `getCorePalette` 与 `getAccentColor`（防回退被删重蹈覆辙）、`buildColorScheme` 经 `buildSystemThemeColorScheme` 路由。验证：`flutter analyze lib/src/models/theme_notifier.dart` 0、`test/models/` 主题相关 44 绿（6 新 + 38 旧无回归）。
- **备注**：主题/平台门控类。代码正确 + 无回归；**Windows（及 macOS/Linux）桌面真机复测待用户**：把主题设为「跟随系统」，改 Windows 设置里的强调色，重启 Hibiki，确认 app 主色跟随 OS 强调色（不再恒为固定 teal）。`getAccentColor` 真值依赖原生插件，host 单测不可注入，故核心逻辑走纯函数行为测试 + 源码守卫（同既有平台门控范式）。
