## BUG-042 · 手机阅读器设置弹窗「布局与显示」子页没法上下滑动（嵌入式 shrinkWrap ListView 吃掉触摸拖动）
- **报告**：2026-06-04（用户：「手机的书籍里面的布局显示没有办法滑动」）。经 AskUserQuestion 确认 = 阅读器内点设置（齿轮/底栏）弹出的快捷设置面板 → 进「布局与显示」子页，里面选项列表手指上下滑动滚不动。采番注：动手前查 `docs/BUGS.md` 当前最大号为 BUG-041，故本条取 042。
- **真实性**：✅ **真 bug（嵌套滚动手势竞技场，Android/Material 专有），可复现**。沿真实代码路径定位：阅读器快捷设置弹窗（`reader_quick_settings_sheet.dart`）整体经 `HibikiModalSheetFrame(scrollable: true)` 包成外层 `SingleChildScrollView`（`hibiki_material_components.dart:948` 的 `Flexible(SingleChildScrollView)`）+ `AnimatedSize`；「布局与显示」子页内容由 `_buildSettingsDestinationContent`（`reader_quick_settings_sheet.dart:405`，**不传** scrollController）调 `MaterialSettingsRenderer.buildDetailContent(shrinkWrap: true)` 渲染成 `ListView.builder(shrinkWrap: true)`（`material_settings_renderer.dart:138`）。该内层 ListView shrinkWrap 后自身滚动 extent=0，但**仍安装了自己的垂直拖动手势识别器**——手指落在它的行（边距 stepper / 跨页·书写方向 segmented / 排版 switch，铺满整面板）上拖动时，内层 Scrollable 在手势竞技场里赢下手势却滚不动任何东西，手势也不冒泡给外层 `SingleChildScrollView` → 子页内容无法被触摸滚动。只在 header 空白区（ListView 之外）能滚，故体感「没法滑动」。布局组是唯一项目多到超过弹窗 0.80 屏高、真正需要滚动的 shrinkWrap 子页（行为/查词子页项少、不溢出、未触发），所以用户专挑它报。**Cupertino 渲染器同分支早已有 `physics: NeverScrollableScrollPhysics()`（`cupertino_settings_renderer.dart:134`）→ iOS 无此 bug，确认是 Android/Material 专有**。根因点 = `material_settings_renderer.dart:138` 的 shrinkWrap `ListView.builder` 缺 `NeverScrollableScrollPhysics`（注释里 “it doesn't own the scroll” 是意图，从未被强制）。
- **[x] ① 已修复** — `9061a4e6f`：根因修——给嵌入式 shrinkWrap ListView 加 `physics`，与 cupertino 对齐（消除「内层列表既不能滚又抢手势」这个特殊情况，非补丁）。`material_settings_renderer.dart` 的 shrinkWrap 分支改 `physics: scrollController == null ? const NeverScrollableScrollPhysics() : null`。**discriminator = 有没有传 scrollController**：嵌入式调用方（`reader_quick_settings_sheet`、`display_settings_page` —— 均不传 controller、靠父滚动）→ `NeverScrollable`，所有拖动冒泡到父滚动；唯一把这个 ListView 当自身滚动者用的 `hibiki_settings_page` master-detail（material 路径传 `_contentScrollController`、ListView 本身就是滚动者）→ 保留默认 physics、不回归。顺手修好同模式的 `display_settings_page` 潜伏 bug。`flutter analyze`（改动文件）0 issue、`dart format` 0 改动。
- **[x] ② 已加自动化测试** — `9061a4e6f` · `hibiki/test/settings/settings_renderer_shrinkwrap_scroll_guard_test.dart`：双层。① 行为级——忠实重建嵌套结构（外层 `SingleChildScrollView` > `Column` > 内层 `ListView.builder(shrinkWrap)`），手机视口（400×800）下 `tester.drag` **落在内层 ListView 的行上**：默认 physics 时断言外层 `offset==0`（复现 BUG-042），换 `NeverScrollableScrollPhysics` 时断言 `offset>0`（证明修复）。② 源码扫描守卫——切出两个渲染器的 `if (shrinkWrap)` 分支，断言 material 分支含 `NeverScrollableScrollPhysics` 且以 `scrollController == null` 门控、cupertino 分支含 `NeverScrollableScrollPhysics`（任何人删掉 physics 行即红）。`flutter test test/settings/ test/reader/ test/pages/` 全量 **457 绿**无回归。
- **备注**：reader/设置/嵌套滚动/手势类。代码 + 单测绿、`flutter analyze` 0 issue、`dart format` 0 改动。**真机肉眼复测原始失败路径待用户**（Android 手机）：进书 → 点设置弹快捷面板 → 进「布局与显示」→ 手指在选项行上上下滑动应能滚到底（看到「高级排版」开关组）。同修复顺带覆盖「显示设置」独立页（`display_settings_page`）同款触摸滚动。
