## BUG-370 · 手机看远端视频字幕字体/阴影偏大、进度条位置偏高
- **报告**：2026-06-21（用户：手机看远端视频，字体和字体阴影太大，进度条位置太高）
- **真实性**：✅ 部分真。进度条偏高=真 bug（运行时根因）；字体/阴影「过大」=无远端专属代码分支（需 PM 定数值）。
  - **进度条偏高根因**：移动端进度条/字幕底缘的避让几何依赖 `_videoBottomSystemInset()=MediaQuery.viewPadding.bottom`（`hibiki/lib/src/pages/implementations/video_hibiki_page.dart:4310`），作为加项进 `videoSubtitleControlsReserve` 移动端分支（`hibiki/lib/src/media/video/video_subtitle_style.dart:80-85`）与 `videoSeekBarTrackBand`（`video_subtitle_style.dart:118-121`）。沉浸隐栏只在 `initState`（`video_hibiki_page.dart:1082`）申一次；**远端** `_initRemote`（`video_hibiki_page.dart:1191`）要先 `await client.remoteVideoStreamUrls`（`:1204`）+ 下字幕再 `_applyLoad`，controller 就绪晚——若 immersiveSticky 在网络等待期被系统/用户触屏临时唤回导航栏，首个带进度条的帧读到非零 `viewPadding.bottom`，把进度条/字幕整体抬高。本地 load 快、过了这个窗口故正常。
  - **字体/阴影**：字号是绝对 36px（`video_subtitle_style.dart:176`，`_subtitleStyle.fontSize` 直传不乘缩放，`video_hibiki_page.dart:5909`），阴影粗细=`5×appUiScale`（`resolveShadowThickness`），本地与远端**同源同值无分支**（两路 `_init`/`_initRemote` 都用 `appModel.videoSubtitleStyle`，且都包同款 `HibikiAppUiScaleNeutralizer`）。远端不可能比本地大；用户感知「远端更大」更可能是被抬高的进度条把字幕一起顶到中上部 + 36px 在小屏固有偏大。**需 PM 决策是否把默认 36px 调小 / 字号按屏幕缩放**——这是设计取舍，不应在远端单独缩字（会让远端与本地不一致，反成新 bug）。
- **[x] ① 已修复** — 视频就绪（`_applyLoad` 成功 setState）后重申 `_applyVideoImmersiveMode()`（`video_hibiki_page.dart`，紧随就绪 `_refocusVideo` 之后、`_syncWindowAspectRatioLock` 之前），让慢速远端 load 完成后重隐导航栏，首帧进度条 inset 归零、几何归位。对称惠及本地远端，不碰 BUG-184 的 inset 几何（导航栏真可见时进度条仍正确避让）。桌面 no-op。提交：<PENDING>
- **[x] ② 已加自动化测试** — 源码守卫 `hibiki/test/pages/video_remote_immersive_reassert_guard_test.dart`（断言 `_applyLoad` 就绪后在窗口纵横比同步前重申 `_applyVideoImmersiveMode`）。media_kit + 真实播放/真机 inset 时序跑不了 headless。
- **备注**：字号/阴影「过大」属设计取舍待 PM（建议值：默认字号 36→30 或加「字幕随屏幕缩放」开关）。进度条偏高的运行时根因修复需**真机复测**远端 vs 本地（打点 `_videoBottomSystemInset()` 对照）。
