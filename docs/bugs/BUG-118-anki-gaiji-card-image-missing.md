## BUG-118 · 视频/书内查词制卡：词义外字(gaiji)图在 AnkiConnect 卡片上不显示
- **报告**：2026-06-08（用户：「这个制卡图片没有」+ 卡片截图；澄清为「视频查词」制卡、缺的是「词义里的外字图(gaiji)」）。
- **真实性**：✅ **真 bug（沿真实代码路径定位）**。制卡时义项 HTML 由 `popup.js createDefinitionImage`（`hibiki/assets/popup/popup.js:688-708`）在 `window.embedMedia` 为真时渲染成 `<img class="gloss-image" src="hoshi_dict_N.ext">`（占位符文件名由 `getMediaFilename` 生成），并在 `dictionaryMedia` 负载里登记 `{dictionary, path, filename}`。两个 Anki repo 制卡时遍历 `dictionaryMedia`，把媒体字节 storeMediaFile，再 `value.replaceAll(media.filename, tag)` 把字段里的占位符替换成真实引用。**AnkiDroid**（`packages/hibiki_anki/lib/src/ankidroid/anki_repository.dart:403`）经 `ankiInlineMediaReference(result)` 把 tag 裸化成纯文件名，replaceAll 得 `<img ... src="real.svg">` ✅。**AnkiConnect**（桌面端=用户路径，`packages/hibiki_anki/lib/src/ankiconnect/ankiconnect_repository.dart:453` 旧实现）却返回**完整** `'<img src="$filename">'` 标签 → replaceAll 把它塞进义项已有的 `<img src="hoshi_dict_N.ext">` 的 `src` 属性里成 `<img ... src="<img src="real.svg">">` 的**嵌套坏图** → Anki 卡片上外字 `src` 无效、不显示。根因=两端 `dictionaryMediaTags[media.filename]` 的值契约不一致：AnkiDroid 裸文件名、AnkiConnect 完整标签。查词弹窗本身能正常显示外字（走 `image://` scheme，与制卡链路无关），故症状是「弹窗里有、制卡到卡片没有」。这条链路（reader/video 共用同一 `DictionaryPopupWebView` + `writeDictionaryMediaCache`）的外字制卡此前从未用真 AnkiConnect 验证过（见 [[project_card_gaiji_and_video_cue_fixes]] 「成图需真Anki」），故嵌套 BUG 一直潜伏。非 video 专属——reader 查词制卡同样受影响。
- **[x] ① 已修复** — 提交 `bdb85e4b4`。根因修（消除两端非对称 + 统一实现，非补丁；遵用户「统一两边代码、别维护两份」）：① `AnkiConnectRepository._storeDictionaryMedia` 改经 `ankiInlineMediaReference(wrapped)` 返回**裸文件名**（与 AnkiDroid 同契约，`ankiInlineMediaReference` 作为「媒体引用→裸 inline 引用」的单一真相）；② 两个 repo 各有一份 byte 级重复的「构建 `dictionaryMediaTags` + 渲染字段 + replaceAll + normalize」逻辑下沉到 `BaseAnkiRepository.buildDictionaryMediaTags` / `buildMinedFields` 共用，杜绝再次漂移。根因 `file:line`：`packages/hibiki_anki/lib/src/ankiconnect/ankiconnect_repository.dart`（`_storeDictionaryMedia` 返回值）。
- **[x] ② 已加自动化测试** — `hibiki/test/anki/anki_dict_media_embed_test.dart`：①行为测试（经 `BaseAnkiRepository` 共享 helper）—— bare 引用 → `<img ... src="real.svg">` 无嵌套、占位符已替换；完整 `<img>` 引用 → 复现 `src="<img` 嵌套坏图（记录旧 AnkiConnect 行为）；null 引用 → 丢弃该条。②`ankiInlineMediaReference` 裸化 `<img>`/`[sound:]`。③源码守卫 —— AnkiConnect `_storeDictionaryMedia` 必经 `ankiInlineMediaReference`、不再直接 `return '<img`/`return '[sound:`。`flutter test test/anki/` 全量 53 绿。
