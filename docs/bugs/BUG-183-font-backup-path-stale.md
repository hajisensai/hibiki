## BUG-183 · 备份恢复后自定义字体不生效（字体文件未打包+配置绝对路径未重定位）
- **报告**：2026-06-11（用户：平板备份恢复到手机后，字体文件过来了、应用里也显示已开启，但全都不生效；删掉后用手机本机文件重新导入同样的字体就好了）
- **真实性**：✅ 真 bug。备份导出/恢复完全不处理自定义字体，双重缺陷：
  - **缺陷 A（字体文件未打包进备份）**：`hibiki/lib/src/sync/backup_service.dart` 的 `exportBackup` 只把 `hoshi_books` / `audiobooks` / `dictionaryResources` 三棵树打进 zip（`_collectTreeFiles`，约 line 216-223），`<appDoc>/custom_fonts` 目录不在任何 prefix 下 → 字体文件从未进入备份。
  - **缺陷 B（字体配置绝对路径未重定位）**：自定义字体配置存在 `preferences` 表 key `src:reader_ttu:{custom_fonts,app_ui_fonts,dict_fonts}`（`ReaderSettings`，`reader_settings.dart:276/279/282`），值是 `[{name,path,enabled}]` JSON，`path` 是源设备绝对路径 `<appDoc>/custom_fonts/<name>_<ts>.<ext>`（`custom_fonts_page.dart:510`）。导入时 `_rebaseContentPaths`（`backup_service.dart`）只重定位 books/audiobooks 路径，**完全不碰字体路径**。恢复后新设备 `getApplicationDocumentsDirectory()` 路径不同（`app_model.dart:1196`），配置仍指向旧绝对路径 → 加载用 `File(path).existsSync()`（`app_font_loader.dart:72` / `reader_settings.dart:491 safeFontPath`）找不到文件 → 字体不生效。重新导入走新设备真实路径才对。
- **[x] ① 已修复** — 根因修：①导出把 `custom_fonts` 树打进备份（新增 `_fontsRootDirectory` 字段 + `_fontsPrefix='custom_fonts'`，`exportBackup` 调 `_collectTreeFiles`），meta 新增 `fontsRoot` 记录源设备字体根；②导入新增 `_prepareTreeRestore(archive, _fontsPrefix, fontsRootDirectory)` 恢复字体文件树（与 books/audio 同一 prepare→commit 原子语义），并新增 `_rebaseFontPaths` 在导入末尾把三个字体 pref key 的 JSON 经纯函数 `rebaseFontListJson` 把每个 file-font 的 `path` 从 `meta.fontsRoot` 重定位到本设备字体根（系统字体 `path:null` 与不在旧根下的路径不动）；③调用点 `sync_settings_schema.dart` 导出/导入两处传 `fontsRootDirectory: <appDoc>/custom_fonts`。向后兼容：legacy 备份 meta 无 `fontsRoot` → 字体 rebase 与字体树恢复均 no-op，现有 sync 套件 637 测试全绿。提交哈希：见 commit。
  - 文件：`hibiki/lib/src/sync/backup_service.dart`、`hibiki/lib/src/sync/sync_settings_schema.dart`
- **[x] ② 已加自动化测试** — `hibiki/test/sync/backup_font_path_test.dart`（10 例）：纯函数 `rebaseFontListJson`（旧根→新根重定位 / 系统字体 null 不动 / 非旧根路径不动 / Windows 反斜杠 / 坏 JSON 不抛）+ `BackupMeta.fontsRoot` round-trip + 端到端集成（export 打包字体树并写 `fontsRoot` meta；import 把字体**文件**落到目标根 + 三个 pref key 路径 rebase 成可解析的真实文件路径；legacy 无 fontsRoot 不崩）。撤掉修复时集成断言转红。
- **备注**：撞号 182（todo-088 占用）→ 改 183。真实跨设备备份恢复（平板→手机不同 appDoc）需真机/真备份复测原始失败路径（CLAUDE.md 验证纪律），单测覆盖到逻辑层。注意 `importSettings:false`（保留本机设置）时本机字体配置经 `_restoreSettingsLayer` 换回，其路径不在源根下 → `rebaseFontListJson` 对它天然 no-op，安全。
