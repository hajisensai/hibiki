## BUG-390 · 阅读器查词高亮 evaluateJavascript 在半销毁 WebView 上抛 MissingPluginException 崩溃
- **报告**：2026-06-21（用户：TODO-678）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/pages/implementations/reader_hibiki/lookup.part.dart:64`（合并前行号）的 `_highlightAndShowPopup`：`_controller!.evaluateJavascript(...)` 在 WebView 半销毁（页面 teardown / 设置 reload 重建瞬态）时，其 per-instance method channel 的 `setMethodCallHandler(null)` 已摘除 → `evaluateJavascript` 抛 `MissingPluginException`。该处 `try` **只有 finally 没 catch** → 异常逃逸到 `onTextSelected` handler 崩。属 BUG-005（`docs/bugs/BUG-005.md`，修了 4 个 eval 点 try/catch）漏掉的查词高亮 callsite。同链路另有 `_selectTextAt`（onTap / onShiftHover / onDismissBarrierHover fire-and-forget 调用，无 catch 逃 zone）、`_clearLookupState`（onAllPopupsDismissed fire-and-forget）、歌词 cue context eval（在既有 try 块之外）三处同竞态裸点。`_controller != null` 守卫只防 null，防不了通道已摘 —— 必须 try/catch 兜底。
- **[x] ① 已修复** — 4 个查词路径 eval 点全部补 try/catch + ErrorLogService（保留正常路径零行为变化）：`_highlightAndShowPopup`（catch 后 finally 的 showDeferredPopup 照常执行，高亮失败退回 fallbackRect 弹窗仍显示）/ `_selectTextAt`（就地 try/catch no-op）/ `_clearLookupState`→`_clearSelectionJs`（fire-and-forget 经 unawaited，helper 内 try/catch）/ 歌词 cue context eval（纳入既有 `ReaderHibiki.lyricsCueContext` try 块）。提交见本 worktree。
- **[x] ② 已加自动化测试** — `hibiki/test/reader/reader_lookup_eval_guard_test.dart`（源码扫描守卫：reader 含真 InAppWebView 无法 widget 挂载，照 BUG-005 / BUG-024 成例断言查词路径 evaluateJavascript 均被 try/catch 或 ErrorLogService tag 覆盖）。
- **备注**：半销毁竞态难确定性复现（reader 含真实 InAppWebView 无法 widget 挂载），以源码扫描守卫替代；**真机复测待补**（Android 查词时页面 teardown / 设置 reload 瞬态查词，确认不再崩、弹窗照显）。
