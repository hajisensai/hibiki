## BUG-235 · 拖动视频进度条松手崩溃（seek bar onPointerUp 解引用已 dispose 的 context）
- **报告**：2026-06-13（用户：拖动进度条松手后崩溃，TODO-287；崩溃栈 `FlutterError: Null check operator used on a null value` at `MaterialDesktopSeekBarState.onPointerUp (media_kit_video/.../material_desktop.dart:987)`）
- **真实性**：真 bug。`media_kit_video` 是纯 pub.dev 依赖 `2.0.1`（`hibiki/pubspec.yaml:129` + `hibiki/pubspec.lock:1463`，原无 override）。上游源码 `material_desktop.dart` 的 `MaterialDesktopSeekBarState.onPointerUp()`（约 980 行）无条件执行 `widget.onSeekEnd?.call()`（981）+ `controller(context).player.seek(...)`（987）；`controller(context)` → `VideoStateInheritedWidget.of(context)` 解引用 `State.context`，当该 State 已 dispose 时 `context` 为 null，松手指针事件落到已销毁 State 即崩。
- **根因**：`media_kit_video-2.0.1/lib/media_kit_video_controls/src/controls/material_desktop.dart` 的 `MaterialDesktopSeekBarState`：该 State 的 `setState` 已有 `if (mounted)` 守卫（913-918），但 `onPointerUp()`（980）与 `onPointerMove()`（964）两个会解引用 `controller(context)` 的处理函数没有任何 mounted 守卫。本仓库 `hibiki/lib/src/pages/implementations/video_hibiki_page.dart` 在进/退全屏、切集时通过 `VideoControlsFocusGate` 主动卸载 controls 子树；拖动进度条时正好退全屏/切集 → 控件子树 dispose → 松手的 `onPointerUp` 解引用 null `context` 崩溃。`onPointerDown()`（973）只调 `setState`（已被 mounted 守卫），不解引用 context，无需守卫。
- **[x] ① 已修复（vendor + patch）**：把 `media_kit_video 2.0.1` vendor 到 `third_party/media_kit_video/`（整包复制，pubspec SDK 上界本已 `<4.0.0` 无需 bump）；`hibiki/pubspec.yaml` 的 `dependency_overrides` 加 `media_kit_video: { path: ../third_party/media_kit_video }`。最小 patch：在 vendored `material_desktop.dart` 的 `onPointerUp()` 与 `onPointerMove()` 函数体开头各加 `if (!mounted) return;`（与该 State 既有 `if (mounted)` 守卫风格一致，patch 处加 Hibiki 注释说明上游 UAF + 原因），分别位于 vendored 文件 `onPointerMove`(971) 与 `onPointerUp`(995)。补丁说明见 `third_party/media_kit_video/PATCHES.md`。
- **[x] ② 已加自动化测试**：`hibiki/test/third_party/media_kit_video_seekbar_guard_test.dart` 源码守卫——读 vendored `material_desktop.dart`，断言 `onPointerUp` 与 `onPointerMove` 函数体内均含 `if (!mounted) return` 守卫（防回归 patch 在后续重新 vendor 时丢失）。
- **备注**：vendored 整包约 824KB（含 example/native 平台目录，与 `third_party/flutter_inappwebview_android` 等既有 vendor 一致需入库以保 plugin 结构）。后续若升级 media_kit_video，需重新 vendor 并重打此 patch（守卫测试会失败提醒）；理想是把 `if (!mounted) return` 提 PR 给上游 media-kit/media-kit 后回退本 vendor。本修复未在真机/桌面复测松手时序，待设备环节验证原始崩溃路径。
