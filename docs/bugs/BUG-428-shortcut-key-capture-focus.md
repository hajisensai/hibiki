## BUG-428 · 快捷键录制单键经常没反应 (TODO-838)
- **报告**：2026-06-26（用户：）
- **真实性**：✅ 真 bug — 根因 `hibiki/lib/src/pages/implementations/shortcut_settings_page.dart:441-447`（录制 `Focus` 间歇拿不到 primary focus，裸单键冒泡到全局 `wrapWithGlobalNavigation` 的 Shortcuts 后被丢弃）。录制相等逻辑（`input_binding.dart` setEquals）与 `_onKeyEvent` 本身无 bug；裸字母/数字键在全局层无对应条目→无人消费无人记录→「按了没反应」。`Tab`/`Esc`/`Space` 即便焦点跑偏也被全局/框架 default 吃掉有可见后果，故普通单键尤其失灵。「经常」(间歇)正是 focus race 特征。
- **[x] ① 已修复** — `_startCapture` 中 `_captureFocusNode.requestFocus()` 原本紧跟 `setState` 同步调用，但捕获 `Focus` 在 `if (_capturing)` 条件子树里、下一帧才挂载，同步 requestFocus 命中未挂载节点是 no-op，只剩 `autofocus:true` 与对话框/路由其它可聚焦元素抢首个 primary focus。改为 `WidgetsBinding.instance.addPostFrameCallback` 在子树挂载后再 `requestFocus`（带 `mounted && _capturing` 守卫），令焦点获取确定性落在捕获节点。提交：见本轮 commit。
- **[x] ② 已加自动化测试** — `hibiki/test/shortcuts/shortcut_binding_edit_dialog_test.dart`：新增 `bare single key is captured even under global navigation Shortcuts`，在 `wrapWithGlobalNavigation`（含 Space/Tab→DoNothingIntent 全局 Shortcuts）包裹下 pump 录制对话框，开始录制后 `sendKeyEvent(LogicalKeyboardKey.keyB)` 裸单键，断言出现 `HibikiTagChip 'KeyB'` + 录制提示消失 + 对话框仍开——同时覆盖「裸单键被记录」「全局层不抢」两点。现有用例从未测裸单键（唯一字母用例带 Ctrl 修饰），此为盲区补齐。
- **备注**：focus race 为真机间歇时序，headless 测试环境 autofocus 稳定取胜不复现 race；测试作为层级回归守卫（验证修复后裸单键 + 全局 Shortcuts 共存正确），根因修通过确定性 postFrame requestFocus 消除对 autofocus 时序的依赖。未动 `input_binding.dart` 相等逻辑与全局 Shortcuts 默认表。
