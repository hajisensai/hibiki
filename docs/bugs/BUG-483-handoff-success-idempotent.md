## BUG-483 · Windows 每次启动弹出已更新至 xxx 对话框
- **报告**：2026-06-30（用户：）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/utils/misc/update_handoff.dart:575-585`（`WindowsUpdateHandoff.reconcile` 的 installed 成功分支）。触发链：`main.dart:1122` 每次 build → `reconcilePendingWindowsInstallerHandoff`（`update_checker_release.dart:830`）→ `WindowsUpdateHandoff.reconcile()` → 命中成功分支返回 `WindowsUpdateHandoffStatus.installed` → `WindowsUpdateHandoffResultDialog`（`update_checker_ui.dart:213`）每次启动弹「已更新至 $version 版本」。
- **[x] ① 已修复** — 分支 `fix-1035-handoff-idempotent` HEAD（见提交信息 TODO-1035 / BUG-483）。成功分支原本防重复**唯一**手段是删 marker 文件 `update-handoff.json`，删除被 `catch(_){}` 整个吞掉；真机上 updates 目录（`getApplicationSupportDirectory()/updates`）被杀软/索引器锁文件或权限异常导致删除失败 → marker 一直在 → 每次启动 `read()` 返回同记录、`currentVersion>=targetVersion` 恒真 → 成功弹窗每启动复弹（同版本反复）。incomplete/launchFailed 分支（:589-593）早有 `lastPromptedAppVersion` 持久幂等守卫，成功分支缺失。修复：给成功分支补同款守卫——① 先比对 `record.lastPromptedAppVersion == currentVersion`，本版本已提示过则直接 `return null`（仍 best-effort 重试删除）；② 弹之前先把 `lastPromptedAppVersion = currentVersion` 写回 marker（即便后续 delete 失败，下次启动因 lastPromptedAppVersion 命中而静默）；③ 删除从「唯一守卫」降级为 best-effort 清理。核心修复全部收敛进 `update_handoff.dart` 纯逻辑层，未碰 `update_checker_release.dart`。
- **[x] ② 已加自动化测试** — `hibiki/test/utils/misc/update_handoff_test.dart`：新增用例「reports success once and stays silent when the delete fails」——writePending→markLaunchSucceeded→reconcile(installed) 命中成功分支后，marker 仍存在（模拟删除失败），第二次 reconcile 应 `return null`（对齐既有「do not pop on every startup」风格），并断言 marker 中 `lastPromptedAppVersion` 已被写回为当前版本。
