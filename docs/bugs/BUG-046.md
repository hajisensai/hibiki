## BUG-046 · Windows 上查词点「♪」播放本地音频没声音、按钮变「✕」（播放分发漏判 Windows 盘符路径）
- **报告**：2026-06-04（用户：「点击音频播放的按钮没声音啊」→ 追问澄清 = Windows + 查词（阅读器划词弹窗）+ 已配本地音频 + 点 ♪ 变 ✕）。采番注：动手前查 `docs/BUGS.md` 当前最大号为 BUG-045，故本条取 046。
- **真实性**：✅ **真 bug（播放分发用 Unix 路径特例、漏判 Windows 盘符路径，Windows 专有），本机经验性复现 + prefs/DB 取证**。沿真实代码路径定位：阅读器划词弹窗（`dictionary_popup_webview.dart`）的 `♪` 按钮（`assets/popup/popup.js:1326 createAudioButton`）点击 → `resolveWordAudio` 解析出音频引用 → `playWordAudio` 处理器播放；解析返回 null **或** 播放返回 false 都触发 `showAudioError`（按钮变 `✕`）。对正在运行的用户实例取证：`hibiki.db` 的 `audio_source_configs` = localAudio enabled、`local_audio_dbs` = `android.db` enabled、9 子源全开、路径存在；6.2GB 的本地库 schema 正确（`entries(expression,reading,source,speaker,display,file)` + `android(file,source,data)`），用 `LocalAudioDb.queryMeta` 完全相同的 SQL 实跑 `帯広`/`食べる`/`日本語` 均命中、blob 可取（8612 字节）→ **解析阶段没问题，能解出本地文件路径**。真因在播放分发：桌面端 `LocalAudioDb.extractBlob` 把音频写到临时目录，返回 **Windows 绝对路径 `C:\Users\...\Temp\local_audio.mp3`**；而 `dictionary_popup_webview.dart` 的 `playWordAudio` 处理器（旧 626-636）、以及 `dictionary_page_mixin.dart`/`base_source_page.dart` 的 `autoReadWord`（旧 131-138 / 250-260）三处都手写同一套分支 `startsWith('file://')` / `startsWith('/')`（Unix 绝对路径）/ `startsWith('http')`，Windows 的 `C:\` 盘符路径三者全不中 → `return false`/不播放 → ✕、无声。Android/macOS/Linux 的提取路径以 `/` 开头命中 `startsWith('/')`，故只有 Windows 本地音频中招；远端音频（`lookupRemoteAudio`/`remoteAudio` 返回 http(s) URL）命中 http 分支故不受影响。根因点 = `dictionary_popup_webview.dart:626` / `dictionary_page_mixin.dart:131` / `base_source_page.dart:250` 三处把「本地路径」特例化成只认 `/…`，漏掉 Windows 盘符路径。
- **[x] ① 已修复** — `86cab5190`：根因修——消除「本地路径只认 `/…`」这个特殊情况（非补丁、非再加 `startsWith('C:')` 分支）。`tts_channel.dart` 新增纯函数 `@visibleForTesting static ResolvedAudioPlayback classifyAudioRef(String)`（空→none、`http*`→url、**其余（`file://` / Unix `/…` / Windows `C:\…`）一律→file**）+ `Future<bool> playAudioRef(String)`（按分类调 `playUrl`/`playFile`，`file://` 经 `toFilePath` 转换）。三处调用点（`dictionary_popup_webview` 的 `playWordAudio` 处理器、`dictionary_page_mixin`/`base_source_page` 的 `autoReadWord`）删掉各自手写的 Unix-only 分支、统一调 `TtsChannel.instance.playAudioRef(...)`（好品味：消除特例 + 去三处重复）。`flutter analyze`（改动文件）0 issue、`dart format` 0 改动。
