## BUG-025 · 固定布局 EPUB 封面（SVG）在阅读器里右贴边不居中、且无法点击放大
- **报告**：2026-06-04（用户，附截图：landscape 窗口里竖版封面贴在右侧约 40% 宽处、左侧大片空白；并指出本文中间的插图能点开放大，封面不行）。
- **真实性**：✅ **真 bug（SVG 封面被漏过 block 图待遇）**。沿真实路径取证：该书（`hoshi_books/28`，《転生王女と天才令嬢の魔法革命》）spine[0] `p-cover` 是 `rendition:layout-pre-paginated` + `properties="svg"` 的固定布局封面，原始 XHTML 为 `<svg width="100%" height="100%" viewBox="0 0 1443 2048"><image xlink:href="../image/cover.jpg"/></svg>`（**不是 `<img>`**）。阅读器直接把该 XHTML 当 reflowable 渲染（无封面合成，见 `reader_hibiki_page.dart` 的 `_interceptRequest`）。`_sharedInitImages`（`reader_pagination_scripts.dart:472`）只把自然尺寸 >256 的 `<img>` 升级成 `block-img` + 包进 `.block-img-wrapper`（flex 居中）+ 让 `img.block-img{cursor:pointer}`，对 `<svg><image>` 仅改 `preserveAspectRatio`、不升级。后果两条：① 封面 `<svg>` 是内联级元素、未被居中包裹，默认竖排（vertical-rl，`reader_content_styles_test` 证实是默认）reflow 下漂到最右列 = 右贴边，且 `svg{width/height:100%}` 在不确定高度的分栏列里解析不出确定盒、无法像 `.block-img-wrapper` 那样居中；② 点击路径（`reader_hibiki_page.dart` 单击分支）只在 `target.tagName === 'IMG'` 时发 `onImageTap`，`<svg>`/`<image>` 不是 IMG → 落到 `onTap` → 无法放大。根因 = `<img>` 与 `<svg><image>` 两种「整页大图」待遇不对称（特殊情况），固定布局 SVG 封面落进缝里。
- **[x] ① 已修复（代码就绪，提交见备注）** — Linus 式消除特殊情况：让大尺寸 `<svg><image>` 与 `<img>` 同等待遇。(a) `reader_pagination_scripts.dart` `_sharedInitImages`：SVG 分支排除 gaiji 后，按内层 `<image>` 的 width/height（退化时取 `viewBox` 宽高）判 >256，给 `svg.classList.add('block-img')` 并包进 `.block-img-wrapper`（与 `<img>` 同一居中包裹），`!svg.closest('.block-img-wrapper')` 防重包。(b) `reader_content_styles.dart` 新增 `svg.block-img{}` 规则：用每页 `--hoshi-image-max-*` 给确定盒（否则 `svg` 默认 300×150 塌缩）+ `margin:auto` + `cursor:pointer`，内层 `<image>` 经 `preserveAspectRatio: xMidYMid meet` 居中铺满。(c) `reader_hibiki_page.dart` 抽出 `_hoshiBlockImageUrl(target)` 统一解析：直接命中 `<img>` 取 `.src`；否则 `closest('.block-img-wrapper')` 内找 `img.block-img` 或 `svg.block-img`，SVG 取 `<image>` 的 `xlink:href`/`href` 经 `new URL(href, document.baseURI)` 解析成绝对 `https://hoshi.local/epub/...` URL（已端到端核对：cover 章 URL = `…/epub/item/xhtml/p-cover.xhtml`，解析 `../image/cover.jpg` → `…/epub/item/image/cover.jpg`，`_openImageViewer` 去 `/epub/` 前缀 join `_extractDir` 命中真实文件），单击分支改走该解析器。复用既有 `.block-img-wrapper`/图片查看器机制，非补丁式绕过。
- **[x] ② 已加自动化测试** — `test/reader/reader_svg_cover_block_image_guard_test.dart`（5 条，全绿）：① CSS 生成器断言 `svg.block-img{}` 规则存在且含确定盒 `--hoshi-image-max-*` + `margin:auto` + `cursor:pointer`；② `shellScript()` 分页/连续两模式都含 SVG 升级逻辑（`svg.classList.add('block-img')` + `block-img-wrapper` + `viewBox` 判尺寸 + gaiji 排除）；③ `reader_hibiki_page.dart` 源码扫描：`_hoshiBlockImageUrl` 存在、解析 `svg.block-img` 的 `xlink:href` 经 `document.baseURI`、单击分支走该解析器。谁把 SVG 退回内联/把点击退回只认 IMG 即红。`flutter test test/reader/` 177 绿、`flutter analyze` 0 issue、`dart format` 已跑。
