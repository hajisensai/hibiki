## BUG-402 · 桌面阅读器选中文字后无法复制（Ctrl+C / 右键复制无效）
- **报告**：2026-06-22（用户：）
- **真实性**：✅ 真 bug（平台限制下的应用层兼容层）。根因 `packages/flutter_inappwebview_windows/windows/in_app_webview/in_app_webview.cpp:105`（WebView2 合成模式）+ `packages/flutter_inappwebview_windows/lib/src/in_app_webview/...custom_platform_view.dart`（Focus 无 onKeyEvent、channel 无 setKey、native 无 SendKeyInput/MoveFocus）—— Windows 端 WebView2 走合成模式，fork 只把**鼠标**事件转发进 WebView2，**不转发键盘**。所以左键拖选能建立浏览器原生选区（BUG-368 鼠标路径），但 Ctrl+C / 右键「复制」按键到不了 WebView2 → 永远触发不了浏览器原生 `copy`。能选中、断在复制。移动端与 macOS 的 WebView 自带原生 copy，不受影响。
- **[x] ① 已修复** — 在阅读器键盘入口 `hibiki/lib/src/pages/implementations/reader_hibiki/caret.part.dart` 的 `_handleKeyEvent`（:141 起）加 Windows 复制兼容层：纯谓词 `readerShouldHandleDesktopCopy`（`hibiki/lib/src/shortcuts/reader_space_override.dart`，门控 `isWindowsPlatform` + 仅 Ctrl + 键 C）命中后调 `_copyNativeSelectionToClipboard`（同文件 helper），经 `_controller!.evaluateJavascript(ReaderSelectionScripts.nativeSelectionTextInvocation())` 取**浏览器原生选区** `window.getSelection().toString()`（刻意**不碰** `window.hoshiSelection` 查词选区——BUG-368 注释，是另一套坐标/状态），解析后 `Clipboard.setData(ClipboardData(text: ...))` 写系统剪贴板。已核对 `shortcut_defaults.dart`：reader/audiobook scope **无任何** Ctrl+C 或裸 C 绑定（`keyC` 只属 `videoToggleShaderCompare` 视频 scope，阅读器永不解析；`readerToggleBookmark`=Ctrl+D、`readerCreateCardFromPopup`=Ctrl+Enter），无冲突。空选区静默跳过不覆盖剪贴板；非 Windows / 非纯 Ctrl+C 一律不进，default 行为不变。提交哈希：__COMMIT__。
- **[x] ② 已加自动化测试** — 纯谓词真值表 `hibiki/test/shortcuts/reader_desktop_copy_test.dart`（Windows 纯 Ctrl+C 命中 / 非 Windows 不接管 / 裸 C / Ctrl+Shift+C 多修饰 / Ctrl+其它键 / Meta+C 均不接管）；JS 选区解析 `hibiki/test/reader/reader_native_selection_text_test.dart`（取 getSelection 非 hoshiSelection / JSON 引号串解码 / 裸 String / 空选区 / null / 坏 JSON 不抛 / 非 String）；源码扫描守卫 `hibiki/test/reader/reader_desktop_copy_guard_test.dart`（谓词门控 Windows + 取原生 getSelection + 写 Clipboard.setData + 不碰 hoshiSelection）。
- **备注**：复制链路涉 WebView2 + 平台键盘转发，widget 测试照不到真实复制——**必须 Windows 真机**最终验证：阅读器左键拖选 → Ctrl+C → 粘贴到记事本得到所选文字；并回归查词（单击 / Shift hover）、划词高亮、翻页拖动不变。fork 缺键盘转发是更大隐患（本任务只止血 copy，不做 fork 键盘转发——那是另开大任务）。
