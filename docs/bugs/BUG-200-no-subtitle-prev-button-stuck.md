## BUG-200 · 转场/无字幕段「上一句字幕」按钮没反应回退不了
- **报告**：2026-06-11（用户）
- **真实性**：✅ 真 bug。根因 `hibiki/lib/src/pages/implementations/video_hibiki_page.dart:2536`（移动/全屏底栏「上一句」按钮）与 `:2877`（桌面控制条 + 胶囊条共用的 `_skipCueAndPokeControls` backward 分支）都接的是**裸 `controller.skipToPrevCue()`**。`skipToPrevCue` 经 `prevCueIndexFor` 在空 cue 列表 / 已是首句 / 转场长 gap 里要么 no-op、要么一脚跳到很远的上一句——动画转场片段没配音和字幕时按「上一句」毫无反应，回退不了。
- **对称背景**：TODO-073（BUG-189）已经把对称的「下一句」按钮改成 `skipToNextCueOrSeekForward`（无字幕段前进 seekSeconds 秒、不卡住）；键盘 Ctrl+← 也早在 TODO-085/090（BUG-185）走了 `skipToPrevCueOrSeekBack`（无字幕/上句太远退化回退 seekSeconds 秒）。唯独**底栏/控制条「上一句」按钮**没跟上这套退化决策——`_skipCueAndPokeControls` 注释还明确写「上一句保持纯 skipToPrevCue（语义不退化，BUG-185）」，是 TODO-073 已对「下一句」推翻的旧决策的对称遗漏。controller 层 `skipToPrevCueOrSeekBack` 早已实现（空 cue → `seekRelative(-seekSeconds*1000)`，上句太远 → `prevSeekDecisionFor` 回退 seekSeconds 秒，开头经 `clampSeekTargetMs` clamp 到 0），本次只需把 2 处按钮接线对称对齐，不新增逻辑。
- **[x] ① 已修复** — `video_hibiki_page.dart`：底栏 skip_previous 按钮（原 2536）+ `_skipCueAndPokeControls` backward 分支（原 2877）从裸 `skipToPrevCue()` 改为 `skipToPrevCueOrSeekBack(seekSeconds: _asbConfig.seekSeconds)`，与「下一句」`skipToNextCueOrSeekForward` 完全对称；切集用的 skip_previous（「上一集」`_switchEpisode`）不动。提交：93e6962ef。
- **[x] ② 已加自动化测试** — ①源码扫描守卫 `hibiki/test/media/video/video_player_keyboard_static_test.dart`：断言 page 内不再有裸 `controller.skipToPrevCue()`、「上一句」走 `skipToPrevCueOrSeekBack`、`_skipCueAndPokeControls` backward/forward 分支对称（撤接线修复 → 转红）。②controller 行为/逻辑守卫 `hibiki/test/media/video/video_player_controller_test.dart` 新增组「TODO-119 转场/无字幕『上一句』不卡住」：转场 gap 上句太远 → 回退 seekSeconds 秒、够近 → 跳该 cue、开头边界 `clampSeekTargetMs` clamp 到 0、空 cue `skipToPrevCueOrSeekBack` 安全 no-op。
- **备注**：media_kit headless 跑不了真播放，逻辑/源码层守卫；真机回退体验留用户复测。
